## 本編

### セクション1: CSSとは何か（2.5分）

まず、CSSって何なのか、もう少しちゃんと確認しておきましょう。

CSSは、Cascading Style Sheetsの略です。カスケーディング スタイル シーツ。今は名前の意味は気にしなくて大丈夫です。「HTMLの見た目を整えるための言語」と覚えておいてください。

パートワンの最初の講義で、ウェブサイトを支える3つの言語の話をしましたね。HTMLが「構造」、CSSが「見た目」、JavaScriptが「動き」。今日からその2つ目の言語に入るわけです。

CSSの役割を具体的に言うと、こういうことです。文字の色を赤にする。背景を青にする。文字のサイズを大きくする。要素と要素の間に余白を入れる。ボタンの角を丸くする。画像を中央に配置する。スマートフォンで見たときは1列で表示して、パソコンで見たときは3列で表示する。こういった「見た目に関するすべてのこと」がCSSの守備範囲です。

逆に言うと、HTMLには見た目のことを書きません。「この文字を赤くしたい」と思っても、HTMLに色の指定はしない。見た目のことはCSSに任せる。この役割分担がウェブの基本ルールです。HTMLは「何を表示するか」、CSSは「どう表示するか」。この二つはしっかり分かれています。

なぜ分けるのかというと、その方が管理しやすいからです。たとえば、サイト全体のデザインをリニューアルしたいとき、CSSだけ書き換えればいい。HTMLには触らなくていい。コンテンツを追加したいときは、HTMLだけ書き足す。CSSには触らなくていい。お互いに影響しないから、作業が楽になるんですね。

エーアイにサイトを作ってもらったとき、「デザインはいいけど文章を変えたい」とか「文章はいいけど色を変えたい」という場面がたくさん出てきます。HTMLとCSSが分かれていれば、変えたい方だけ触ればいい。この仕組みを知っていると、エーアイへの指示もぐっと的確になりますよ。

では、そのCSSをHTMLにどうやって適用するのか。方法は3つあります。順番に見ていきましょう。

### セクション2: 方法1 — インラインスタイル（2.5分）

【スライド: CSSの3つの適用方法】

CSSをHTMLに適用する方法は3つあります。一つ目がインラインスタイル、二つ目が内部スタイルシート、三つ目が外部スタイルシートです。まず、一つ目のインラインスタイルから見てみましょう。

【スライド: インラインスタイル】

インラインスタイルは、HTMLタグの中に直接CSSを書く方法です。たとえば、パラグラフタグの文字を赤くしたいとき、こう書きます。

```html
<p style="color: red;">こんにちは</p>
```

タグの中に「スタイル イコール」と書いて、ダブルクォーテーションの中にCSSを書く。カラー コロン レッド。これで文字が赤くなります。

やっていることは単純ですよね。タグに直接「この見た目にして」と指定しているわけです。手軽だし、すぐに結果が見える。パート2で学んだ属性と同じ書き方ですね。イメージタグのソース属性やリンクタグのエイチレフ属性と同じように、タグの中に属性を追加する形です。

でも、インラインスタイルには大きな問題があります。同じ見た目にしたい要素が10個あったら、10個全部に同じスタイルを書かなきゃいけないことです。パラグラフが10個あって、全部赤くしたい。一つ一つのタグに「スタイル イコール カラー コロン レッド」と書いていく。そして、やっぱり青に変えたいとなったら、10箇所全部を直さないといけない。これは大変です。

イメージとしては、洋服を体に直接テープで貼り付けるような感じです。確かに服は着られますが、着替えるたびに全部剥がしてまた貼り直す。あまりスマートではありませんよね。

なので、実際の開発ではインラインスタイルはほとんど使いません。ただ、エーアイが生成したコードにたまに出てくることがあるので、「ああ、これはインラインスタイルだな」と認識できるようにしておくと便利です。見かけたら「タグの中にスタイルが直接書いてある方法だな」とわかれば十分です。

### セクション3: 方法2 — 内部スタイルシート（2.5分）

【スライド: 内部スタイルシート】

二つ目の方法は、内部スタイルシートです。HTMLファイルのヘッド部分に、スタイルタグを使ってCSSをまとめて書く方法です。

```html
<head>
  <style>
    p {
      color: red;
    }
  </style>
</head>
```

ヘッドタグの中にスタイルタグを書いて、その中にCSSを書きます。「パラグラフの色を赤にする」。こう書くと、そのページにある全部のパラグラフタグの文字が赤くなります。一箇所書くだけで、ページ全体に適用される。さっきのインラインスタイルよりずっと効率的ですよね。10個のパラグラフがあっても、CSSは一回書けばいい。変更するときも一箇所直せばいい。

インラインスタイルをテープで貼り付けることにたとえましたが、内部スタイルシートは服を一着ちゃんと作って着るイメージです。テープよりずっといいですよね。ただし、その服はこのページ専用です。他のページには持っていけません。

実は、内部スタイルシートにも弱点があるんです。ウェブサイトって普通、何ページもありますよね。トップページ、メニューページ、アクセスページ、お問い合わせページ。内部スタイルシートだと、それぞれのページに同じCSSをコピーして書かなければいけません。全ページのヘッダーの色を変えたいとき、全ページのスタイルタグを一つずつ直す。ページが5枚、10枚と増えてくると、これもなかなか大変です。

では、もっといい方法はないのか。全ページ共通のCSSを一箇所で管理して、変更は一回で済む方法。あります。三つ目の方法を見てみましょう。

### セクション4: 方法3 — 外部スタイルシート（3分）

【スライド: 外部スタイルシート】

三つ目の方法が外部スタイルシートです。そして、これが実際の開発で使う標準的な方法です。

外部スタイルシートは、CSSを別のファイルに書いて、HTMLから読み込む方法です。CSSだけを書いた専用のファイルを作るんですね。ファイル名は、慣例的にスタイルドットシーエスエスとすることが多いです。

HTMLファイルとCSSファイル、2つのファイルが別々にある。そして、HTMLファイルからCSSファイルを読み込む。その接続に使うのがリンクタグです。

```html
<link rel="stylesheet" href="style.css">
```

HTMLのヘッドタグの中に、このリンクタグを一行書くだけで、外部のCSSファイルを読み込めます。「レル イコール スタイルシート」はこのファイルがスタイルシートだという意味、「エイチレフ イコール スタイルドットシーエスエス」はCSSファイルの場所を指定しています。パート2で学んだリンクタグのエイチレフ属性と同じ考え方ですね。外部のファイルを指し示す。

外部スタイルシートの最大のメリットは、一つのCSSファイルを複数のHTMLページで共有できることです。トップページもメニューページもアクセスページも、みんな同じスタイルドットシーエスエスを読み込む。デザインを変えたいときは、このCSSファイル一つを修正するだけで、全ページに反映される。これがメンテナンス性です。10ページあっても100ページあっても、CSSファイルは一つ直すだけ。

たとえるなら、外部スタイルシートは「クローゼットに入っている服」です。どのページにいても、同じクローゼットから服を取り出して着られる。一度服を選び直せば、全部のページの見た目が変わる。とても効率的ですよね。

ここで一つお伝えしておきたいことがあります。実は皆さんがずっと使っているCodePenは、この外部スタイルシートの接続を裏側で自動的にやってくれているんです。CodePenのHTMLペインとCSSペインは、別々のパネルに見えますが、裏ではちゃんとつながっています。だからCSSペインに書いたスタイルが、HTMLに自動的に適用されるんですね。

実際のウェブサイトでは、今見たリンクタグを自分で書いてHTMLとCSSをつなげます。パート5のStackBlitzで自分でこの接続を書いてみますので、今は「CodePenが裏側でこの面倒な接続を代行してくれている」と理解しておけば大丈夫です。

### セクション5: 3つの方法まとめ（1.5分）

【スライド: 3つの方法の比較表】

3つの方法をまとめましょう。

インラインスタイルは、タグに直接書く。手軽だけど、同じスタイルを何度も書かなきゃいけない。メンテナンスが大変。服をテープで貼るようなものでしたね。手軽だけど、着替えが面倒。

内部スタイルシートは、ページのヘッド内にまとめて書く。一箇所でそのページ全体に適用できるけど、他のページには共有できない。そのページ専用の服でした。一着はちゃんとしているけど、他のページには着回せない。

外部スタイルシートは、別ファイルにCSSを書いて読み込む。複数ページで共有でき、変更も一箇所で済む。クローゼットの服でしたね。どの部屋からでもアクセスできる。

実際のウェブサイト制作では、外部スタイルシートが標準です。エーアイの出力はさまざまですが、実務では外部CSSが基本です。ですので、パート3の実習はすべてCodePenのCSSペインを使います。これは外部スタイルシートと同じ考え方ですね。

では、理屈の話はここまでにして、実際にCSSを書いてみましょう。CodePenを開いてください。

### セクション6: CSSの基本構文（2分）

CodePenを開きましたか？ここからは一緒に手を動かしていきます。新しいペンを作って始めましょう。まずHTMLペインに簡単なHTMLを用意します。

```html
<h1>私のページ</h1>
<p>CSSで見た目を変えてみます。</p>
```

見出しとパラグラフを一つずつ書きました。プレビューには黒い文字で見出しと段落が表示されているはずです。おなじみの、素朴な見た目ですね。ここにCSSを加えて変身させていきます。

さて、CSSを書く前に、CSSの書き方のルールを確認しておきましょう。CSSには決まった形があります。

【スライド: CSSの基本構文】

セレクタ、波括弧の中にプロパティ コロン 値 セミコロン。これがCSSの基本の形です。

もう少し日本語で言い換えると、「どの要素に」「何を」「どう変えるか」。この3つを指定するのがCSSです。たとえば「h1の文字の色を青にする」なら、h1が「どの要素に」、カラーが「何を」、ブルーが「どう変えるか」です。

注意してほしいのは、プロパティと値の間のコロンと、値の後のセミコロンです。コロンは縦に二つの点、セミコロンは点とカンマが組み合わさった記号ですね。この二つの記号を忘れるとCSSが効かなくなります。最初は忘れがちですが、何度か書いていくうちに手が覚えますので心配はいりません。

セレクタについてはこの次の講義で詳しく学びますので、今日はHTMLのタグ名をそのままセレクタとして使う、一番シンプルな形だけで進めます。では、実際にCSSペインに書いてみましょう。

### セクション7: 初めてのCSS — 背景色を変える（4分）

いよいよCSSを書きます。CodePenのCSSペインをクリックしてください。HTMLペインの隣にあるパネルですね。今は空っぽのはずです。

まず、ページ全体の背景色を変えてみましょう。CSSペインにこう書いてください。

```css
body {
  background-color: lightblue;
}
```

ボディ、波括弧開き、バックグラウンド カラー コロン ライトブルー セミコロン、波括弧閉じ。

書けましたか？プレビューを見てください。

おっ、変わりましたよね。背景が水色になりました。たった3行のCSSで、ページの印象ががらっと変わりました。これがCSSの力です。

一つずつ見ていきましょう。ボディというのがセレクタです。HTMLのボディタグ、つまりページ全体を指しています。CodePenではHTMLペインにボディタグを書いていませんが、ブラウザが自動的にボディタグで囲んでくれています。波括弧の中にスタイルの指定を書きます。バックグラウンドカラーが「背景色」というプロパティで、ライトブルーがその値です。最後のセミコロンは、文末のピリオドのようなものです。この指定はここで終わりですよ、という区切りの記号ですね。

では、もう少しスタイルを追加してみましょう。CSSペインに続けて書いてください。

```css
h1 {
  color: darkblue;
}
```

h1がセレクタで、カラー コロン ダークブルー。プレビューを見てください。見出しの文字が濃い青色に変わりましたね。水色の背景に濃い青の見出し。さっきよりだいぶ雰囲気が出てきました。

HTMLには一切手を加えていません。CSSペインに書くだけで、見た目が変わる。この感覚を覚えておいてください。HTMLとCSSは別々に管理する。これがウェブ制作の基本です。

もう一つ追加してみましょう。

```css
p {
  color: darkgreen;
  font-size: 18px;
}
```

パラグラフの文字色をダークグリーンに、文字サイズを18ピクセルにしました。プレビューを見ると、段落の文字が緑色になって、少し大きくなっていますね。

ここで一つ大事なポイントです。一つの波括弧の中に、プロパティと値のペアを複数書くことができます。今、カラーとフォントサイズ、2つの指定を書きましたよね。3つでも4つでも、プロパティはいくつでも追加できます。それぞれの末尾にセミコロンを忘れないようにしてくださいね。

さて、ここまで書いたCSSを振り返ってみましょう。ボディの背景色をライトブルーに。h1の文字色をダークブルーに。パラグラフの文字色をダークグリーンに、文字サイズを18ピクセルに。さっきまでモノクロで素朴だったページが、色が入ってずいぶん雰囲気が変わりましたよね。

ここで皆さん、試しにCSSペインのライトブルーの部分を、別の色に変えてみてください。たとえばライトイエローとか、ライトピンクとか、ラベンダーとか。書き換えてプレビューを見ると、すぐに背景色が変わるはずです。このリアルタイムで結果が見える感覚が、CSSの楽しさです。色の名前がわからなくても大丈夫ですよ。CSSで使える色の名前はたくさんあって、全部で140色以上あります。暗記する必要はまったくありません。必要なときに「CSSカラーネーム」で検索すれば一覧が出てきますから。好きな色に変えて、少し遊んでみてくださいね。

これがCSSの醍醐味です。HTML側は何も変えていないのに、CSSを書くだけで見た目がどんどん変わっていく。HTMLが骨格で、CSSが服。今まさに骨格に服を着せる体験をしてもらいました。この先の講義で、色だけでなく、フォント、余白、配置など、もっといろいろなプロパティを学んで、本格的なデザインに近づけていきますよ。今日の3行ずつのCSSでもこれだけ変わるんですから、プロパティを組み合わせればどんなデザインだって表現できます。

## まとめ（1.5分）

今日の講義のポイントを3つ振り返りましょう。

一つ目。CSSをHTMLに適用する方法は3つ。インラインスタイル、内部スタイルシート、外部スタイルシート。実際に使うのは外部スタイルシートです。CodePenのCSSペインは外部スタイルシートと同じ仕組みですので、これからの実習はすべてCSSペインを使っていきます。

二つ目。CSSの基本構文は、セレクタ、波括弧、プロパティ コロン 値 セミコロン。「どの要素の」「何を」「どう変えるか」を指定する。この形はこの先ずっと使いますので、しっかり覚えておいてくださいね。

三つ目。HTMLとCSSは役割が違います。HTMLは構造、CSSは見た目。それぞれ別に管理するのが基本です。HTMLには見た目のことを書かない。見た目はCSSに任せる。この役割分担がウェブ制作の大事なルールです。エーアイにサイトを作ってもらったとき、HTMLファイルとCSSファイルが分かれていたら、「ああ、ちゃんと役割分担されているな」とわかりますよね。

次回の講義では、セレクタをもっと詳しく学びます。今日はh1やpといったタグ名をそのままセレクタとして使いましたが、「この段落だけ色を変えたい」「このボタンだけ大きくしたい」というとき、タグ名だけでは足りませんよね。同じパラグラフタグなのに、一つだけ違うスタイルにしたい。そういうときに使うのが、クラスやIDという仕組みです。エーアイが生成したCSSでよく見かける「ドット」や「シャープ」の記号の謎が解けますよ。楽しみにしていてくださいね。
