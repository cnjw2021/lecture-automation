## 本編

### セクション1: フォームとは何か（3分）

まず、フォームってどういうものか、もう少し具体的に確認しておきましょう。

皆さんが普段使っているウェブサイトで、何かを入力する場面を思い出してみてください。たとえば、お店のサイトで予約をするとき、名前や電話番号を入力しますよね。あるいは、Google で何かを検索するとき、検索ボックスにキーワードを入力して Enter を押します。ネットショッピングで住所やクレジットカード番号を入力するのもそうですし、ログイン画面でメールアドレスとパスワードを入力するのもそうです。

これらはすべて「フォーム」です。HTMLの世界では、ユーザーに何かを入力してもらう仕組みのことを、フォームと呼びます。言い方を変えると、フォームは「ウェブサイトとユーザーの対話窓口」のようなものですね。ここまではウェブサイトがユーザーに一方的に情報を伝えるだけでしたが、フォームがあることで、ユーザーの方からウェブサイトに情報を送ることができるようになります。

今日は、このフォームをHTMLで組み立てる方法を一から学んでいきます。

では、さっそくCodePenを開きましょう。今日は新しいペンを作って始めます。前回まで使っていた自己紹介ページのペンとは別に、新しいペンを作成してください。左上のメニューから新しいペンを作成できますね。

HTMLの入力エリアに、こう書いてみましょう。

```html
<form>

</form>
```

このフォームタグが、フォーム全体を囲む「入れ物」になります。前回やったメインタグやセクションタグと同じ考え方ですね。中に入力欄やボタンを入れていく、コンテナの役割です。

今の状態ではプレビューには何も表示されません。フォームタグ自体は目に見えるものではないんです。メインタグやセクションタグも目に見えませんでしたよね。あくまでも「ここからここまでがフォームですよ」という意味をHTMLに持たせるためのタグです。

ここから、このフォームタグの中に、一つずつ入力欄を追加していきます。完成するころにはお問い合わせフォームになっていますので、楽しみにしていてくださいね。

### セクション2: テキスト入力、インプットタグ（6分）

フォームの中にまず入れるのは、インプットタグです。名前のとおり、ユーザーが何かを「入力する」ための部品ですね。

フォームタグの中に、こう書いてみてください。フォームの開始タグと閉じタグの間に、2行追加します。

```html
<form>
  <label>お名前</label>
  <input type="text">
</form>
```

二つの新しいタグが出てきましたね。一つずつ見ていきましょう。

まず、ラベルタグ。これは入力欄の「名前札」のようなものです。「お名前」と書いてあれば、ユーザーは「ああ、ここに名前を入れるんだな」とわかりますよね。この名前札がないと、何を入力すればいいのかユーザーに伝わりません。フォームを作るときは、必ず入力欄の前にラベルを書く癖をつけておきましょう。

そして、インプットタグ。これが実際の入力欄です。プレビューを見てください。「お名前」という文字の横に、小さなテキストボックスが表示されていますよね。ここにユーザーが文字を入力できるようになっています。試しにプレビューのテキストボックスをクリックして、何か文字を入力してみてください。自分の名前でも、好きな言葉でも構いません。

入力できましたか？これが、フォームの基本です。ラベルで「何を入力するか」を伝えて、インプットで「入力する場所」を用意する。この組み合わせを覚えておいてくださいね。

インプットタグの中に「タイプ イコール テキスト」と書いてありますね。このタイプという部分で、入力欄の種類を指定しています。テキストは「一行のテキスト入力」という意味です。名前や住所など、短い文章を入力してもらうときに使います。このタイプの指定がフォーム作りのポイントになっていくので、ここからいくつかのタイプを見ていきましょう。

さて、ここでインプットタグの面白い特徴を一つお伝えします。インプットタグには閉じタグがありません。普通のタグは開始タグと終了タグがペアになっていますよね。たとえばラベルタグは、開始のラベルと閉じのラベルがペアです。でもインプットタグは一つだけで完結します。こういうタグを以前少しだけ触れましたね。イメージタグも閉じタグがありませんでした。最初は不思議に感じるかもしれませんが、すぐ慣れますので安心してください。

では、次にもう一つ入力欄を追加しましょう。メールアドレスの入力欄です。インプットタグの下に、ラベルとインプットのペアをもう一組追加してみてください。

```html
<form>
  <label>お名前</label>
  <input type="text">

  <label>メールアドレス</label>
  <input type="email">
</form>
```

インプットタグをもう一つ追加しました。今度は「タイプ イコール イーメール」になっていますね。プレビューを見ると、見た目はさっきのテキストとほとんど同じに見えます。でも実は、中身が少し違うんです。

タイプがイーメールの場合、ブラウザが「これはメールアドレスの入力欄だ」と認識してくれます。たとえば、アットマークが入っていない文字列を入力して送信しようとすると、ブラウザが「メールアドレスの形式が正しくありません」と教えてくれることがあります。私たちが何も追加のコードを書かなくても、タイプを正しく指定するだけで、ブラウザが入力内容をチェックしてくれるわけですね。とても便利です。

このように、インプットタグはタイプを変えるだけで、いろいろな種類の入力欄になります。暗記する必要はありません。「インプットにはタイプがある。タイプを変えると入力欄の種類が変わる」ということだけ覚えておけば、必要なときに調べればいいんです。

### セクション3: もっといろんなインプットタイプ（4分）

テキストとイーメールの2つのタイプを見てきました。インプットタグにはまだまだタイプがあります。今日はあと二つだけ見ておきましょう。

まず、タイプがテルの場合です。テルは英語のテレフォンの略で、電話番号用の入力欄です。フォームに電話番号の入力欄を追加してみましょう。メールアドレスの入力欄の後に、ラベルとインプットのペアを追加します。

```html
<form>
  <label>お名前</label>
  <input type="text">

  <label>メールアドレス</label>
  <input type="email">

  <label>電話番号</label>
  <input type="tel">
</form>
```

追加できましたか？今度はタイプがテルです。プレビューを見ると、見た目はテキストタイプとほとんど同じですよね。でも、タイプをテルにしておくと、ブラウザが「これは電話番号の入力欄だ」と認識してくれます。たとえばスマートフォンでこのフォームを開いたとき、普通のキーボードではなく、数字のキーパッドが最初から表示されるんです。ユーザーが電話番号を入力しやすくなる、ちょっとした気遣いですね。タイプをイーメールにするとメールアドレスのチェックがかかったように、テルにすると電話番号に適した入力体験になるわけです。

もう一つ、タイプがデイトの場合です。お問い合わせフォームに予約日の入力欄も追加しましょう。電話番号の後に書き足してください。

```html
<form>
  <label>お名前</label>
  <input type="text">

  <label>メールアドレス</label>
  <input type="email">

  <label>電話番号</label>
  <input type="tel">

  <label>ご予約日</label>
  <input type="date">
</form>
```

プレビューで確認してみてください。新しい入力欄が追加されていますね。この入力欄をクリックすると、年月日を選べるカレンダーが開きますよね。日付を選ぶためのインターフェースが、自動的に表示されるんです。自分でカレンダーの仕組みをプログラミングする必要はまったくなくて、タイプにデイトと書くだけでブラウザが用意してくれます。これだけで、予約日の入力が簡単にできるようになるんですね。

ここまでで、テキスト、イーメール、テル、デイトの4つのタイプを見てきました。実はインプットタグのタイプは他にもたくさんあるんです。でも、全部覚える必要はまったくありません。今日紹介した4つが基本中の基本です。他に必要になったときは「HTMLインプットタイプ一覧」で検索すれば、すぐに見つかります。何を検索すべきかを知っていれば、それで十分です。

### セクション4: 複数行テキストとセレクトボックス（4分）

ここまではインプットタグを見てきましたが、フォームで使う部品はインプットタグだけではありません。ここでは、あと2つの大事な部品を紹介します。

まず一つ目、テキストエリアタグです。お問い合わせフォームには、メッセージを自由に書いてもらう欄が必要ですよね。「お問い合わせ内容をお書きください」みたいな、長い文章を入力するところです。でも、インプットタグのテキストタイプは一行しか書けません。名前やメールアドレスなら一行で十分ですが、メッセージは何行にもなることがあります。そういう場合に使うのが、テキストエリアタグです。

フォームの中、予約日の入力欄の後に追加してみましょう。ただし、この後すぐセレクトタグも追加するので、順番としてはセレクトの後にテキストエリアを置きます。まずはテキストエリアだけ先に書いてみましょう。

```html
  <label>メッセージ</label>
  <textarea></textarea>
```

今までのインプットタグと違うところに気づきましたか？テキストエリアタグには閉じタグがありますね。開始タグのテキストエリアと、閉じタグのテキストエリアで囲む形です。インプットタグは閉じタグがなかったので、ここが違うポイントです。

プレビューを見てください。今までの入力欄より大きな、複数行入力できるテキストボックスが表示されていますよね。右下の角をマウスでドラッグすると、テキストエリアのサイズを変えることもできます。試してみてください。お問い合わせ内容やコメントなど、まとまった文章を書いてもらいたいときにぴったりの部品です。

では次に、二つ目の部品、セレクトタグを見てみましょう。セレクトタグは、ドロップダウンメニューとも呼ばれます。いくつかの選択肢の中から一つを選んでもらいたいときに使います。たとえば、お問い合わせの種類を「予約」「質問」「その他」から選んでもらう、といった場面ですね。

お問い合わせの種別を選べるようにしてみましょう。ご予約日の入力欄の後、メッセージの欄の前に追加します。

```html
  <label>お問い合わせ種別</label>
  <select>
    <option>ご予約</option>
    <option>ご質問</option>
    <option>その他</option>
  </select>
```

セレクトタグの中に、オプションタグが3つ並んでいますね。一つのオプションタグが、ドロップダウンの一つの選択肢に対応しています。プレビューで実際にクリックしてみてください。「ご予約」「ご質問」「その他」の3つの選択肢が表示されて、そこから一つを選べるようになっていますよね。

選択肢を増やしたいときは、オプションタグを追加するだけです。5個でも10個でも、必要なだけ並べられます。たとえば美容室のサイトなら「カット」「カラー」「パーマ」「トリートメント」といった施術メニューをオプションに並べればいいわけですね。とてもシンプルです。

ここまでで、フォームの中に使う主要な部品がそろいました。インプットタグで短い一行の入力、テキストエリアタグで複数行の長い入力、セレクトタグで選択肢から選ぶ。この3つの部品を組み合わせれば、たいていのフォームは組み立てることができます。エーアイにフォームを作ってもらったときも、この3つの部品が使われているのが読み取れるようになりますよ。

### セクション5: プレースホルダーとリクワイアド（3.5分）

フォームの部品はそろいましたが、もう少し使いやすくする工夫をしてみましょう。今度はタグではなく、「属性」を二つ紹介します。属性というのは、タグに追加情報を付け足す仕組みでしたね。以前やったイメージタグのソース属性や、リンクタグのエイチレフ属性と同じ考え方です。

一つ目は、プレースホルダー属性です。お名前のインプットタグを、こう書き換えてみてください。

```html
  <input type="text" placeholder="山田太郎">
```

タイプの後に、半角スペースを空けて「プレースホルダー イコール 山田太郎」と追加しました。プレビューを見てください。入力欄の中にうっすらと「山田太郎」と表示されていますね。この薄いグレーの文字がプレースホルダーです。

プレースホルダーの役割は、「ここにはこういう形式で情報を入力してくださいね」という見本を、入力欄の中に表示することです。ユーザーがその入力欄をクリックして、実際に文字を入力し始めると、プレースホルダーの文字は自動的に消えます。あくまでもヒントなので、入力の邪魔にはなりません。試しにクリックして何か入力してみてください。薄い文字が消えますよね。

では、他の入力欄にもプレースホルダーを追加していきましょう。メールアドレスの欄にはこう書きます。

```html
  <input type="email" placeholder="example@mail.com">
```

メールアドレスの形式の見本が表示されるので、ユーザーは「ああ、こういう形で入力すればいいんだ」とすぐにわかります。

電話番号の欄にも追加しましょう。

```html
  <input type="tel" placeholder="09012345678">
```

テキストエリアにもプレースホルダーは使えます。

```html
  <textarea placeholder="お問い合わせ内容をご記入ください"></textarea>
```

それぞれの入力欄に、薄い文字で入力の見本やヒントが表示されるようになりましたね。ユーザーにとって、何を入力すればいいのかがひと目でわかるようになりました。ちょっとした工夫ですが、フォームの使いやすさがぐっと上がります。プロが作ったウェブサイトのフォームにも、ほぼ必ずプレースホルダーが設定されていますよ。

さて、二つ目の属性は、リクワイアド属性です。リクワイアドは英語で「必須」という意味です。お名前のインプットタグを、もう一度書き換えてみましょう。

```html
  <input type="text" placeholder="山田太郎" required>
```

末尾にリクワイアドと追加しました。プレースホルダーと違って、リクワイアドにはイコールの後の値がありません。書くだけでオンになる属性です。

この属性の効果を確認してみましょう。お名前の欄を空のままにして、フォームの送信ボタンを押してみてください。ボタンはまだ作っていませんが、後で試しますので、ここでは「リクワイアドが付いた入力欄が空のまま送信しようとすると、ブラウザが自動的にストップをかけてくれる」ということを覚えておいてください。「このフィールドを入力してください」のようなメッセージが表示されます。

これ、すごいことなんです。自分でチェックする仕組みをプログラミングする必要がなくて、リクワイアドと一言書くだけで、ブラウザが自動的にチェックしてくれるんです。HTMLの属性だけでここまでやってくれるのは、本当にありがたいですね。

メールアドレスの欄にもリクワイアドを追加しておきましょう。

```html
  <input type="email" placeholder="example@mail.com" required>
```

お名前とメールアドレスは必須、電話番号や予約日は任意、という形にしました。実際のお問い合わせフォームでもよくあるパターンですよね。必須項目と任意項目をリクワイアドで簡単に分けることができます。

### セクション6: ボタンで送信（2.5分）

フォームの入力欄はすべてそろいました。最後に必要なものは何でしょうか。そう、送信ボタンです。入力が終わったら「送信する」を押して、情報を送るためのボタンですね。

フォームの一番下、テキストエリアの後にボタンタグを追加しましょう。閉じのフォームタグの直前に書いてください。

```html
  <button>送信する</button>
```

プレビューにボタンが表示されましたね。「送信する」と書かれたボタンです。ボタンタグは、開始タグと終了タグの間に書いた文字がボタンの表示テキストになります。「送信する」でも「送信」でも、「お問い合わせを送る」でも、好きな文字を入れられます。

ボタンタグの書き方は、他のタグと同じですよね。開始タグと閉じタグで文字を囲む。パラグラフタグや見出しタグと同じ構造です。HTMLのパターンが少しずつ体に染みてきた感じがしませんか。新しいタグが出てきても、「ああ、開始タグと閉じタグで囲むんだな」と予測できるようになっていれば、もう立派にHTMLが読める状態に近づいています。

さて、それでは試しにボタンをクリックしてみましょう。プレビューで「送信する」ボタンを押してみてください。

何か起きましたか？ページが一瞬ちらっとしたかもしれませんが、特に変化はないですよね。送信ボタンを押したのに、何も起きない。これはなぜなのか、後ほどお話しします。

ここで先ほどのリクワイアドの動作も確認してみましょう。お名前の欄を空のまま、送信ボタンを押してみてください。「このフィールドを入力してください」のようなメッセージがお名前の入力欄のそばに表示されましたよね。これがリクワイアド属性の効果です。ブラウザが自動的に「必須項目が空ですよ」と教えてくれているんです。これならユーザーが入力を忘れたまま送信してしまうことを防げますね。

### セクション7: 完成したフォームを確認しよう（3分）

お疲れさまです。ここまでで、お問い合わせフォームの全体が完成しました。CodePenのHTMLエリアを上から見てみましょう。こういうコードになっているはずです。

```html
<form>
  <label>お名前</label>
  <input type="text" placeholder="山田太郎" required>

  <label>メールアドレス</label>
  <input type="email" placeholder="example@mail.com" required>

  <label>電話番号</label>
  <input type="tel" placeholder="09012345678">

  <label>ご予約日</label>
  <input type="date">

  <label>お問い合わせ種別</label>
  <select>
    <option>ご予約</option>
    <option>ご質問</option>
    <option>その他</option>
  </select>

  <label>メッセージ</label>
  <textarea placeholder="お問い合わせ内容をご記入ください"></textarea>

  <button>送信する</button>
</form>
```

上から順番に振り返ってみましょう。

まず、フォームタグで全体を囲んでいます。これがフォームの入れ物、コンテナですね。

次に、ラベルとインプットのペアが4つ並んでいます。お名前はタイプがテキストで一行のテキスト入力、メールアドレスはタイプがイーメールでメール形式のチェック付き、電話番号はタイプがテルで電話番号用、予約日はタイプがデイトでカレンダーから日付を選べる。インプットタグのタイプを変えるだけで、それぞれの用途に合った入力欄になっていましたよね。

お名前とメールアドレスにはリクワイアド属性がついていて、空のまま送信しようとするとブラウザが止めてくれます。各入力欄にはプレースホルダーで入力の見本が薄く表示されています。

お問い合わせ種別にはセレクトタグを使って、ドロップダウンで「ご予約」「ご質問」「その他」から選べるようにしました。メッセージにはテキストエリアタグを使って、複数行の自由な入力ができるようにしています。

そして最後に、ボタンタグで送信ボタンを置いています。

今はまだCSSを当てていないので、見た目はかなり素朴ですよね。入力欄がただ並んでいるだけで、あまりきれいとは言えません。でも、フォームとしての「骨組み」はしっかりできています。見た目をきれいに整えるのはCSSの仕事です。パート3以降で、余白を調整したり、色をつけたり、ボタンのデザインを変えたりする方法を学んでいきます。

プレビューで実際に操作してみてください。名前を入力して、メールアドレスを入力して、種別をドロップダウンから選んで、メッセージを書いて、送信ボタンを押す。ウェブサイトのお問い合わせページで見かける、あの一連の流れが、自分の手で作れましたね。ここまでよくがんばりました。

### セクション8: このフォームを実際に動かすには？（2.5分）

さて、先ほど送信ボタンをクリックしても何も起きなかった理由をお話ししましょう。

フォームに入力した情報は、どこかに届けなくてはいけませんよね。お問い合わせフォームなら、お店の人にメールで届いてほしいわけです。予約フォームなら、予約の情報がお店のシステムに届いてほしい。

でも、今私たちが書いたHTMLだけでは、入力された情報を受け取って処理する仕組みがないんです。HTMLができるのは、あくまでも「入力欄を画面に表示すること」まで。入力されたデータを「どこかに届ける」のは、HTMLだけではできません。

これを処理するのは、サーバーと呼ばれるコンピュータの仕事です。サーバーというのは、ウェブサイトを動かしている裏側のコンピュータですね。フォームに入力されたデータは、このサーバーに送られて、そこで「メールとして送信する」とか「データベースに保存する」といった処理が行われます。

たとえるなら、HTMLで作ったフォームは「紙の申込用紙」のようなものです。用紙はできた。でも、書き込んだ用紙を「受付の窓口」に届ける仕組みがまだない。サーバーがその「受付の窓口」に当たるわけですね。

「えっ、じゃあサーバーの仕組みまで自分で作らないといけないの？」と思った方、心配しないでください。サーバー側の処理はこの講座の範囲外ですが、実際にはそういった仕組みを簡単に用意できるサービスもあります。

今日大事なのは、HTMLでフォームの「見た目」と「構造」を作れるようになったことです。これが基礎になります。今は「HTMLで入力欄を並べて、フォームの骨組みを作る。実際にデータを送るにはサーバーが必要。でも今はHTMLで見た目を作る練習をしている」と、ざっくり把握しておけば大丈夫です。焦る必要はまったくありません。一歩一歩進んでいきましょう。

## まとめ（1.5分）

今日の講義のポイントを3つ振り返りましょう。

一つ目。フォームタグの中に、インプット、テキストエリア、セレクト、ボタンといった部品を並べることで、ユーザーから情報を受け取るフォームが作れます。ラベルタグで入力欄の名前を付けるのも忘れないでくださいね。

二つ目。インプットタグはタイプを変えるだけで、テキスト、イーメール、テル、デイトなど、いろいろな種類の入力欄に変わります。全部暗記する必要はありません。必要なときに「インプットタイプ」で検索すれば見つかります。

三つ目。プレースホルダー属性で入力のヒントを表示でき、リクワイアド属性で必須入力のチェックをブラウザに任せられます。たった一言追加するだけで、フォームがぐっと使いやすくなりましたよね。

今日作ったお問い合わせフォームは、美容室やネイルサロン、教室のサイトでも、そのまま使える基本形です。今回はサロンのお問い合わせフォームを想定して作りましたが、たとえば整体院なら「施術メニュー」をセレクトに入れたり、英会話教室なら「希望の時間帯」を追加したり。項目を入れ替えるだけで、いろいろな業種に対応できます。もちろん、見た目はこれからCSSで整えていきますが、HTMLの骨組みとしては十分に実用的なものが完成しました。

さて、次回の講義ではテーブルを学びます。テーブルというのは、データを行と列で整理して表示する仕組みです。たとえば料金表やメニュー表など、情報をきれいに並べたいときにとても便利です。次回もCodePenで一緒に作っていきましょう。
