## 本編

### セクション1: Flexbox以前の世界（2.5分）

【スライド: Flexbox以前のレイアウト方法】

まず、Flexboxがなぜこんなに大事なのか。その「ありがたみ」を感じてもらうために、少しだけ昔の話をしましょう。

Flexboxが広く使われるようになる前、ウェブ制作者たちは要素を横に並べるために「フロート」という仕組みを使っていました。フロートというのは、もともと新聞記事のような表現を作るためのもので、画像の横にテキストを回り込ませる機能です。新聞や雑誌で、写真の横に本文がくっついているレイアウトを見たことがありますよね。あれをウェブで再現するための機能がフロートです。

ところが、当時はほかに要素を横並びにする手段がなかったので、みんなこのフロートを無理やりレイアウト全体に使っていたんです。本来の目的とは違う使い方をしていたわけですね。

結果どうなるかというと、とにかくレイアウトが崩れやすい。高さがずれる。「クリアフィックス」という呪文のようなおまじないコードを毎回書かないといけない。初心者はもちろん、プロのウェブ制作者でもフロートレイアウトには相当苦労していました。当時を知っている人に聞くと、「もう二度と戻りたくない」って口を揃えて言いますね。

【スライド: Flexboxの登場】

そこに登場したのがFlexboxです。正式名称は「フレキシブルボックスレイアウト」。Flexboxは「レイアウトのために作られた仕組み」です。フロートが「本来の目的と違う使い方」で無理をしていたのに対して、Flexboxは最初から「要素を柔軟に並べる」ことを目的に設計されています。だからシンプルで、直感的で、崩れにくい。

今日のウェブサイトのほとんどがこのFlexboxを使ってレイアウトされていますし、AIにコードを生成させても、まず間違いなくFlexboxが使われています。つまり、Flexboxを理解しておくと、AIが生成したCSSを読む力が大きく上がるということです。では、その仕組みを見ていきましょう。

### セクション2: Flexboxの核心概念（4分）

【スライド: コンテナとアイテム】

Flexboxを使うときに、まず知っておいてほしい概念が2つあります。「コンテナ」と「アイテム」です。

コンテナというのは、並べたいものを囲む親要素のことです。そしてアイテムは、その中に入っている子要素のことです。

具体的にイメージしてみましょう。たとえば、ナビゲーションバーを思い浮かべてください。ナビゲーション全体を囲んでいる枠がありますよね。これがコンテナです。そしてその中に「ホーム」「商品」「お問い合わせ」といったリンクが並んでいます。このひとつひとつがアイテムです。

あるいは、カード型のレイアウトなら、カードを並べるエリア全体がコンテナ。中の個別のカードがアイテムですね。

Flexboxの仕組みは、この「コンテナに対してCSSを指定すると、中のアイテムの並び方が変わる」という構造になっています。これがとても大事なポイントです。アイテムの一つ一つに何かを指定する必要はほとんどありません。親であるコンテナにCSSを書くだけで、中の子要素が自動的に整列してくれる。これがFlexboxの便利なところです。

【スライド: 主軸と交差軸の図解】

もうひとつ、大事な概念があります。「主軸」と「交差軸」です。ちょっと聞き慣れない言葉ですが、図で見るとシンプルです。

主軸というのは、アイテムが並ぶ方向の軸です。Flexboxの初期状態では、主軸は横方向。つまり左から右に向かう方向です。そして交差軸は、主軸に対して垂直な方向の軸です。主軸が横なら、交差軸は縦。つまり上から下に向かう方向になります。

なぜこの2つの軸が大事かというと、Flexboxでは「主軸方向の配置」と「交差軸方向の配置」を別々のプロパティでコントロールするからです。たとえば、「アイテムを横方向に中央寄せしたい」なら主軸の設定を変えます。「アイテムを縦方向に中央寄せしたい」なら交差軸の設定を変えます。

ちょっと抽象的に聞こえるかもしれませんが、大丈夫です。この後すぐにコードを書いて実際に動きを見ますから、そこで「あ、こういうことか」と実感できるはずです。では、CodePenを開いて、手を動かしていきましょう。

### セクション3: display: flex — 魔法の1行（4分）

では、CodePenを開いてください。新しいペンを作りましょう。まず、HTMLにシンプルなボックスを3つ作ります。

```html
<div class="container">
  <div class="box">1</div>
  <div class="box">2</div>
  <div class="box">3</div>
</div>
```

外側のディブにコンテナというクラス名をつけました。これがFlexboxのコンテナ、つまり親要素になります。中に入っている3つのディブがアイテム、つまり子要素ですね。今の状態だと、CSSを何もつけていないので、3つのディブは縦に積み重なっています。ディブはブロック要素なので、1つずつ改行されて下に並ぶ。これが初期状態です。

では、ボックスが見やすいように、まずスタイルを当てましょう。CSSペインにこう書いてください。

```css
.box {
  background: #3b82f6;
  color: white;
  padding: 20px 40px;
  font-size: 24px;
  border-radius: 8px;
}
```

はい、青いボックスが3つ、縦に並んでいますね。数字の1、2、3がそれぞれ見えています。この3つを横に並べたいわけです。

では、ここからが今日のハイライトです。コンテナに、たった1行のCSSを追加してみましょう。

```css
.container {
  display: flex;
}
```

どうでしょうか。ボックスが横に並びましたよね。たった1行、ディスプレイの値をフレックスに変えただけで、中のアイテムが横並びになりました。これがFlexboxの基本中の基本です。親要素、つまりコンテナにディスプレイ・フレックスを指定するだけ。たったこれだけで、中身が横に並ぶ。とてもシンプルですよね。

ここで何が起きているのか整理しましょう。ディスプレイ・フレックスを指定した瞬間に、このコンテナは「フレックスコンテナ」になります。そして中の子要素たちは自動的に「フレックスアイテム」になります。さっきスライドで説明した主軸は、初期状態では横方向。だからアイテムが左から右へ横に並んだわけです。

この「ディスプレイ・フレックス」は、これからものすごく頻繁に出てきます。AIが生成するCSSでも、必ずと言っていいほど見かけます。横並びレイアウトの出発点はいつもこの1行です。

### セクション4: justify-content — 主軸方向の配置（4.5分）

横に並んだのはいいですが、3つのボックスが全部左に寄っていますよね。これを中央に配置したり、均等にばらけさせたりしたい。そんなときに使うのが、ジャスティファイコンテントというプロパティです。

ジャスティファイコンテントは「主軸方向のアイテムの配置」を決めるプロパティです。今、主軸は横方向ですから、横方向にアイテムをどう並べるかを指定できるわけですね。さっき説明した「主軸」が、ここで効いてきます。

では、まずセンターにしてみましょう。コンテナのCSSにジャスティファイコンテント・センターを追加します。

```css
.container {
  display: flex;
  justify-content: center;
}
```

はい、3つのボックスがまとまって中央に寄りましたね。ウェブサイトでコンテンツをページの中央に配置したいとき、非常によく使うパターンです。

では次に、値をスペースビトウィーンに変えてみましょう。

```css
.container {
  display: flex;
  justify-content: space-between;
}
```

今度はどうなったでしょうか。最初のアイテムが左端、最後のアイテムが右端に寄って、その間に均等なスペースができました。「両端に寄せて、間を均等に開ける」という配置ですね。たとえばナビゲーションバーで、左側にロゴ、右側にメニューを配置したいとき。まさにこのスペースビトウィーンが使えます。

もうひとつ見てみましょう。スペースアラウンドです。

```css
.container {
  display: flex;
  justify-content: space-around;
}
```

今度は各アイテムの「周り」に均等なスペースができていますね。スペースビトウィーンとの違いに注目してください。ビトウィーンは左端と右端のアイテムがコンテナの端にぴったりくっつきますが、アラウンドは左端と右端にもスペースが入ります。アイテムの「周囲」にスペースを均等に配置するのがアラウンドです。

では、ここまでのジャスティファイコンテントを整理しましょう。よく使う値は3つです。センターで中央に寄せる。スペースビトウィーンで両端揃えにして間を均等に開ける。スペースアラウンドで全方向に均等なスペースを入れる。全部覚えなくても大丈夫です。「横方向の配置を変えたいな」と思ったときに、「ジャスティファイコンテントだったな」と思い出せれば、あとは値を調べて試せばいいんです。

### セクション5: align-items — 交差軸方向の配置（3.5分）

次は交差軸方向、つまり今の場合は縦方向の配置です。

今のままだと効果がわかりにくいので、コンテナに高さと背景色をつけてみましょう。

```css
.container {
  display: flex;
  justify-content: center;
  height: 300px;
  background: #f1f5f9;
}
```

グレーの背景がコンテナの範囲を示しています。コンテナには300ピクセルの高さがありますが、ボックスは上に寄っていますよね。この状態を変えるのがアラインアイテムズです。アラインアイテムズは「交差軸方向のアイテムの配置」を決めます。今は交差軸が縦方向なので、縦方向の位置を変えられるわけですね。

では、センターをつけてみましょう。

```css
.container {
  display: flex;
  justify-content: center;
  align-items: center;
  height: 300px;
  background: #f1f5f9;
}
```

はい、ボックスが縦方向にも中央にきました。ジャスティファイコンテントのセンターが横方向の中央寄せ。アラインアイテムズのセンターが縦方向の中央寄せ。この2つを組み合わせると、要素を「上下左右のど真ん中」にピタッと配置できます。

これ、実はCSSで要素を完全に中央配置する一番簡単な方法なんです。昔はこれだけのことをするのにも複雑なテクニックが必要でしたが、Flexboxなら2行で済みます。ディスプレイ・フレックス、ジャスティファイコンテント・センター、アラインアイテムズ・センター。この3行セットは「CSS中央配置の定番パターン」として覚えておくととても便利です。

アラインアイテムズにはセンター以外にも値があります。フレックススタートで上寄せ、フレックスエンドで下寄せ、ストレッチにするとアイテムがコンテナの高さいっぱいに引き伸ばされます。よく使うのはセンターですので、まずはそれだけ押さえておけば大丈夫です。

### セクション6: flex-direction — 並ぶ方向の切り替え（3分）

ここまで、アイテムは横方向に並んでいましたよね。でも、場合によっては縦方向に並べたいこともあります。たとえばサイドバーの中のメニューリンクは、上から下に並んでいることが多いです。そういうとき、フレックスディレクションというプロパティを使います。

一度、コンテナのCSSをシンプルに戻しましょう。高さと背景色を消して、ディスプレイ・フレックスだけの状態にします。

```css
.container {
  display: flex;
  flex-direction: column;
}
```

フレックスディレクションをカラムに変えると、アイテムが縦に並びましたね。カラムというのは「列」という意味で、縦方向を表します。

これ、一見すると「ディスプレイ・フレックスを付ける前と同じじゃないか」と思うかもしれません。見た目は似ていますが、中身は違います。Flexboxが適用された状態なので、ジャスティファイコンテントやアラインアイテムズが引き続き使えるんです。つまり、縦並びの中で中央寄せや均等配置ができるということですね。

ここでひとつ注意点があります。フレックスディレクションをカラムにすると、主軸と交差軸が入れ替わります。主軸が縦方向になるので、ジャスティファイコンテントは「縦方向の配置」をコントロールするようになり、アラインアイテムズは「横方向の配置」をコントロールするようになります。ちょっとややこしいですよね。でも、「ジャスティファイコンテントは常に主軸方向」「アラインアイテムズは常に交差軸方向」この原則を覚えておけば迷いません。

では、ローに戻しておきましょう。ローは「行」、つまり横方向のことです。

```css
.container {
  display: flex;
  flex-direction: row;
}
```

これがデフォルトの状態です。フレックスディレクションを書かなければ、自動的にローになります。

### セクション7: gap — アイテム間の間隔（2.5分）

横に並んだボックスを見てください。ボックス同士がぴったりくっついていますよね。これだと少し窮屈に見えます。アイテムの間にスペースを開けたいときはどうすればいいでしょうか。

前回のボックスモデルで学んだマージンを使う方法もありますが、Flexboxにはもっと便利な専用のプロパティがあります。ギャップです。

```css
.container {
  display: flex;
  gap: 20px;
}
```

ギャップを20ピクセルに設定しました。アイテム同士の間に20ピクセルの隙間ができましたね。きれいに間隔が空いています。

ギャップのいいところは、「アイテムの間だけ」にスペースが入ることです。マージンだと、最初のアイテムの左側や最後のアイテムの右側にも余白がついてしまうことがあって、計算が面倒になります。でもギャップなら、アイテムとアイテムの間にだけスペースが入るので、余計なことを考えなくて済みます。

Flexboxでアイテム間のスペースを設定するときは、マージンではなくギャップを使う。これが今のスタンダードなやり方です。AIが生成するコードでもギャップがよく使われていますので、見かけたら「アイテム間の隙間の設定だな」とわかるようにしておきましょう。

### セクション8: flex-wrap — 折り返し（3分）

では、ここでちょっと実験してみましょう。今ボックスは3つですが、HTMLにボックスを2つ追加して5つに増やしてみます。

```html
<div class="container">
  <div class="box">1</div>
  <div class="box">2</div>
  <div class="box">3</div>
  <div class="box">4</div>
  <div class="box">5</div>
</div>
```

5つに増えましたね。画面の幅にもよりますが、ボックスが少し細くなっていませんか。Flexboxの初期状態では、全部のアイテムを無理やり1行に収めようとします。そのためにアイテムの幅を自動的に縮めてしまうんですね。

でも、ボックスの幅は維持したまま、入りきらないものは次の行に折り返したいこともありますよね。商品一覧のページなんかがそうです。商品が10個あったら、全部を1行に詰め込むのではなく、3つずつ折り返して並べたいはずです。

そういうときに使うのが、フレックスラップです。

```css
.container {
  display: flex;
  flex-wrap: wrap;
  gap: 20px;
}
```

フレックスラップをラップに変えると、コンテナの幅に収まりきらないアイテムが自動的に次の行に送られます。ブラウザの幅を縮めたり広げたりしてみてください。幅に応じてアイテムが折り返されるのがわかりますよね。

商品一覧、写真ギャラリー、タグの一覧など、数が変わる可能性があるレイアウトには、このフレックスラップが非常に便利です。次回学ぶレスポンシブデザインでも、このフレックスラップが活躍しますので、覚えておいてくださいね。

### セクション9: 実習1 — ナビゲーションバーを作ろう（5分）

さて、ここまでFlexboxの基本プロパティをひと通り見てきました。ディスプレイ・フレックスで横並びに。ジャスティファイコンテントで横方向の配置を。アラインアイテムズで縦方向の配置を。ギャップで間隔を。フレックスラップで折り返しを。これだけのプロパティで、いろんなレイアウトが作れます。

では、ここからは実際によく見かけるレイアウトを2つ作ってみましょう。まずはナビゲーションバーです。

CodePenで新しいペンを開いてください。HTMLにこのように書きましょう。

```html
<nav class="navbar">
  <div class="logo">MyShop</div>
  <ul class="nav-links">
    <li><a href="#">ホーム</a></li>
    <li><a href="#">商品</a></li>
    <li><a href="#">お問い合わせ</a></li>
  </ul>
</nav>
```

ナブタグの中に、ロゴとメニューリンクがあります。ナブタグはPART 2で学んだセマンティックタグですね。ナビゲーションであることを示すタグでした。

今の状態だと、ロゴが上にあって、その下にリンクが箇条書きで表示されています。これを「左にロゴ、右にメニュー」というよく見るナビゲーションバーのレイアウトにしていきましょう。

まず、ナブバー全体にFlexboxを適用します。

```css
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 16px 32px;
  background: #1e293b;
}
```

ディスプレイ・フレックスで横並びにして、ジャスティファイコンテントをスペースビトウィーンにします。スペースビトウィーンにすると、ロゴが左端、メニューリンクのリストが右端に寄りますよね。そしてアラインアイテムズをセンターにして、縦方向の高さも揃えます。背景色は濃い紺色を指定しました。

さっそくそれっぽくなってきましたね。でも、メニューリンクがまだ縦に並んでいます。リストの箇条書きの点も邪魔ですね。メニューリンクを囲んでいるのはユーエルタグですから、こちらにもFlexboxを適用しましょう。

```css
.nav-links {
  display: flex;
  gap: 24px;
  list-style: none;
}
```

リストスタイルをナンにして箇条書きの点を消しました。そしてディスプレイ・フレックスで横並びに、ギャップで24ピクセルの間隔をつけました。メニューが横に並びましたね。

最後に、文字色を整えましょう。

```css
.logo {
  color: white;
  font-size: 20px;
  font-weight: bold;
}

.nav-links a {
  color: #94a3b8;
  text-decoration: none;
}
```

ロゴを白色の太字に。リンクはグレーの文字にして、テキストデコレーションをナンにすることで下線を消しました。

いかがでしょうか。左にロゴ、右にメニューという、よく見かけるナビゲーションバーが完成しました。今回、Flexboxを2か所に使いましたよね。ナブバー全体の横並びにひとつ、メニューリンクの横並びにもうひとつ。こういうふうに、Flexboxは入れ子にして使うことがよくあります。外側のコンテナの中にあるアイテムが、さらに別のコンテナになって、その中にアイテムが並ぶ。この「入れ子構造」が使いこなせると、かなり複雑なレイアウトも作れるようになります。

### セクション10: 実習2 — カード3枚を横に並べる（5分）

もうひとつ、よくあるレイアウトを作ってみましょう。カード型のレイアウトです。お店のメニューや、サービスの紹介ページで、3つのカードが横に並んでいるデザインを見たことがありますよね。あれをFlexboxで作ります。

CodePenで新しいペンを開いてください。HTMLにこう書きましょう。

```html
<div class="cards">
  <div class="card">
    <h3>カット</h3>
    <p>丁寧なカウンセリングで、お似合いのスタイルをご提案します。</p>
    <span class="price">¥4,400</span>
  </div>
  <div class="card">
    <h3>カラー</h3>
    <p>ダメージを抑えながら、理想の色味に仕上げます。</p>
    <span class="price">¥6,600</span>
  </div>
  <div class="card">
    <h3>パーマ</h3>
    <p>柔らかい質感のデジタルパーマが人気です。</p>
    <span class="price">¥8,800</span>
  </div>
</div>
```

美容室のメニューカードをイメージしました。今はカードが縦に3枚並んでいますね。これを横並びにして、きれいなカードレイアウトにしていきましょう。

まず、カードを囲んでいるコンテナにFlexboxを適用します。

```css
.cards {
  display: flex;
  gap: 24px;
  padding: 32px;
}
```

ディスプレイ・フレックスとギャップだけで、3枚が横並びになりましたよね。ここまでは先ほどと同じパターンです。ここからカード自体のスタイルを整えていきましょう。

```css
.card {
  background: white;
  padding: 24px;
  border-radius: 12px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
  flex: 1;
}
```

白い背景に、角を丸くして、ボックスシャドウで影をつけました。ボーダーレイディアスとボックスシャドウは次回以降のCSSレシピ集でも出てきますが、カードデザインの定番ですね。

ここでひとつ注目してほしいプロパティがあります。フレックス・1です。これはアイテム側に指定するプロパティで、「利用可能なスペースを均等に分け合う」という意味になります。3つのカードにそれぞれフレックス・1を指定しているので、コンテナの幅を3等分して、同じ幅で並んでいるわけです。このフレックス・1は、カード型レイアウトをきれいに揃えたいときにとても便利です。

残りのスタイルも書きましょう。

```css
.card h3 {
  margin-bottom: 8px;
}

.price {
  display: block;
  margin-top: 16px;
  font-size: 20px;
  font-weight: bold;
  color: #3b82f6;
}

body {
  background: #f8fafc;
  font-family: sans-serif;
}
```

はい、美容室のメニューページっぽいカードレイアウトが完成しました。どうでしょうか、なかなかそれっぽく見えませんか。

今回は美容室を例にしましたが、整体院なら施術コース、教室ならレッスン紹介、飲食店ならおすすめメニューに、内容を置き換えるだけでそのまま使えます。ディスプレイ・フレックス、ギャップ、そしてフレックス・1。この3つのプロパティの組み合わせだけで、こういうきれいなカード型レイアウトが作れるんです。

### セクション11: Flexboxをもっと練習したい方へ（1.5分）

【スライド: Flexbox Froggy紹介】

今日はたくさんのプロパティが出てきましたよね。ディスプレイ・フレックス、ジャスティファイコンテント、アラインアイテムズ、フレックスディレクション、ギャップ、フレックスラップ。一度にこれだけ出てくると、正直まだしっくり来ていない部分もあるかもしれません。

そんな方にぴったりのゲームがあります。Flexboxフロッギーというブラウザゲームです。画面にカエルがいて、このカエルを正しい蓮の葉の上に乗せるために、Flexboxのプロパティを書いていくというパズルゲームです。

たとえば「カエルを右に寄せてください」というお題が出たら、ジャスティファイコンテントをフレックスエンドにする。「2匹のカエルを離してください」と出たら、スペースビトウィーンにする。実際にプロパティを書いて、カエルが動くのを見ながら学べるので、楽しみながら練習できます。

全部で24ステージあって、今日の講義で学んだ内容がちょうどカバーされています。「Flexboxフロッギー」で検索するとすぐに見つかります。日本語版もありますので、復習がわりにぜひ遊んでみてください。

## まとめ（2分）

それでは、今日のまとめです。

今日はFlexboxを学びました。大事なポイントを3つ振り返ります。

1つ目。コンテナにディスプレイ・フレックスをつけると、中のアイテムが横に並ぶ。Flexboxの出発点はこの1行です。そしてコンテナとアイテムという「親子関係」がFlexboxの基本構造です。

2つ目。配置はジャスティファイコンテントとアラインアイテムズの2つで決まる。ジャスティファイコンテントが主軸方向、アラインアイテムズが交差軸方向。「どっちの軸の話をしているのか」を意識するだけで、Flexboxはぐっとわかりやすくなります。

3つ目。ギャップでアイテム間のスペースを設定し、フレックスラップで折り返しを制御できる。そしてフレックス・1を使えば、アイテムが均等な幅で並びます。

今日のFlexboxは、AIが生成するCSSコードの中に必ずと言っていいほど出てきます。ジャスティファイコンテント、アラインアイテムズ、ギャップ。これらのプロパティ名を見たときに、「ああ、レイアウトの設定をしているんだな」とわかるだけで、コードの理解度がまったく違ってきます。暗記する必要はありません。何をやっているのか読めること。それが大事です。

次回は、レスポンシブデザインを学びます。今日作ったカードレイアウト、パソコンの画面では3枚横並びできれいに表示されていますが、スマートフォンの狭い画面だとどうなるでしょうか。画面のサイズに応じてレイアウトを自動的に切り替える方法を学んでいきます。今日学んだフレックスラップも、レスポンシブデザインで大活躍しますよ。お楽しみに。お疲れ様でした。
