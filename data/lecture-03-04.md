## 本編

### セクション1: すべてのHTML要素は「箱」だ（5分）

【スライド: ボックスモデル概念図】

CSSでとても大事な考え方があります。それは、「すべてのHTML要素は箱である」ということです。

h1の見出しも、pの段落も、divも、画像も、ボタンも、ブラウザから見ると全部「四角い箱」なんです。丸く見えるボタンも、実は四角い箱の角を丸くしているだけなんですね。目には見えませんが、すべての要素はこの四角い箱の中に収まっています。

この箱の仕組みのことを「ボックスモデル」と呼びます。CSSでレイアウトを考えるときの、一番の基本になる考え方です。今日この概念を理解すれば、CSSの半分は分かったも同然と言ってもいいくらい大事な知識です。

では、この箱がどういう構造になっているか見てみましょう。ボックスモデルは、内側から外側に向かって、4つの領域で構成されています。

【スライド: ボックスモデル4層ダイアグラム】

一番内側が「コンテント」です。これは実際の中身の部分ですね。テキストだったり、画像だったり、その要素が持っている本来のコンテンツが入る領域です。

その外側が「パディング」です。パディングは、コンテントと箱の境界線の間にある余白です。箱の「内側」の余白と覚えてください。たとえば、ボタンの文字と枠線の間にある余白がパディングです。パディングがないと、文字が枠線にぴったりくっついてしまって窮屈な見た目になりますよね。

さらにその外側が「ボーダー」です。ボーダーは箱の境界線そのものです。線を引くこともできますし、線を引かずに透明のままにすることもできます。カードやボタンの枠線がボーダーですね。

そして一番外側が「マージン」です。マージンは箱の外側の余白で、他の要素との間にできるスペースのことです。二つの段落の間に空いている隙間、あれがマージンです。

この4つ、コンテント、パディング、ボーダー、マージン。内側から外側へ、この順番です。

ちょっと身近なもので例えてみましょう。宅配便の段ボール箱をイメージしてください。箱の中に入っている商品、これがコンテントです。商品が傷つかないように周りに詰めてある緩衝材、プチプチとか新聞紙とか、あれがパディングですね。段ボール箱そのものの壁、これがボーダーです。そして、段ボール箱を棚に並べるとき、隣の箱との間に空けるスペース、これがマージンです。

この比喩を覚えておくと、パディングとマージンの違いがすっきり理解できると思います。パディングは箱の内側の余白、マージンは箱の外側の余白。内側か外側か、この違いが本当に大事です。

もう一つ、大事なポイントがあります。パディングには背景色が適用されますが、マージンには適用されません。パディングは箱の一部なので、箱に背景色を付ければパディング部分にも色がつきます。一方、マージンは箱の外側なので、常に透明です。これは実際にCSSを書いてみると、すぐに実感できますよ。

### セクション2: CSSでボックスモデルを体験する（6分）

では、実際にCSSで書いてみましょう。CodePenを開いてください。前回まで使っていた自己紹介ページのペンを使いましょう。

まず、ボックスモデルの効果を分かりやすく見るために、HTMLに新しいdivを一つ追加しておきます。

```html
<div class="box-demo">
  ボックスモデルの実験
</div>
```

クラス名をボックスデモとしたdivの中に、テキストを入れただけのシンプルな構造です。

次に、CSSでこの箱にスタイルを当てていきます。まず、背景色と横幅だけ指定してみましょう。

```css
.box-demo {
  background-color: #e8f5e9;
  width: 300px;
}
```

背景色を薄いグリーンにして、横幅を300ピクセルに指定しました。プレビューで薄いグリーンの箱が表示されていますね。テキストが箱の左端にぴったりくっついているのが分かります。ちょっと窮屈ですよね。

では、ここにパディングを追加してみましょう。

```css
.box-demo {
  background-color: #e8f5e9;
  width: 300px;
  padding: 20px;
}
```

パディングを20ピクセル追加しました。プレビューを見てください。テキストと箱の端の間にゆとりができましたよね。上下左右に20ピクセルずつの余白が生まれました。これがパディングの効果です。箱の内側に余白ができて、テキストが枠にくっつかなくなりました。背景色の薄いグリーンがパディング部分にも広がっているのが分かりますか。パディングは箱の一部なので、背景色がつくんですね。

次にボーダーを追加します。

```css
.box-demo {
  background-color: #e8f5e9;
  width: 300px;
  padding: 20px;
  border: 2px solid #2d6a4f;
}
```

ボーダーは少し書き方が独特です。「2px solid」と書くと、「2ピクセルの太さで、実線で」という意味になります。ソリッドが「実線」ですね。ほかにも、ダッシュドと書くと点線になったり、ドテッドと書くと点々になったりしますが、今はソリッドだけ覚えておけば大丈夫です。最後のシャープ2d6a4fは前回学んだHEXの色指定ですね。深いグリーンの色です。

プレビューを見ると、薄いグリーンの箱の周りに、濃いグリーンの線がきれいに引かれましたね。これがボーダーです。

最後に、マージンを追加しましょう。

```css
.box-demo {
  background-color: #e8f5e9;
  width: 300px;
  padding: 20px;
  border: 2px solid #2d6a4f;
  margin: 30px;
}
```

マージンを30ピクセルにしました。プレビューを見てください。この箱と、ページの端や他の要素の間にスペースができたのが分かりますか。これがマージンの効果です。箱の外側に余白を作って、他の要素と間隔を取っています。そして、マージンの部分は透明ですよね。背景色の薄いグリーンは、ボーダーの内側まで。マージンには色がつきません。

ここまでで、4つの領域がすべて揃いました。コンテント、パディング、ボーダー、マージン。一つずつ追加していくと、それぞれの役割がよく分かりますよね。パディングは箱の内側に余白を作り、ボーダーは枠線を引き、マージンは箱の外側に余白を作る。この3つの使い分けが、CSSのレイアウトの基本になります。

では、パディングとマージンの違いをもう少しはっきり見てみましょう。今のCSSに背景色がついていますよね。薄いグリーンです。この背景色がどこまで広がっているか、注目してください。テキストの周りの余白、つまりパディングの部分まで薄いグリーンが広がっています。でも、ボーダーの外側のマージン部分には色がついていません。

この違いはとても重要です。たとえばボタンを作るとき、ボタンの文字と枠線の間にゆとりが欲しいですよね。そのときはパディングを使います。パディングを広げれば、ボタンの背景色ごと大きくなります。一方、ボタンとボタンの間にスペースを空けたいときはマージンを使います。マージンは透明なスペースなので、隙間ができるだけです。「背景色の中の余白はパディング、背景色の外の余白はマージン」と覚えておくと、迷うことが少なくなりますよ。

### セクション3: 開発者ツールでボックスモデルを確認する（4.5分）

さて、ここまでは自分でCSSを書いてパディングやマージンを設定しましたが、実は、ブラウザの開発者ツールを使えば、どんなウェブサイトでもボックスモデルを目で見て確認できるんです。これがものすごく便利な機能なので、ぜひ覚えておいてください。

Chromeの開発者ツール、覚えていますか。PART 2で少しだけ触りましたね。画面を右クリックして「検証」を選ぶか、キーボードのF12キーを押すと開きます。

CodePenのプレビュー画面上で、今作った薄いグリーンの箱を右クリックして「検証」を選んでみてください。開発者ツールが開いて、HTMLの該当箇所がハイライトされたと思います。

では、右側のパネルの下の方を見てください。カラフルな四角い図が表示されていませんか。四角が何重にも重なったような図です。

これがボックスモデルの図です。開発者ツールが、まさに今日学んだ4つの領域を色分けして見せてくれているんです。一番内側の青い部分がコンテント、その外側の緑色の部分がパディング、さらに外側の黄色い部分がボーダー、そして一番外側のオレンジ色の部分がマージンです。

それぞれの数値もピクセル単位で表示されていますね。先ほど設定した値が確認できると思います。コンテントが300ピクセル、パディングが上下左右20ピクセル、ボーダーが2ピクセル、マージンが30ピクセル。CSSで書いた値と一致していますよね。

この開発者ツールのボックスモデル図は、ウェブ制作でものすごく便利です。たとえば、「なんでこの要素とこの要素の間にこんなに隙間があるんだろう」と思ったとき。その要素を右クリックして検証を選んで、ボックスモデルを見れば、パディングのせいなのかマージンのせいなのか、すぐに分かります。余白の原因が特定できれば、あとはその値をCSSで変えるだけです。

さらに便利なことに、開発者ツール上でこのボックスモデル図の数値を直接変更することもできます。たとえば、パディングの数値をダブルクリックして、20を40に書き換えてみてください。プレビューがリアルタイムで更新されて、パディングが広がったのが分かりますよね。「この余白、もうちょっと広げたらどうなるかな」「もう少し狭くしたらどうだろう」というのを、CSSを書き直さなくてもその場で試せるんです。

もちろん、開発者ツール上での変更は一時的なものなので、ページを更新すると元に戻ります。でも、「ちょうどいい値はいくつかな」と試行錯誤するときにはとても便利です。いい感じの値が見つかったら、その数値をCSSに書き写せばいいわけですね。AIが生成したCSSの余白を微調整したいときにも、この方法が使えますよ。

ちなみに、今はCodePenで自分が書いたCSSを見ていますが、この方法はどんなウェブサイトにも使えます。たとえば、お気に入りのウェブサイトを開いて、気になる要素を右クリックして検証を選べば、そのサイトのボックスモデルが丸見えです。「このサイトのカードのパディングは何ピクセルなんだろう」「このボタンの余白はどうなっているんだろう」と、プロが作ったサイトのレイアウトを参考にすることもできます。

### セクション4: パディングとマージンの個別指定（3分）

先ほどは、パディングもマージンも一つの値だけ指定しましたが、上下左右を個別に設定することもできます。たとえば、「上は広めに余白を取りたいけど、左右はそこまで要らない」ということ、ありますよね。

パディングを個別に指定する書き方を見てみましょう。

```css
.box-demo {
  padding-top: 10px;
  padding-right: 20px;
  padding-bottom: 10px;
  padding-left: 20px;
}
```

このように、パディングトップ、パディングライト、パディングボトム、パディングレフトと書けば、上、右、下、左を個別に指定できます。マージンも同じで、マージントップ、マージンライトというように書けます。

ただ、毎回4行も書くのは少し面倒ですよね。そこで、ショートハンドという短縮した書き方があります。

```css
.box-demo {
  padding: 10px 20px;
}
```

値を2つ書くと、最初の値が上下、2番目の値が左右に適用されます。つまり「上下が10ピクセル、左右が20ピクセル」という意味ですね。

値を4つ書くこともできて、その場合は上、右、下、左の順番で個別に指定できます。

```css
.box-demo {
  padding: 10px 20px 10px 20px;
}
```

時計回りと覚えてください。上から始まって、右、下、左と時計回りに進みます。

実務では、2つの値を使うパターンが一番多いです。上下の余白と左右の余白をまとめて指定できるので便利なんですね。

では、実際にCodePenで試してみましょう。先ほどのボックスデモのパディングを、このショートハンドに変えてみてください。

```css
.box-demo {
  padding: 10px 30px;
}
```

上下が10ピクセル、左右が30ピクセルになりました。プレビューを見ると、上下は狭く、左右は広い余白になっていますね。こうやって上下と左右で違う余白をつけたいとき、このショートハンドが便利です。

もちろん、この書き方を暗記する必要はありません。「上下左右を個別に指定できる書き方がある」と知っていれば、必要なときに調べたりAIに聞いたりすれば大丈夫です。AIが生成したCSSの中に「パディング 16ピクセル 24ピクセル」のように2つの値が並んでいても、「ああ、上下と左右で違う余白を設定しているんだな」と分かれば十分です。

### セクション5: box-sizing: border-box（4分）

さて、ボックスモデルにはもう一つ、とても大事なポイントがあります。先ほどの箱に戻って考えてみましょう。

横幅を300ピクセルに設定しましたよね。パディングが左右20ピクセルずつ、ボーダーが左右2ピクセルずつ入っています。では質問です。この箱が画面上で実際に占める横幅は、何ピクセルでしょうか。

300ピクセルだと思いますか。実は違うんです。

【スライド: ボックスモデルのサイズ計算】

実際には、300プラス20プラス20プラス2プラス2で、合計344ピクセルになります。横幅300ピクセルと指定したのに、実際の箱は344ピクセルもある。なんだかおかしいですよね。

これは、CSSのデフォルトの動作では、横幅の指定がコンテント領域だけに適用されるからです。300ピクセルというのは、コンテントの幅だけ。パディングとボーダーはその「外側」に追加されます。つまり、横幅300ピクセルと書いても、パディングやボーダーを足すと、実際の箱はもっと大きくなってしまうんです。

これはレイアウトを組むときにかなり厄介です。「2つの箱を300ピクセルずつ横に並べて600ピクセルにしよう」と思っても、パディングやボーダーを足すと600ピクセルを超えてしまって、横に並ばなくなったりするわけです。

そこで登場するのが、ボックスサイジングというプロパティです。

```css
.box-demo {
  box-sizing: border-box;
}
```

ボックスサイジングにボーダーボックスと指定すると、横幅300ピクセルの中にパディングとボーダーが含まれるようになります。つまり、300ピクセルと書いたら、パディングやボーダーを含めて本当に300ピクセルの箱になる。コンテント領域が自動的に小さくなって、全体でちょうど300ピクセルに収まるように調整されます。

これ、すごく直感的ですよね。「横幅300ピクセル」と書いたら、見た目の幅が300ピクセルになる。当たり前のようですが、このプロパティを使わないと当たり前にならないんです。

開発者ツールで確認してみましょう。先ほどのボックスデモの要素を検証してみてください。ボックスサイジングをボーダーボックスに変えると、ボックスモデル図のコンテント部分の数値が変わっているのが分かりますか。横幅は300ピクセルのまま変わらないのに、コンテントの幅がパディングとボーダーの分だけ小さくなっています。全体で300ピクセルに収まるように、内側が自動調整されているんですね。

実は、プロのウェブ制作では、すべての要素にこのボーダーボックスを最初に適用するのが一般的です。こんな書き方をします。

```css
* {
  box-sizing: border-box;
}
```

アスタリスクは「すべての要素」を意味するセレクタです。前回のセレクタの講義で、タグセレクタ、クラスセレクタ、IDセレクタを学びましたが、このアスタリスクは「ページ内のすべての要素」を選択する特別なセレクタで、ユニバーサルセレクタと呼びます。これをCSSの一番上に書いておけば、ページ内のすべての要素にボーダーボックスが適用されて、サイズの計算が直感的になります。

AIが生成したCSSを見ると、ほぼ必ずこの指定が入っています。「ああ、これはボックスモデルの計算を楽にするためのおまじないだな」と分かれば、それで十分です。皆さんもこれからCSSを書くときは、まずこの一行を書く癖をつけておくと良いですよ。

### セクション6: 実習 — カード型レイアウトを作る（5.5分）

では、今日学んだことを全部使って、カード型のレイアウトを作ってみましょう。ウェブサイトでよく見かけますよね、角が丸い枠の中に情報がまとまっている、あのカードデザインです。プロフィールサイトの自己紹介カードや、カフェサイトのメニューカードなど、本当にいろいろな場面で使われています。

まず、HTMLから書いていきましょう。先ほどのボックスデモのdivはそのまま残しておいて、その下に新しくカード用のHTMLを追加します。

```html
<div class="card">
  <h2>自己紹介</h2>
  <p>Web制作を学んでいます。AIを活用して、おしゃれなホームページを作れるようになるのが目標です。</p>
</div>
```

シンプルですね。カードというクラス名のdivの中に、h2の見出しとpの段落が入っている構造です。これがカードの中身になります。

では、CSSでカードらしい見た目にスタイリングしていきましょう。まず、先ほど学んだ、すべての要素にボーダーボックスを適用するおまじないから書きます。

```css
* {
  box-sizing: border-box;
}
```

次に、カードのスタイルを書きます。一気にいきますね。

```css
.card {
  width: 350px;
  padding: 24px;
  border: 1px solid #dddddd;
  border-radius: 12px;
  margin: 20px;
  background-color: #ffffff;
  font-family: "Noto Sans JP", sans-serif;
}
```

一行ずつ見ていきましょう。まず横幅350ピクセル。ボーダーボックスを適用しているので、パディングやボーダーを含めて350ピクセルの箱になります。

パディングは24ピクセルです。中身のテキストと枠の間に、ゆったりとした余白ができます。この余白があるかないかで、カードの印象がまったく変わりますよ。

ボーダーは1ピクセルの実線で、シャープddddddは薄いグレーです。さりげない枠線にすることで、上品な見た目になります。

ここで一つ新しいプロパティが出てきました。ボーダーレイディアスです。これは箱の角を丸くするプロパティです。12ピクセルと指定すると、角がやわらかいカーブになります。この値を大きくするほど角が丸くなって、50パーセントと指定すると完全な円になります。今回は12ピクセルで、ほどよい丸みですね。

マージンは20ピクセルで、他の要素との間にスペースを確保しています。背景色は白、フォントは前回設定したNoto Sans JPです。

プレビューを見てみてください。角丸の枠で囲まれた、きれいなカードができていると思います。パディングのおかげでテキストと枠の間にゆとりがあって、読みやすいですよね。

もう少し仕上げていきましょう。カードの中の見出しとテキストにもスタイルを当てます。

```css
.card h2 {
  color: #2d6a4f;
  font-size: 20px;
  margin-top: 0;
  margin-bottom: 12px;
}

.card p {
  color: #555555;
  font-size: 14px;
  line-height: 1.8;
  margin: 0;
}
```

カードの中のh2に、前回使ったグリーン系の色を指定しました。ここで注目してほしいのが、マージントップを0にしている点です。実は、h2にはブラウザのデフォルトスタイルで上にマージンがついています。カードの中では、パディングがあるのでこの上マージンは要らないんですね。0にすることで、パディングだけで上の余白をコントロールできます。マージンボトムを12ピクセルにして、見出しと本文の間に適度な間隔を作っています。

pタグは少し暗めのグレーにして、フォントサイズは14ピクセル、ラインハイト1.8で読みやすくしています。こちらもマージンを0にして、デフォルトのマージンを消しています。

プレビューを見てください。かなり本格的なカードになりましたよね。こういうカード型のデザインは、プロフィールサイトの自己紹介はもちろん、カフェのメニュー紹介や、美容室の施術コース一覧など、本当にいろいろな場面で使えます。

ここで注目してほしいのは、今日学んだボックスモデルの知識がフル活用されていることです。パディングで内側の余白を作り、ボーダーとボーダーレイディアスで枠線と角丸を設定し、マージンで外側の余白を確保している。h2やpタグのデフォルトマージンを消して、パディングで余白を統一的にコントロールしている。そしてボーダーボックスのおかげで、横幅350ピクセルと書いたら本当に350ピクセルの箱になっている。ボックスモデルが分かると、こうしたレイアウトが自由に作れるようになるんです。

もう一つカードを追加して、2つ並べてみましょう。HTMLにもう一つdivを追加してみてください。

```html
<div class="card">
  <h2>好きなこと</h2>
  <p>カフェ巡りと写真撮影が趣味です。いつか自分のカフェのホームページを作りたいと思っています。</p>
</div>
```

2つのカードが縦に並んで表示されましたね。同じクラス名をつけているので、同じスタイルが適用されて、統一感のある見た目になっています。CSSを一箇所変えれば、両方のカードのスタイルが同時に変わる。クラスセレクタの便利さが実感できますよね。試しにカードのパディングを32ピクセルに変えてみてください。両方のカードの内側の余白が同時に広がりましたね。このように、共通のクラスでスタイルを管理すると、あとからの調整がとても楽です。

今はただ縦に並んでいるだけですが、次回の講義で学ぶFlexboxを使えば、このカードを横に並べたり、画面の大きさに合わせて配置を変えたりできるようになります。今日の「箱」の知識と、次回の「並べ方」の知識が合わさると、本格的なレイアウトが作れるようになりますよ。

## まとめ（1.5分）

今日の講義では、CSSの核心概念であるボックスモデルを学びました。

まず、すべてのHTML要素は「箱」であるということ。その箱は内側からコンテント、パディング、ボーダー、マージンの4つの領域で構成されています。パディングは箱の内側の余白で背景色がつく。マージンは箱の外側の余白で常に透明。この違いが大事でしたね。

最後に、ボックスサイジングのボーダーボックス。これを使えば、横幅の中にパディングとボーダーが含まれて、サイズの計算が直感的になります。アスタリスクのユニバーサルセレクタで全要素に適用するのが定番でしたね。AIが生成したCSSにもほぼ必ず入っている、大事な設定です。
