## 本編

### セクション1: CSSで色を指定する3つの方法（7分）

まずは、色の指定方法から見ていきましょう。CSSで色を指定するには、大きく3つの書き方があります。順番に試していきますので、CodePenを開いてください。前回まで使っていた自己紹介ページのペンを開きましょう。

最初は「カラーネーム」です。これは一番シンプルで、色の名前をそのまま英語で書く方法です。CSSのペインに、こう書いてみましょう。

```css
h1 {
  color: red;
}
```

はい、見出しが赤くなりましたね。レッドと書くだけで赤になる。直感的でわかりやすいですよね。カラーネームは、レッド、ブルー、グリーンのような基本的な色だけでなく、実はコーラルとかミッドナイトブルーとか、かなり細かい色名も用意されています。全部で140色くらいあります。

ちなみに、今書いたカラーというプロパティは、文字の色を変えるプロパティです。前回の講義でバックグラウンドカラーを使いましたよね。バックグラウンドカラーは「背景色」、カラーは「文字色」。名前は似ていますが、変える対象が違うので注意してください。

さて、カラーネームは直感的でいいのですが、実際のウェブ制作ではこれだけだと限界があります。たとえば「もうちょっとだけ暗い赤」とか「少しくすんだブルー」とか、微妙なニュアンスを表現したいことの方が多いですよね。140色あるとはいえ、ウェブで表現できる色は1600万色以上あるんです。たった140色では、全然足りません。

そこで2つ目の方法、「HEX」です。ヘックスと読みます。これが実は一番よく使われる書き方です。先ほどのレッドを、HEXに書き換えてみましょう。

```css
h1 {
  color: #ff0000;
}
```

シャープ記号のあとに、6桁のアルファベットと数字の組み合わせが並んでいますね。これがHEXカラーコードです。プレビューを見ると、先ほどのレッドと同じ赤が表示されていますよね。

見た目は少し難しそうに見えますが、仕組みは実はシンプルです。6桁のうち、最初の2桁が赤、真ん中の2桁が緑、最後の2桁が青の量を表しています。つまり、光の三原色ですね。00が「その色なし」で、ffが「その色マックス」です。

だから、ff0000は「赤マックス、緑なし、青なし」なので、真っ赤になるわけですね。ちなみに、シャープ000000だと赤も緑も青も全部なしで黒。シャープffffffだと全部マックスで白になります。試しにシャープ0000ffに変えてみてください。青になりますよね。最後の2桁が青の量なので、ここだけマックスにすると青一色になるわけです。

では、もう少し実用的な色も試してみましょう。

```css
h1 {
  color: #336699;
}
```

こうすると、落ち着いた青色になりますね。赤33、緑66、青99なので、青が強めで、少し緑と赤も混ざっている。こうやっていろんな数値を組み合わせることで、1600万色以上の中から自由に色を選べるんです。

とはいえ、このコードを自分で計算する必要はまったくありません。暗記する必要もありません。大事なのは「シャープ記号のあとに6桁の英数字が書いてあったら、これは色のコードなんだな」と分かることです。実際の色選びには、このあと紹介するツールを使います。

では、3つ目の方法も見ておきましょう。「RGB」です。

```css
h1 {
  color: rgb(255, 0, 0);
}
```

RGB、つまりレッド、グリーン、ブルーの頭文字ですね。括弧の中に、赤、緑、青の順で0から255までの数値を指定します。考え方はHEXと同じで、表記が違うだけです。255, 0, 0は「赤マックス、緑なし、青なし」で、やっぱり真っ赤ですね。

RGBには、RGBAという書き方もあって、4つ目の数値で透明度を指定できます。AIが生成したCSSでたまに見かけますが、「4つ目は透明度なんだな」と分かれば十分です。

では、ここで3つの書き方を整理しておきましょう。カラーネームのレッド、HEXのシャープff0000、RGBの255, 0, 0。この3つは全部同じ赤色です。書き方が違うだけで、結果は同じなんですね。実務でよく使われるのはHEXですが、AIがコードを生成するときにはRGBを使うこともあります。どの書き方で出てきても「あ、これは色の指定だな」と分かれば、それで十分です。

### セクション2: カラーパレットツールの紹介（3.5分）

では、実際にウェブサイトで使う色はどうやって決めるのか。「このサイトにはどんな色が合うだろう」と考えるのは、プロのデザイナーでも悩む作業です。しかも、色は1色だけ選べばいいわけではなくて、メインカラー、サブカラー、アクセントカラーと、複数の色の組み合わせを考える必要があります。

そこで便利なのが、カラーパレットツールです。今日は2つ紹介します。

まずひとつ目は、Coolorsというツールです。クーラーズと読みます。このツールは、スペースキーを押すだけで、5色のおしゃれな配色パターンを自動で提案してくれるんです。押すたびに新しい組み合わせが出てきます。「お、この組み合わせいいな」と思ったら、気に入った色の鍵マークをクリックしてロックできます。ロックした色は固定されたまま、残りの色だけが変わるので、だんだん理想の配色に近づけていけるんです。気に入った色が決まったら、その色のHEXコードをコピーして、CSSに貼り付ければ使えます。

もうひとつは、Adobe Colorです。こちらは、「類似色」「補色」「トライアド」といった配色理論に基づいてパレットを作れるツールです。少し本格的ですが、色の組み合わせを理論的に学びたい方にはおすすめです。他のユーザーが作った配色パターンを探すこともできるので、参考になります。

どちらのツールも無料で使えます。もちろん今の時点では、色選びに悩んだらAIに「カフェのサイトに合う暖かい配色を提案して」と聞くのも手ですよね。ただ、AIに頼むにしても、Coolorsで見つけた色を「この色をベースに配色を考えて」と指定したりすると、より自分のイメージに近い結果が得られます。「こういう雰囲気がいい」という具体的な色を見せながらお願いする方が、AIも的確に応えてくれるんです。

では、Coolorsで色を一つ選んでみましょう。好きな色のHEXコードをコピーして、先ほどのh1のカラーに貼り付けてみてください。

```css
h1 {
  color: #2d6a4f;
}
```

こんな感じで、シャープのあとにコピーしたコードを貼り付けるだけです。ここでは深いグリーンにしてみましたが、皆さんは好きな色を選んでくださいね。プレビューで色が変わるのを確認してください。先ほどのレッドとは全然違う、落ち着いた印象になりましたよね。

### セクション3: Google Fontsでフォントを変える（7分）

色の次は、文字の見た目を変えていきましょう。フォント、つまり書体を変えるだけで、ページの雰囲気は大きく変わります。同じ文章でも、ゴシック体で書くのと明朝体で書くのとでは、受ける印象がだいぶ違いますよね。ウェブサイトでも同じで、フォント選びはデザインの印象を決める大事な要素です。

CSSでフォントを指定するには、フォントファミリーというプロパティを使います。まずは試しに書いてみましょう。

```css
body {
  font-family: serif;
}
```

セリフというのは、明朝体のような、文字の先端に小さな飾りがついたフォントのことです。新聞や書籍でよく見かけるタイプですね。プレビューを見ると、文字の雰囲気が変わったのが分かりますよね。フォーマルで伝統的な印象になります。

これをサンセリフに変えると、ゴシック体のような、飾りのないすっきりしたフォントになります。サンはフランス語で「なし」という意味で、つまり「飾りなし」のフォントですね。

```css
body {
  font-family: sans-serif;
}
```

ただ、セリフやサンセリフは「フォントの種類」を大まかに指定しているだけで、実際にどのフォントが表示されるかは、見ている人のパソコンによって変わってしまいます。これだと、デザインの統一がとれませんよね。

そこで登場するのが、Google Fontsです。Google Fontsは、Googleが無料で提供しているウェブフォントのサービスで、ここから好きなフォントを選んで、自分のウェブページに適用できます。フォントのデータをインターネット経由で読み込むので、見る人のパソコンにそのフォントが入っていなくても、同じフォントで表示されるんです。

では、実際にやってみましょう。Google Fontsのサイトにアクセスします。たくさんのフォントが一覧で表示されていますね。

今日は日本語のフォントを使いたいので、左側のフィルターでランゲージのところからジャパニーズを選びましょう。日本語に対応したフォントだけが表示されます。

日本語フォントでおすすめのものを3つ紹介しておきますね。

まず、Noto Sans JPです。ノートサンスジェーピーと読みます。Googleが開発したフォントで、すっきりしていて読みやすく、一番使いやすいフォントです。迷ったらこれを選んでおけば間違いありません。日本語のウェブサイトでは最もよく使われているフォントの一つです。

次に、M PLUS Rounded 1cです。こちらは角が丸くて、やわらかい印象のフォントです。カフェや教室、サロンのサイトなんかに合いそうですよね。親しみやすい雰囲気を出したいときにぴったりです。

最後に、Zen Kaku Gothic Newです。少しモダンで洗練された雰囲気のフォントで、ビジネス寄りのサイトや、すっきりした印象にしたいサイトに合います。

今日はNoto Sans JPを使ってみましょう。フォント名をクリックすると、詳細ページが開きます。右上にある「ゲットフォント」ボタンを押して、次に「ゲットエンベッドコード」を押します。すると、2つのコードが表示されますね。リンクタグのコードと、CSSのフォントファミリーのコードです。

実際のウェブサイトを作るときは、このリンクタグをHTMLのヘッドタグの中に追加して使います。ただ、CodePenではもっと簡単な方法があります。

CodePenの画面で、上のメニューから「Settings」をクリックしてください。設定画面が開いたら、CSSのタブを選びます。下の方に「External Stylesheets」という入力欄がありますね。ここに、Google FontsのURLを貼り付けます。

Google Fontsの画面に戻って、リンクタグの中にあるURLの部分だけをコピーしましょう。エイチティーティーピーエスから始まるURLです。リンクタグ全体ではなく、hrefのイコールの後ろの引用符の中にあるURL部分だけをコピーしてくださいね。これをCodePenのエクスターナルスタイルシーツの欄に貼り付けて、「Save & Close」を押します。

これで、Noto Sans JPがこのペンで使えるようになりました。ここまでの手順を簡単に振り返ると、Google Fontsでフォントを選んで、URLをコピーして、CodePenの設定に貼り付ける、という3ステップですね。では、CSSで適用してみましょう。

```css
body {
  font-family: "Noto Sans JP", sans-serif;
}
```

フォントファミリーに、ノートサンスジェーピーと書いて、カンマのあとにサンセリフを書いています。このカンマの後ろのサンセリフは「もしNoto Sans JPが読み込めなかったら、標準のゴシック体を使ってね」という意味のフォールバック、つまり予備指定です。

プレビューを見てみましょう。文字の形が変わっていますね。先ほどの標準フォントよりも、すっきりしてモダンな印象になったと思います。特に日本語の漢字やひらがなが、きれいに整って表示されていますよね。フォントを一つ変えるだけで、ページ全体の印象がこんなに変わるんです。

ちなみに、フォント名をダブルクォーテーションで囲んでいるのは、フォント名にスペースが入っているからです。Noto Sans JPはスペースで区切られた3つの単語からなるフォント名なので、引用符で囲んで「これは一つのフォント名ですよ」とCSSに伝えています。セリフやサンセリフのように一つの単語なら、引用符は不要です。

ちなみに、実際のウェブサイトでは、先ほどのリンクタグをHTMLのヘッドタグに追加する方法を使います。CodePenでは設定画面から簡単に追加できましたが、この仕組みについては、公開用のファイルにするときに改めてやりますので、今はCodePenのやり方で大丈夫です。

### セクション4: 文字のスタイリングプロパティ（5.5分）

フォントが変わったところで、次は文字のスタイリングをもう少し細かく調整していきましょう。フォントの種類だけでなく、文字の大きさや太さ、行間や字間といった要素も、読みやすさやデザインに大きく影響します。ここでは4つのプロパティを見ていきます。

まずは、フォントサイズです。文字の大きさを変えるプロパティですね。

```css
h1 {
  color: #2d6a4f;
  font-size: 32px;
}
```

先ほど色を変えたh1に、フォントサイズを追加してみました。32ピクセルと指定しています。ピクセルは画面上の点の単位ですね。数字を変えてみて、文字がどう変わるか確認してみてください。24ピクセルにすると小さくなりますし、48ピクセルにするとかなり大きくなります。

ウェブサイトでよく使われるサイズの目安としては、本文が16ピクセル前後、小見出しが20から24ピクセル、大見出しが28から36ピクセルくらいです。あくまで目安なので、自分のページに合わせて調整してみてくださいね。

次は、フォントウエイトです。文字の太さを変えるプロパティです。

```css
h1 {
  color: #2d6a4f;
  font-size: 32px;
  font-weight: bold;
}
```

ボールドと書くと太字になります。h1はもともと太字なのであまり変化が分からないかもしれませんが、pタグなど通常の文字に対して使うと効果がはっきり見えます。試しにpタグにフォントウエイトのボールドを追加してみると、本文がすべて太字になりますね。

フォントウエイトは、ボールドやノーマルといったキーワードの代わりに、数値で指定することもできます。100から900まであって、400が通常の太さ、700がボールドに相当します。Noto Sans JPのようなGoogle Fontsは、300の細い文字から700の太い文字まで複数のウエイトを用意していることが多いので、数値を変えるとちゃんと見た目が変わります。たとえば300にすると、かなり細い文字になりますよ。

3つ目は、ラインハイトです。行の高さ、つまり行間を調整するプロパティです。これは地味ですが、読みやすさに直結する非常に大事なプロパティです。

```css
p {
  line-height: 1.8;
}
```

pタグにラインハイト1.8と指定しました。この数値は、フォントサイズに対する倍率です。1.8なら、フォントサイズの1.8倍の行の高さになります。数値が大きいほど行間が広がって、文章が読みやすくなります。

実際に数値を変えて比べてみましょう。試しに1.0にしてみてください。行と行がぎゅっと詰まって、読みにくくなりますよね。逆に2.5にすると、行間がスカスカに開きすぎて、こちらも読みにくい。日本語の場合は1.6から1.8くらいが読みやすいバランスだと言われています。プロのウェブサイトでも、だいたいこのあたりの数値が使われていますね。

最後に、レタースペーシングです。文字と文字の間隔を調整するプロパティです。

```css
h1 {
  color: #2d6a4f;
  font-size: 32px;
  font-weight: bold;
  letter-spacing: 0.05em;
}
```

0.05emと指定しました。emというのはフォントサイズを基準にした単位で、0.05emなら文字サイズの5%ぶんだけ間隔を広げるという意味です。ほんの少しの違いですが、見出しに適度なレタースペーシングを入れると、ゆとりが出てきれいに見えます。

試しにレタースペーシングを0.2emにしてみてください。かなり文字が離れますよね。逆にマイナス0.02emにすると、少しだけ詰まります。見出しには少し広めに、本文はほとんどいじらないか、ほんの少しだけ広げる、というのがよくあるパターンです。

この4つのプロパティ、フォントサイズ、フォントウエイト、ラインハイト、レタースペーシング。CSSの文字スタイリングでよく使うものです。すべて暗記する必要はありません。「文字の大きさ、太さ、行間、字間を変えるプロパティがある」と知っていれば、あとは必要なときに調べたり、AIに聞いたりすれば大丈夫です。

### セクション5: 実習 — 自己紹介ページをスタイリングする（6分）

では、今日学んだことを使って、自己紹介ページを本格的にスタイリングしてみましょう。いままで個別に試していたCSSを、一つにまとめて書いていきます。CSSペインに書いてある練習用のコードは一度消して、ゼロから書き直していきましょう。Google Fontsの設定はそのまま残っていますので、フォントはすでに使える状態です。

まず、ページ全体のベースを整えます。

```css
body {
  font-family: "Noto Sans JP", sans-serif;
  color: #333333;
  line-height: 1.8;
}
```

ボディに対して、3つのプロパティを指定しています。まずフォントをNoto Sans JPに変更。次に文字色をシャープ333333にしています。これは真っ黒ではなく、少しだけグレーがかった黒です。

実は、ウェブサイトの本文には真っ黒よりもこうした濃いグレーを使うことが多いんです。真っ黒、つまりシャープ000000だと、白い背景とのコントラストが強すぎて、長い文章を読むとき目が疲れやすいんですね。プロのウェブサイトをよく見ると、本文の文字色は真っ黒ではなく、シャープ333333やシャープ444444といった濃いグレーを使っていることがほとんどです。こういう細かいところが、素人っぽさとプロっぽさの差だったりします。

そしてラインハイト1.8で行間にゆとりを持たせています。先ほど練習で試した通りですね。

次に、見出しをスタイリングします。

```css
h1 {
  color: #2d6a4f;
  font-size: 32px;
  letter-spacing: 0.05em;
}

h2 {
  color: #40916c;
  font-size: 24px;
  letter-spacing: 0.03em;
}
```

h1を深いグリーンに、h2を少し明るいグリーンにしてみました。ここで大事なのは、色に統一感を持たせるということです。h1が緑で、h2が赤で、リンクが青で、とバラバラにすると、まとまりのない見た目になってしまいます。同じ系統の色でトーンを変えることで、ぐっとまとまりのある印象になるんですね。

h2はh1より小さいフォントサイズにして、レタースペーシングも少し控えめにしています。見出しのレベルに合わせて、サイズや装飾に段階的な差をつけるのがポイントです。

リンクの色も変えてみましょう。

```css
a {
  color: #2d6a4f;
}
```

リンクの色も見出しと同じグリーン系にそろえることで、ページ全体の色の統一感が出ます。デフォルトのリンク色は青でしたが、サイトのテーマカラーに合わせて変えるのが普通です。

では、プレビューを見てみてください。フォントが変わって、色に統一感が出て、行間にゆとりができて、かなり見やすくなったと思いませんか。先ほどまでのデフォルトの状態と比べると、だいぶ印象が変わりましたよね。

たった数行のCSSを追加しただけで、この変化です。色とフォントの力は本当に大きいんですね。もし「自分のページの色はグリーンじゃなくて、青系にしたいな」と思ったら、ぜひCoolorsでお気に入りの青系の配色を探して、HEXコードを入れ替えてみてください。
