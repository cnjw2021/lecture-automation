## 導入（2分）

皆さん、こんにちは。前回の講義では、HTMLのテキスト系タグを一気に学びましたね。h1からh6までの見出し、pタグで段落、strongやemで強調、そしてリストのulとli。CodePenの自己紹介ページに、見出しや紹介文、趣味のリストを追加するところまで進みました。

今日はその自己紹介ページを、もっとウェブサイトらしくしていきます。テーマは「リンクと画像」です。

リンクというのは、テキストや画像をクリックすると別のページに飛べる、あの仕組みのことです。ニュースサイトで記事の見出しをクリックして本文に飛んだり、お店のサイトで「アクセス」をクリックして地図のページに移動したり。皆さんも普段から何気なく使っていますよね。

画像は、写真やイラストをページに表示する方法です。テキストだけのページと、写真が1枚入ったページを比べると、印象がまるで違いますよね。

今日この2つを学ぶだけで、テキストばかりだった自己紹介ページが、ぐっとウェブサイトらしい見た目に変わります。今日の講義が終わる頃には、皆さんのページにプロフィール写真と、SNSへのリンクが入っている状態を目指しましょう。では始めていきます。

## 本編

### セクション1: リンクを作る、aタグ（8分）

まず、リンクから始めましょう。

普段ウェブサイトを見ているとき、青い文字や青い下線の付いたテキストをクリックして別のページに移動しますよね。ニュースサイトの記事一覧からタイトルをクリックして本文に飛んだり、ショッピングサイトで商品名をクリックして詳細ページに移動したり。あれがすべて「リンク」です。

リンクは、ウェブの一番大切な仕組みと言ってもいいくらいです。世界中のページが、リンクという糸でお互いにつながっている。だからワールドワイドウェブ、つまり「世界中に広がる蜘蛛の巣」と呼ばれるんですね。もしリンクがなかったら、見たいページがあるたびに毎回アドレスバーにURLを手入力しなきゃいけません。それは大変ですよね。

では、このリンクをHTMLでどう書くのか見ていきましょう。CodePenを開いてください。前回作った自己紹介ページがそのまま残っていますか？CodePenにログインしていれば、前回保存したペンが残っているはずです。残っていれば、その続きから始めます。もし消えてしまった方がいたら、h1タグで名前、pタグで簡単な自己紹介文だけ書いておいてくださいね。準備ができたら進めていきましょう。

リンクに使うのは、aタグです。aというのはアンカーの頭文字で、日本語にすると「錨」という意味です。船の錨が海底にガッチリ固定するように、aタグは別のページへの接続をしっかり結びつけてくれる。そんなイメージを持っていただければと思います。

では実際に書いてみましょう。自己紹介ページの一番下あたりに、こう書いてみてください。

```html
<a href="https://www.google.com">Googleを開く</a>
```

はい、書けましたか？ゆっくりで大丈夫ですよ。特にhrefの綴りは間違えやすいポイントです。h、r、e、fの4文字。そしてイコールの後ろの引用符も忘れずに。

プレビューを見てみましょう。「Googleを開く」という青い文字が表示されていますよね。何も特別なCSSを書いていないのに、aタグで囲んだテキストは自動的に青色になって、下線が引かれます。これがブラウザの初期設定なんです。「あ、これはリンクだな」と訪問者がすぐわかるようになっているわけですね。

では、このコードを一つずつ確認していきましょう。

まず、aの後ろに半角スペースを空けて「href」と書いてあります。エイチレフと読みます。正式にはハイパーリファレンスの略で、リンク先のアドレスを指定する場所です。イコールの後ろのダブルクォーテーション、つまり引用符の中に、移動先のURLを入れます。

このhrefのように、タグに追加の情報を持たせる仕組みのことを、HTMLでは「属性」と呼びます。前回までのh1やpタグでは属性を使いませんでしたが、aタグではこのhref属性が欠かせません。リンク先を指定しないリンクなんて意味がないですからね。これからいろいろなタグを学んでいく中で、属性はたくさん出てきますが、基本は全部同じです。タグ名の後ろに半角スペースを空けて、属性名イコール引用符の中に値を書く。この形を覚えておいてください。

そして、aの開始タグと終了タグの間に書いた「Googleを開く」。これが画面に表示されるリンクの文字になります。前回学んだpタグやh1タグと同じ構造ですよね。開始タグと終了タグで中身を囲む。HTMLの基本パターンです。

一つ大事なポイントがあります。リンクの文字は「Googleを開く」と書きましたが、これは好きなように変えられます。ただし、リンクの文字を読んだだけで「これをクリックしたらどこに行くのか」がわかるように書くのが親切です。たまに「ここをクリック」とだけ書いてあるリンクを見かけますが、あれだとクリックするまで何のページに飛ぶのかわかりませんよね。

では、もう一つリンクを追加してみましょう。今度はYouTubeへのリンクです。先ほどのaタグの下に、もう1行書いてみてください。

```html
<a href="https://www.youtube.com">YouTubeを見る</a>
```

プレビューに2つ目のリンクが出ましたね。2つのリンクが縦に並んでいる方もいれば、横にくっついている方もいるかもしれません。これはHTML側の並び方の仕組みに関係しているんですが、今は気にしなくて大丈夫です。

さて、ここで一つ困ったことに気づきませんか。CodePenのプレビューでリンクをクリックすると、CodePenの画面から離れてしまいますよね。実際のウェブサイトでも同じ問題が起きます。皆さんが将来自分のサイトを作ったとき、訪問者がリンクをクリックしてそのまま外部のサイトに移動してしまったら、もう戻ってきてもらえないかもしれません。せっかく見に来てくれたのに、もったいないですよね。

そこで使うのが、「新しいタブで開く」という機能です。先ほどのYouTubeのリンクを、こう書き換えてみましょう。

```html
<a href="https://www.youtube.com" target="_blank">YouTubeを見る</a>
```

hrefの後ろに半角スペースを空けて、target="_blank"を追加しました。targetは「リンクをどこで開くか」を指定する属性です。ブランクは英語で「空白」という意味ですから、「空白の新しいタブで開く」という指定になります。

これを付けると、リンクをクリックしたときに新しいタブが開いて、元のページはそのまま残ります。訪問者は外部サイトを見終わったら、タブを切り替えるだけで元のページに戻れるわけです。

外部のサイトへリンクするときは、基本的にこのtarget="_blank"を付けておくと親切ですね。一方、自分のサイト内で別のページに移動するだけのときは、付けなくても大丈夫です。

では、Googleのリンクにもtarget="_blank"を追加しておきましょう。

```html
<a href="https://www.google.com" target="_blank">Googleを開く</a>
```

これでリンクの基本はバッチリです。もう一度整理しますね。aタグでリンクを作る。href属性でリンク先のURLを指定する。外部サイトへのリンクにはtarget="_blank"を付けて、新しいタブで開くようにする。この3点セットです。暗記する必要はありませんよ。「aタグにはhrefが必要で、外部リンクにはtarget="_blank"を付ける」、これだけ覚えていれば十分です。

### セクション2: 画像を表示する、imgタグ（6分）

次は画像です。テキストとリンクだけのページも情報としては成り立ちますが、やっぱり写真が1枚あるだけで印象がまるで違います。美容室のサイトなら施術のビフォーアフター写真、カフェなら店内の雰囲気がわかる写真、教室なら生徒さんたちの楽しそうな様子。文字だけでは伝わらない魅力を、写真は一瞬で伝えてくれますよね。

画像を表示するタグは、imgタグです。imgはイメージの略ですね。HTMLのタグは、こういうふうに英語の単語を短くしたものが名前になっていることが多いです。aはアンカー、imgはイメージ、pはパラグラフ。名前の由来を知っておくと、初めて見るタグでも「あ、これはこういう意味かな」と推測しやすくなります。

では書いてみましょう。先ほどのリンクの下に、こう書いてみてください。

```html
<img src="https://picsum.photos/400/300" alt="サンプル画像">
```

プレビューを見てみましょう。写真が1枚表示されましたか？何かしらの写真が表示されているはずです。ページをリロードするたびに違う写真に変わるかもしれませんが、心配しないでください。今使っているURLは、テスト用のダミー画像をランダムに返してくれるサービスです。後ほど本物の写真に差し替えます。

さて、このimgタグをよく見てください。今まで学んできたタグと、ちょっと違うところがありませんか？

そうです。閉じタグがないんです。今まで学んだタグには必ず閉じタグがありました。h1には /h1、pには /p、aには /a。開始タグと終了タグのペアが基本でしたよね。でも、imgタグにはペアの相手がいません。

これはセルフクロージングタグ、日本語で言うと「自己完結タグ」と呼ばれるものです。pタグは開始タグと終了タグの間にテキストを挟みますが、imgタグは「ここに画像を表示して」という指示だけなので、間に何かを挟む必要がないんです。だからタグ1つで完結する。前回学んだbrタグも同じ仲間でしたよね。改行のbrも閉じタグがなかった。こういうタグは数が限られていますので、新しく出てきたときに「あ、これは閉じタグなしのやつだ」と覚えていけば大丈夫です。

では、imgタグの属性を詳しく見ていきましょう。2つの属性がありますね。

まず、srcです。ソースの略で、画像ファイルの場所、つまり画像の住所を指定します。先ほどのaタグのhrefと役割がよく似ていますね。hrefがリンク先の住所、srcが画像の住所。引用符の中にURLを入れるところも同じです。

次に、altです。オルトと読みます。オルタナティブ、つまり「代替」という意味です。何の代替かというと、画像が表示できなかったときに、代わりに画面に出すテキストなんです。

例えば、こんな場面を想像してみてください。電車の中でスマートフォンがトンネルに入って、一時的にインターネットの接続が切れた。そのとき、画像が読み込めなかったページでは、画像の代わりにこのaltのテキストが表示されるんです。訪問者は「ああ、ここにはこういう写真があるんだな」と理解できますよね。

もう一つ、とても大事な役割があります。目の不自由な方が使うスクリーンリーダーという読み上げソフトは、画像そのものを見せることができません。代わりに、このaltのテキストを音声で読み上げてくれるんです。つまりaltを書くことは、すべての人がウェブを利用できるようにするための配慮なんですね。

ですから、altには「この画像に何が写っているか」を簡潔に書くのがルールです。先ほど「サンプル画像」と書きましたが、実際にはもっと具体的に書きましょう。

```html
<img src="https://picsum.photos/400/300" alt="海辺のカフェのテラス席">
```

こんなふうに、見た人が画像の内容をイメージできるテキストを入れてあげてください。altを空っぽにしたり、そもそも書かなかったりするのは、できるだけ避けたいですね。

ちなみに、srcに指定しているURLの最後に「400/300」という数字がありますよね。これはこのテスト用サービスの機能で、横400ピクセル、縦300ピクセルの画像を返してね、という指定です。試しに数字を変えてみましょう。600と200にしてみます。

```html
<img src="https://picsum.photos/600/200" alt="横長のサンプル画像">
```

プレビューの画像が横長に変わりましたよね。こんなふうに画像のサイズは自由に変えられます。ただ、実際のウェブサイトでは画像のサイズ調整はCSSを使うのが一般的です。それはパート3で学びますので、今は「imgタグで画像を表示できる」ということが理解できていれば十分です。

### セクション3: 画像パスの理解（3.5分）

ここで少しだけ、画像の「パス」についてお話しします。パスというのは、コンピューターの世界で「ファイルの場所」を表す言葉です。先ほどimgタグのsrcにURLを書きましたよね。あのURLがまさに画像のパス、つまり画像の住所です。

この住所の書き方には、大きく2種類あります。わかりやすいように、現実の住所で例えてみましょう。

一つ目は、「東京都渋谷区〇〇町1丁目2番3号」のように、正式な住所をすべて書く方法です。日本のどこにいる人がこの住所を見ても、間違いなくたどり着けますよね。これを「絶対パス」と呼びます。HTMLの世界では、「https://」から始まる完全なURLのことです。

二つ目は、「この建物の3階、右側の302号室」のように、今いる場所を基準にして案内する方法です。同じ建物にいる人なら「3階の302号室」で問題なく通じますが、別の建物にいる人にこれを言っても、どの建物の3階なのかわかりませんよね。これを「相対パス」と呼びます。HTMLでは「images/photo.jpg」のように、自分のHTMLファイルがある場所を起点にして、そこからの位置関係で書く方法です。

今、CodePenで使っているのは絶対パスです。「https://picsum.photos/400/300」という、「https://」から始まる完全なURLを指定していますよね。CodePenはオンラインのエディタなので、自分のパソコンから画像ファイルをアップロードする場所がありません。ですから、インターネット上にすでにある画像のURLをそのまま指定する、つまり絶対パスを使うわけです。

一方、本格的にウェブサイトを作るときは、自分のプロジェクトの中に画像ファイルを置いて、相対パスで指定するのが一般的です。例えば「images/profile.jpg」のような書き方ですね。これはパート5で、CodePenからファイル構成のあるツールに移ったタイミングで、実際に手を動かして体験します。自分のプロジェクトの中にimagesフォルダを作って、そこに画像を入れて、相対パスで表示する練習をします。

ですので、今の段階では「画像のパスには絶対パスと相対パスの2種類がある」ということ、そして「CodePenでは絶対パスを使う」ということだけ押さえておいてください。難しく感じた方もいるかもしれませんが、大丈夫です。実際に使うときにはすぐに慣れますから。

### セクション4: 無料画像ソースの紹介（3分）

さて、いよいよ自己紹介ページに本物の写真を入れていきましょう。先ほどのテスト用画像ではなく、ちゃんとした写真を使いたいですよね。

ウェブサイトで使える無料の写真素材サイトを2つ紹介します。1つ目はUnsplash、2つ目はPexelsです。どちらもプロのカメラマンが撮影した高品質な写真を、無料で使うことができます。しかも商用利用もOKですので、将来お店のサイトやビジネス用のページを作るときにも安心して使えますよ。インターネットで「フリー素材」と検索するといろいろなサイトが出てきますが、この2つはクオリティが高くて種類も豊富なので、まずはここから探すのがおすすめです。

では、Unsplashから画像のURLを取得してみましょう。新しいタブを開いて、アドレスバーに「unsplash.com」と入力してください。

サイトが表示されましたね。英語のサイトですが、使い方はシンプルです。画面の上の方に検索バーがありますので、ここに好きなキーワードを英語で入れてみましょう。例えば「coffee」や「nature」、「workspace」など。自己紹介ページのプロフィール画像に合いそうなキーワードを入れてみてください。

検索すると、きれいな写真がたくさん表示されますよね。どれもプロが撮った写真なので、クオリティが高いです。この中から気に入った写真を一つ選んで、クリックして大きく表示しましょう。

写真が大きく表示されたら、その写真の上で右クリックしてください。メニューが表示されますので、「画像アドレスをコピー」という項目を選びます。Chromeを英語で使っている方は「Copy image address」と表示されます。これで画像のURLがクリップボードにコピーされました。

CodePenに戻りましょう。先ほど書いたimgタグのsrcの部分を、今コピーしたURLに書き換えます。srcのイコールの後ろの引用符の中身を一度消して、そこに貼り付けてください。

```html
<img src="ここにコピーしたURLを貼り付け" alt="プロフィール画像">
```

altにも、選んだ画像の内容に合ったテキストを入れてくださいね。コーヒーの写真を選んだなら「コーヒーカップの写真」、自然の風景なら「山と湖の風景」といった具合です。

プレビューにUnsplashの写真が表示されましたか？先ほどのテスト用画像と比べると、一気に本格的な見た目になりましたよね。もちろん、ご自身の写真を使いたい場合もあると思いますが、今の段階ではUnsplashやPexelsの写真で十分です。写真の選び方に迷ったら、自分の趣味や好きなものに関連する写真を選んでみてください。自己紹介ページですから、自分らしさが伝わる写真がいいですよね。

Pexelsのほうも使い方はほとんど同じです。「pexels.com」にアクセスして、キーワードで検索して、右クリックで画像アドレスをコピー。UnsplashとPexelsの両方をブックマークしておくと、将来ウェブサイトを作るときに素材探しがとても楽になりますよ。

### セクション5: 実習、SNSリンクを追加する（3.5分）

最後の仕上げに、SNSリンクを追加しましょう。

プロフィールサイトやWeb名刺には、SNSアカウントへのリンクを載せるのが定番です。自己紹介ページを見てくれた人が、そこからSNSをフォローしてくれるかもしれませんよね。実際のフリーランスの方のサイトや、お店のホームページにも、よくSNSリンクが並んでいます。

ここで、今日学んだaタグと前回学んだリストのタグを組み合わせてみましょう。画像の下に、こんなふうに書いてみてください。

```html
<h2>SNS</h2>
<ul>
  <li><a href="https://twitter.com" target="_blank">X 旧Twitter</a></li>
  <li><a href="https://www.instagram.com" target="_blank">Instagram</a></li>
  <li><a href="https://www.youtube.com" target="_blank">YouTube</a></li>
</ul>
```

少し長いですが、よく見ると今まで学んだタグの組み合わせです。前回学んだulとliでリストを作って、その中に今日学んだaタグを入れている。HTMLのタグは、こうやって入れ子にして組み合わせることができるんですね。liタグの中にaタグが入っている、という構造です。

URLの部分は、もし皆さんがSNSのアカウントをお持ちであれば、ご自身のプロフィールページのURLに書き換えてみてください。例えばXなら、ブラウザで自分のプロフィールページを開いて、アドレスバーに表示されているURLをコピーして、hrefの引用符の中に貼り付ける。Instagramも同じ要領です。アカウントがない方は、今書いたようにサービスのトップページのURLのままでまったく問題ありません。

ここで一つ、コードの構造を確認しておきましょう。liタグの中にaタグが入っていますよね。このように、HTMLではタグの中に別のタグを入れることができます。パート1でも少し触れましたが、これを「入れ子」とか「ネスト」と呼びます。ulの中にliがあって、そのliの中にaがある。タグが何重にも重なっていく感じですが、開始タグと終了タグの対応さえ崩れなければ、問題なく動きます。

プレビューを見てみましょう。「SNS」という見出しの下に、箇条書きの各項目が青いリンクになっていますよね。target="_blank"を付けてあるので、クリックすると新しいタブで開きます。元の自己紹介ページはそのまま残りますから、訪問者がSNSを見た後もすぐ戻ってこられる。バッチリですね。

さあ、ここで画面を上から順にゆっくり見てみてください。名前の見出しがあって、自己紹介の文章があって、趣味のリスト。そしてプロフィール写真。さらにSNSへのリンク一覧。テキストだけだった前回の状態と比べると、ずいぶんウェブサイトらしくなりましたよね。

## まとめ（1.5分）

今日の内容を振り返りましょう。

1つ目、リンク。aタグとhref属性で、クリックすると別のページに移動するリンクを作れます。hrefにリンク先のURLを書いて、タグの間のテキストが画面に表示される文字。外部サイトへのリンクにはtarget="_blank"を付けて、新しいタブで開くようにしてあげると親切です。

2つ目、画像。imgタグとsrc属性で、画像を表示できます。alt属性には画像の内容を表すテキストを必ず入れる。画像が表示できないときの代替テキストであり、スクリーンリーダーへの配慮でもあります。そしてimgタグは閉じタグのないセルフクロージングタグでしたね。brタグと同じ仲間です。

3つ目、画像のパス。住所を全部書く絶対パスと、今いる場所からの位置を書く相対パスの2種類がある。CodePenでは絶対パスを使う。相対パスはパート5で実際に体験します。

そして、UnsplashやPexelsといった無料の写真素材サイトも紹介しました。きれいな写真が無料で、しかも商用利用もOK。ウェブサイトを作るときの強い味方になりますよ。ぜひブックマークしておいてくださいね。

次の講義では、HTMLの「構造を組み立てるタグ」を学びます。今作っている自己紹介ページのHTMLを見ると、h1、p、ul、img、aといったタグが上から順にずらっと並んでいる状態ですよね。次回はこのコードを、もっと意味のあるまとまりに整理するための「セマンティックタグ」を学びます。ヘッダーはここ、メインコンテンツはここ、フッターはここ、というふうに区分けする。見た目は変わらないんですが、エーアイや検索エンジンにとって読み取りやすい構造にする。そんなお話です。
