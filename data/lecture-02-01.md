## 本編

### セクション1: タグの基本構造（5分）

【スライド: タグの構造 — 開くタグと閉じタグ】

前回の講義で、h1タグとpタグを使いましたよね。あのとき、なんとなく山括弧で囲まれた記号を書いたと思いますが、あの記号にはちゃんとしたルールがあるんです。今日はそのルールをきちんと理解していきましょう。

HTMLの「タグ」とは何か。ひとことで言うと、ブラウザへの指示書です。「ここからここまでは見出しとして扱ってね」「ここからここまでは段落として表示してね」と、ブラウザに伝えるための記号なんです。

そして、HTMLのタグには、基本的に「開くタグ」と「閉じタグ」の2つがセットで存在します。

開くタグは、山括弧で囲まれた名前です。たとえば「h1」なら、左の山括弧、h1、右の山括弧で書きます。これが「ここから見出しが始まりますよ」という合図です。

そして閉じタグは、山括弧の中にスラッシュが入ります。左の山括弧、スラッシュ、h1、右の山括弧。こちらは「ここで見出しが終わりますよ」という合図ですね。

この開くタグと閉じタグの間に、画面に表示したい内容を書くわけです。

これ、本の「かぎ括弧」によく似ていると思いませんか？日本語で「こんにちは」と書くとき、最初に開くかぎ括弧を書いて、最後に閉じるかぎ括弧を書きますよね。HTMLのタグもまったく同じです。「ここからここまでが見出しですよ」「ここからここまでが段落ですよ」と、範囲を示しているんです。

ちなみに、タグの名前には意味があります。h1のhは「ヘディング」、つまり見出しの頭文字です。pは「パラグラフ」、段落の頭文字です。英語の略称ですが、全部覚える必要はありません。よく使うものは自然に覚えますし、忘れてもエーアイに聞けばすぐ教えてくれます。

【スライド: タグの3つのパーツ — 「要素」】

もう少し整理しましょう。HTMLのタグは、3つのパーツで構成されています。

1つ目が開くタグ。タグの名前を山括弧で囲んだものです。

2つ目がコンテンツ。これは開くタグと閉じタグの間に書かれる内容です。テキストだったり、あるいは別のタグだったりします。

3つ目が閉じタグ。タグの名前の前にスラッシュを付けて山括弧で囲んだものです。

この3つをまとめて「要素」と呼びます。エレメントという英語が使われることもありますが、要するに「タグとその中身のワンセット」が1つの要素です。

「要素」という言葉はこの先たくさん出てきますので、「ああ、タグと中身のセットのことだな」と覚えておいてください。

### セクション2: 属性 — タグへの追加情報（3分）

【スライド: 属性 — タグに情報を付け加える】

さて、タグの基本形を学んだところで、もう一つ大事な仕組みをお話しします。それが「属性」です。アトリビュートという英語が使われることもあります。

属性とは何か。ひとことで言うと、タグに追加の情報を付け加えるための仕組みです。

たとえるなら、荷物に貼る「ラベル」のようなものです。段ボール箱にモノを入れるだけでなく、「この箱はワレモノです」「この箱は2階に運んでください」と、ラベルを貼って情報を追加しますよね。タグにも同じように、追加情報を書き込むことができるんです。

書き方はこうです。開くタグの中に、タグ名の後ろにスペースを空けて、属性名イコール引用符で囲んだ値を書きます。

たとえば、次のような形です。

```html
<p lang="ja">こんにちは</p>
```

このlangという属性は、「この段落の言語は日本語ですよ」とブラウザに伝えるためのものです。pタグに「ラベル」を1枚貼って、情報を付け加えたわけですね。

属性にはいくつかの特徴があります。

まず、属性は必ず「開くタグ」の中に書きます。閉じタグには書きません。

次に、1つのタグに複数の属性を付けることもできます。荷物のラベルも1枚だけとは限りませんよね。「ワレモノ」と「2階行き」、2枚貼ることだってあります。

そして、属性の種類はタグによって異なります。どのタグにどんな属性が使えるか、今は全部覚える必要はありません。この先の講義で、リンクを作るときや画像を表示するときに、実際に属性を使っていきます。そのときに「ああ、これがあのラベルの仕組みだな」と思い出してもらえれば大丈夫です。

今日のポイントは、「タグには追加情報を書き込める場所がある」ということ。これだけ覚えておいてください。

### セクション3: タグの親子関係（5分）

【スライド: マトリョーシカ人形】

さて、HTMLのもう一つの大事なルールについてお話しします。それは、タグの中にタグを入れることができるということです。

これを説明するのに、マトリョーシカ人形がぴったりです。皆さん、マトリョーシカをご存じですか？ロシアの伝統的な人形で、開けると中に一回り小さい人形が入っている、あの入れ子の人形ですね。大きな人形の中に中くらいの人形が入っていて、中くらいの人形の中にさらに小さな人形が入っている。

HTMLもまったく同じ構造です。外側に大きなタグがあって、その中に別のタグが入り、さらにその中にまた別のタグが入る。この関係を「ネスト」、日本語で言うと「入れ子構造」と呼びます。

そして、外側のタグを「親」、内側のタグを「子」と呼びます。これはプログラミングの世界で広く使われる表現ですが、とてもわかりやすいですよね。家族の関係と同じです。

【スライド: HTML親子関係の図】

具体的な例を見てみましょう。画面に表示されている図を見てください。

いちばん外側にbodyタグがあります。これがいちばん大きなマトリョーシカです。このbodyタグの中に、h1タグとpタグが入っています。

この場合、bodyが「親」で、h1とpが「子」です。そしてh1とpの関係は、同じ親を持つ「兄弟」、つまり「きょうだい」ですね。家族で言えば、bodyがお父さんやお母さんで、h1とpがその子どもたち。子どもたち同士はきょうだいの関係です。

ここで大事なルールがあります。子のタグは、必ず親のタグの中で開いて、親のタグの中で閉じなければいけません。マトリョーシカで言えば、中の人形は外の人形の中に完全に収まっていなければいけない。半分はみ出していたら、蓋が閉まりませんよね。

これがHTMLの入れ子のルールです。親の開くタグの後に子の開くタグが来て、子の閉じタグの後に親の閉じタグが来る。この順番を守ることが、HTMLを正しく書くための基本中の基本です。

【スライド: インデント — 見た目で構造を示す】

さて、この親子関係をコードで書くとき、もう一つ覚えておきたいことがあります。それは「インデント」です。

インデントとは、行の先頭にスペースを入れて、一段下げることです。子のタグは親のタグよりも一段右にずらして書きます。こうすると、コードを見ただけで「ああ、このタグはこのタグの中に入っているんだな」と、構造が一目でわかるようになります。

インデントはHTMLのルールとして必須ではありません。インデントがなくても、ブラウザはちゃんと表示してくれます。でも、人間がコードを読むとき、そしてエーアイが生成したコードを皆さんがチェックするとき、インデントがあるのとないのとでは読みやすさがまるで違います。

エーアイが生成するHTMLコードも、ほとんどの場合ちゃんとインデントが付いた状態で出てきます。ですから、インデントの意味がわかれば、エーアイのコードを読むのがずっと楽になりますよ。

### セクション4: CodePenで実際に書いてみよう + よくあるミス（5分）

では、実際にCodePenで書いてみましょう。前回の講義で使ったCodePenを開いてください。新しいペンを作成しましょう。左上のCodePenのロゴをクリックして、「Pen」を選ぶと新しいペンが開きます。

まず、シンプルな構造から始めます。HTMLのペインに入力していきますね。

```html
<h1>私のページ</h1>
<p>HTMLのルールを学んでいます。</p>
```

はい、これは前回もやりましたね。h1タグで見出し、pタグで段落。プレビューに「私のページ」という大きな文字と、その下に「HTMLのルールを学んでいます。」という文が表示されているのが確認できると思います。

ここでひとつ確認しておきましょう。今書いたコードでは、h1タグの閉じタグのスラッシュ、ちゃんと入っていますか？pタグの閉じタグも同様です。開くタグと閉じタグがペアになっていることを、目で確認する癖をつけるといいですよ。

では、もう少し内容を増やしてみましょう。

```html
<h1>私のページ</h1>
<p>HTMLのルールを学んでいます。</p>
<p>今日はタグの書き方を覚えました。</p>
<p>楽しいです。</p>
```

pタグを3つに増やしました。プレビューを見ると、見出しの下に3つの段落が並んでいますね。それぞれのpタグがきちんと開いて、きちんと閉じている。これがHTMLの基本形です。

さて、ここからが実はとても大事なパートです。HTMLを書くときに初心者がよくやってしまうミスを見ていきましょう。ミスを知っておくと、自分で書いたときにも、エーアイが生成したコードをチェックするときにも役立ちます。

怖がらなくて大丈夫ですよ。ミスは誰でもやるものですし、CodePenならすぐにやり直せます。

まず、いちばん多いミスが「閉じタグの書き忘れ」です。CodePenで試してみましょう。

```html
<h1>こんにちは
<p>テストです。</p>
```

h1の閉じタグを書き忘れました。プレビューを見てみてください。どうなっていますか？「テストです。」の部分まで、大きな見出しのスタイルで表示されてしまっていますね。ブラウザは「h1がまだ終わっていない」と解釈して、その後のテキストも見出しとして扱おうとしたんです。

閉じタグを1つ忘れるだけで、それ以降の表示がすべておかしくなってしまう。これは引っかかりやすいところなんです。

そして、厄介なことがもう一つあります。プログラミング言語の多くはエラーが出ると動作が止まって「ここがおかしいですよ」と教えてくれるんですが、HTMLはエラーがあっても「なんとか表示しよう」としてくれるんです。赤いエラーメッセージは出ません。親切なようですが、逆に言えば「ミスに気づきにくい」とも言えますね。だからこそ、自分の目でチェックする力が必要になるわけです。

では、直してみましょう。h1の閉じタグを追加します。

```html
<h1>こんにちは</h1>
<p>テストです。</p>
```

はい、これで正しい表示に戻りました。見出しは見出しだけ、段落は段落だけ。ちゃんと分かれていますね。

もう一つ、タイプミスも紹介しておきましょう。たとえば、pタグの閉じタグを書くときに、スラッシュを忘れてしまうことがあります。

```html
<p>テストです。<p>
```

閉じタグのスラッシュがないので、ブラウザはこれを「2つ目のpタグが始まった」と解釈します。結果として、予期しない空白や改行が生まれることがあります。スラッシュ1本の有無で意味がまったく変わってしまうんですね。

こういったミスを見つけるコツをお伝えしますね。まず1つ目、「開くタグと閉じタグが必ずペアになっているか」をチェックすること。コードを書いたら、上から順に「開く、閉じる、開く、閉じる」とペアを確認していきます。

2つ目、インデントをきちんと付けること。インデントが整っていれば、構造が崩れていることにも視覚的に気づきやすくなります。

3つ目、CodePenのプレビューをこまめに確認すること。コードを何行も一気に書いてからプレビューを見るよりも、数行書くたびにプレビューを見る方が、どこでおかしくなったかがすぐにわかります。

プロのエンジニアでも閉じタグの書き忘れはやりますし、エーアイが生成したコードにも時々こういうミスが混ざることがあります。だからこそ、「タグはペアで書く」「入れ子のルールを守る」という基本を知っておくことが大切なんです。

### セクション5: HTMLの基本構造（2分）

【スライド: HTMLの基本構造 — おまじない】

最後にもう一つ、HTMLの「全体の形」についてお話しておきます。

実は、正式なHTMLのファイルには、決まった骨組みがあるんです。画面を見てください。

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <title>ページタイトル</title>
</head>
<body>
  <h1>こんにちは</h1>
  <p>ここが本文です。</p>
</body>
</html>
```

いちばん上の「ドックタイプ エイチティーエムエル」は、「このファイルはHTMLですよ」とブラウザに宣言するためのものです。その下のhtmlタグが全体を包んでいて、その中にheadとbodyの2つがあります。

headは、ページのタイトルや設定情報など、画面には直接表示されない裏方の情報を入れる場所です。bodyは、画面に表示される内容を入れる場所。つまり、私たちがCodePenで書いてきたh1やpは、本来はこのbodyの中に入るわけですね。

さきほどセクション3で「bodyが親、h1やpが子」という例を出しましたが、まさにこの構造のことです。

ただ、今の段階で「毎回この構造を全部書かなきゃいけないの？」と心配する必要はありません。実は、CodePenはこの基本構造を自動で補ってくれるんです。だから私たちはbodyの中身だけ、つまりh1やpだけ書けばちゃんと動くんですね。

「じゃあ覚えなくていいの？」と思うかもしれませんが、この構造を知っておくことには意味があります。エーアイにHTMLを生成してもらうと、このドックタイプから始まる完全な形で出てくることが多いんです。そのときに「ああ、これはおまじないの部分だな、中身はbodyの中だな」とわかるだけで、コードの読みやすさがまるで違います。

今日のところは、「HTMLには決まった骨組みがあるけれど、CodePenでは省略できる」、これだけ覚えておけば大丈夫です。

## まとめ（2分）

【スライド: 今日のまとめ】

それでは、今日のポイントをまとめます。

1つ目。HTMLのタグは、開くタグと閉じタグのペアで構成されています。閉じタグにはスラッシュが入る。この2つで囲まれた範囲が1つの「要素」です。かぎ括弧と同じで、開いたら必ず閉じる。これがHTMLの基本ルールです。

2つ目。タグには「属性」で追加情報を付けることができます。荷物のラベルのように、タグに情報を書き加える仕組みです。この先の講義で実際に使っていきます。

3つ目。タグの中にタグを入れることができます。これが入れ子構造、ネストです。外側が親、内側が子。マトリョーシカ人形のように、子は必ず親の中に完全に収まっていなければいけません。そしてインデントで構造を見やすくします。

4つ目。HTMLには決まった基本構造がありますが、CodePenが自動で補ってくれるので、今はbodyの中身だけ書けばオーケーです。

そして、閉じタグの忘れやタイプミスは、初心者もプロもエーアイもやってしまうミスです。「タグはペアになっているか」「プレビューをこまめに確認する」。この習慣を持つだけで、多くのミスを防げます。

次の講義では、テキストを扱うタグを学びます。見出し、段落、リスト、強調。これらのタグを使うと、文章にちゃんとした構造を持たせることができるようになります。今日学んだ「タグの書き方のルール」を土台にして、使えるタグの種類をどんどん広げていきましょう。

それでは、次の講義でお会いしましょう。お疲れ様でした。
