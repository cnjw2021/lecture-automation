## 本編

### セクション1: 見出しタグ h1からh6（5分）

では、まず見出しタグから始めましょう。前回の講義でh1タグを使いましたよね。h1は「いちばん大きな見出し」でした。実はHTMLには、見出しタグがh1だけではなく、h1からh6まで、6段階も用意されています。数字が小さいほど重要度が高い見出しです。

CodePenで新しいペンを作成して、実際に6つ全部書いてみましょう。HTMLのペインに入力していきますね。

```html
<h1>いちばん大きな見出し</h1>
<h2>2番目の見出し</h2>
<h3>3番目の見出し</h3>
<h4>4番目の見出し</h4>
<h5>5番目の見出し</h5>
<h6>いちばん小さな見出し</h6>
```

プレビューを見てみましょう。上から順に、文字がだんだん小さくなっていくのがわかりますね。h1がいちばん大きくて、h6がいちばん小さい。

これ、新聞や雑誌の構造によく似ています。新聞の一面には大きなトップ見出しがありますよね。これがh1です。その下にセクションの見出しがあって、これがh2。さらにその中の小見出しがh3。こんなふうに、情報の重要度に応じて見出しのレベルを使い分けるんです。

ここで大事なポイントがあります。見出しタグは「文字を大きくするためのタグ」ではありません。「この文章の構造において、ここが一番重要な見出しですよ」「ここはその下のセクション見出しですよ」と、文章の階層構造を伝えるためのタグなんです。

なぜこの区別が大事かというと、ブラウザだけでなく、検索エンジンやエーアイも見出しタグを読んで「このページはどんな構造になっているか」を理解するからです。h1を見て「ああ、このページのメインテーマはこれだな」と判断するわけですね。

実際に使うときのルールとしては、h1はページに1つだけにするのが基本です。そしてh2、h3と順番に使っていく。h1の次にいきなりh4を使うのは、新聞で大見出しの次にいきなり小さなコラム見出しが来るようなもので、構造としてわかりにくくなります。

ただ、これも暗記する必要はありません。エーアイにページを作ってもらうと、だいたい正しい見出し構造で出力してくれます。皆さんはそれを見て「ああ、h1がメインで、h2がセクションで、h3がその中の小見出しなんだな」と読めればオーケーです。

ちなみに、実際のウェブサイトではh1からh3くらいまでがよく使われます。h4以降はあまり使う機会がありません。ですから、今の段階では「見出しは6段階あるけど、よく使うのはh1、h2、h3の3つ」と覚えておけば十分です。

では、いったんこのコードは消して、次のタグに進みましょう。

### セクション2: 段落タグとテキスト装飾（5分）

次は、段落タグのpタグです。これも前回少し使いましたね。pはパラグラフ、つまり段落という意味です。本文の文章を書くときに使うタグで、ウェブページの中でいちばんたくさん使うタグの一つです。

CodePenのHTMLペインを空にして、新しく書いていきましょう。

```html
<h1>私の好きな食べ物</h1>
<p>私はカレーライスが好きです。週に一度は食べています。</p>
<p>特にスパイスから作るカレーが好きで、休日によく作ります。</p>
```

プレビューを見てください。2つのpタグの間に、自然にスペースが入っていますよね。これがpタグの特徴です。ブラウザが「ここは段落の区切りだな」と判断して、適度な余白を自動的に入れてくれるんです。

さて、ここでもう1つ大事なことをお見せします。今のコードの中で、1つ目のpタグの文章を途中で改行してみますね。

```html
<h1>私の好きな食べ物</h1>
<p>私はカレーライスが好きです。
週に一度は食べています。</p>
<p>特にスパイスから作るカレーが好きで、休日によく作ります。</p>
```

HTMLのコード上では2行に分かれていますが、プレビューを見てみてください。改行されていませんよね。1行につながって表示されています。

これ、最初は「あれ？」と思うポイントです。ワードやメモ帳なら、Enterを押せば改行されますよね。でもHTMLでは、コード上でEnterキーを押して改行しても、ブラウザは改行として表示しません。HTMLにとって、コード上の改行はただの空白スペース1つ分として扱われるんです。

これはHTMLの仕様です。HTMLは「タグで意味を伝える」言語なので、改行したいなら改行専用のタグを使う必要があるんですね。

「じゃあ、改行したいときはどうするの？」という疑問が出てきますよね。それは後ほどbrタグで解決します。今は「HTMLではEnterキーだけでは改行されない」ということを覚えておいてください。これ、実際にウェブページを作るときに必ず出てくるポイントなので、しっかり押さえておきましょう。

では次に、テキストを強調するタグを見てみましょう。文章の中で特に目立たせたい部分を強調するために、2つのタグがあります。

```html
<h1>私の好きな食べ物</h1>
<p><strong>カレーライス</strong>が私のいちばんの好物です。</p>
<p>特に<em>スパイスから作るカレー</em>がおすすめです。</p>
```

プレビューを見てください。strongタグで囲んだ「カレーライス」が太字になっていますね。そしてemタグで囲んだ「スパイスから作るカレー」がイタリック、つまり斜体になっています。

strongはストロング、「強い強調」という意味です。「これは特に重要ですよ」とブラウザや検索エンジンに伝えるタグです。見た目としては太字で表示されます。

emはエンファシス、「強調」という意味です。こちらは「ここにニュアンスがありますよ」という柔らかい強調で、見た目としては斜体で表示されます。

ここで注目してほしいのは、strongタグとemタグがpタグの中に入っているということです。前回学んだ入れ子構造ですね。pという段落の中に、strongやemで一部のテキストだけを囲んで強調している。タグの入れ子は、こういうふうに実際の場面で使われるんです。

ちなみに、strongとemのどちらを使えばいいか迷ったら、基本的にはstrongを使えば大丈夫です。ウェブページで強調したいときは太字にすることが圧倒的に多いので、strongの方がよく使われます。emはどちらかというと「ここに注意してね」くらいのニュアンスで、使用頻度はstrongより少ないです。

### セクション3: リストタグ（6分）

さて、ここからは今日のハイライト、リストタグです。ウェブページで「箇条書き」を作るためのタグですね。

リストには2種類あります。「順序なしリスト」と「順序ありリスト」です。まず、順序なしリストから見てみましょう。CodePenのHTMLペインを空にして、新しく書いていきます。

```html
<h2>好きな食べ物リスト</h2>
<ul>
  <li>カレーライス</li>
  <li>ラーメン</li>
  <li>お寿司</li>
</ul>
```

プレビューを見てください。黒い丸のマーカーが付いた箇条書きになっていますね。ワードやメモアプリでよく見る箇条書きが、HTMLでも作れるんです。

ここで2つの新しいタグが出てきました。ulとliです。ulはアンオーダードリスト、つまり「順序のないリスト」という意味です。アンオーダードは「順序がない」ということですね。そしてliはリストアイテム、「リストの各項目」という意味です。

構造を整理しましょう。ulタグが外側の箱で、その中にliタグが並んでいます。前回のマトリョーシカの関係ですね。ulが親で、liが子。そしてインデントでその関係がわかるようになっています。

大事なポイントは、liタグは必ずulタグの中に入れるということです。liだけをポンと書いても、ブラウザはリストとして正しく表示してくれません。ulという「リストの箱」の中にliを入れる。これがルールです。

では次に、順序ありリストを見てみましょう。

```html
<h2>カレーの作り方</h2>
<ol>
  <li>野菜を切る</li>
  <li>肉を炒める</li>
  <li>水を入れて煮込む</li>
  <li>ルーを入れる</li>
</ol>
```

プレビューを見てください。今度は数字が付いた箇条書きになっていますね。1、2、3、4と自動的に番号が振られています。

olはオーダードリスト、「順序のあるリスト」です。ulとの違いは、黒い丸の代わりに番号が付くこと。そして何より、項目の順番に意味があるということです。カレーの作り方は順番が大事ですよね。ルーを入れてから野菜を切っても意味がありません。

使い分けの基準はシンプルです。順番に意味があるならol、順番はどうでもいいならul。好きな食べ物のリストは、カレーが1位でラーメンが2位という意味ではないので、ulを使います。手順やランキングのように順番が重要なら、olを使います。

迷ったらulを使えば大丈夫です。実際のウェブサイトでは、ulの方が圧倒的に多く使われます。メニュー一覧とか、機能リストとか、順番を気にしない場面の方が多いんですね。

では、もう1つ面白いことをお見せしましょう。リストの中にリストを入れてみます。

```html
<h2>今日の予定</h2>
<ul>
  <li>午前
    <ul>
      <li>メールチェック</li>
      <li>会議</li>
    </ul>
  </li>
  <li>午後
    <ul>
      <li>資料作成</li>
      <li>勉強</li>
    </ul>
  </li>
</ul>
```

プレビューを見てください。リストが階層構造になっていますね。「午前」の下にサブ項目として「メールチェック」と「会議」、「午後」の下に「資料作成」と「勉強」。これもまた入れ子構造です。liの中にulが入って、その中にさらにliが入っている。

インデントをよく見てください。構造が深くなるほど、右にずれています。前回学んだインデントの効果が、ここではっきりわかりますよね。インデントがなかったら、どのliがどのulに属しているか、とてもわかりにくくなります。

ちなみに、この入れ子のリストもエーアイが生成するHTMLによく出てきます。メニューやサイドバーの構造などで使われることが多いので、「ああ、リストの中にリストが入っているんだな」と読めるようになっておくと、エーアイのコードを理解するのに役立ちますよ。

### セクション4: 改行と水平線（3分）

さて、セクション2で「HTMLではEnterキーだけでは改行されない」というお話をしましたよね。ではどうやって改行するのか。ここで登場するのが、brタグです。

```html
<p>東京都渋谷区<br>〇〇ビル3階<br>電話: 03-XXXX-XXXX</p>
```

プレビューを見てください。住所と階数と電話番号が、それぞれ別の行に表示されていますね。brはブレイク、つまり「改行」という意味です。さっき「HTMLではEnterを押しても改行されない」と言いましたが、brタグを使えば好きな場所で改行できるんですね。

ここで気づきましたか？ brタグには閉じタグがありません。前回の講義で「一部のタグには閉じタグがない」とお話ししましたよね。brタグはまさにそれです。中にコンテンツを入れるタグではなく、「ここで改行してね」という指示だけのタグなので、閉じタグは不要です。これが「空要素」、セルフクロージングタグでしたね。

ただし、brタグを連続して何個も使って大きなスペースを作る、というのはおすすめしません。余白の調整はCSSで行うのが正しいやり方です。brは住所のように「ここで行を変えたい」という場面で使います。これについてはパート3のCSSで詳しくやりますので、今は「brは改行のタグ」とだけ覚えておいてください。

次に、水平線のhrタグです。

```html
<h1>自己紹介</h1>
<p>趣味や好きなことについて書いています。</p>
<hr>
<h2>趣味</h2>
<p>映画鑑賞が好きです。</p>
<hr>
<h2>好きな食べ物</h2>
<p>カレーライスが好きです。</p>
```

プレビューを見てください。セクションとセクションの間に、横一本の線が入っていますね。hrはホリゾンタルルール、「水平線」という意味です。コンテンツの区切りを視覚的に示すのに使います。

hrタグもbrタグと同じく、閉じタグがない空要素です。中にコンテンツを入れないタグは閉じタグが不要、というルールですね。

今日学んだタグの中で、閉じタグがないのはbrとhrの2つだけです。それ以外のタグ、h1、p、strong、em、ul、ol、liには、すべて閉じタグが必要です。ここを整理しておくと、この先コードを読むときに迷わなくなりますよ。

### セクション5: 実習 自己紹介ページを作ろう（5分）

それでは、いよいよ実習です。今日学んだタグを全部使って、自己紹介ページを作ってみましょう。これまでのセクションでバラバラに試してきたタグを、1つのページにまとめていきます。CodePenで新しいペンを作成してください。いったんHTMLのペインを空にして、最初から一緒に書いていきましょう。

まず、ページの大見出しから始めます。

```html
<h1>山田太郎の自己紹介</h1>
```

名前の部分は、皆さんの好きな名前に変えてくださいね。プレビューに大きな見出しが表示されました。では次に、簡単な紹介文を段落で書きましょう。h1の下にpタグを追加します。

```html
<h1>山田太郎の自己紹介</h1>
<p>はじめまして、山田太郎です。東京で美容師をしています。</p>
```

次に、趣味のセクションを追加します。見出しはh2を使いましょう。ページの中のセクション見出しなので、h1より一段下のh2が適切ですね。

```html
<h1>山田太郎の自己紹介</h1>
<p>はじめまして、山田太郎です。東京で美容師をしています。</p>

<h2>趣味</h2>
<ul>
  <li>映画鑑賞</li>
  <li>料理</li>
  <li>ランニング</li>
</ul>
```

趣味は順番に意味がないので、ulを使いました。そしてliで各項目を並べています。プレビューを見てみましょう。見出しの下に紹介文があって、その下に趣味のリストが黒い丸で並んでいます。だんだんページらしくなってきましたね。

では、もう少し情報を足してみましょう。せっかくolも学んだので、好きな映画のランキングを順序ありリストで作ってみましょう。

```html
<h1>山田太郎の自己紹介</h1>
<p>はじめまして、山田太郎です。東京で美容師をしています。</p>

<h2>趣味</h2>
<ul>
  <li>映画鑑賞</li>
  <li>料理</li>
  <li>ランニング</li>
</ul>

<h2>好きな映画ベスト3</h2>
<ol>
  <li>千と千尋の神隠し</li>
  <li>ショーシャンクの空に</li>
  <li>バック・トゥ・ザ・フューチャー</li>
</ol>
```

映画のランキングは順番に意味があるので、olを使いました。自動的に1、2、3と番号が振られていますね。

いい感じですね。では最後に、strongタグとemタグで強調を入れて、hrタグで区切り線を加えて、連絡先セクションも追加して、ページを仕上げましょう。

```html
<h1>山田太郎の自己紹介</h1>
<p>はじめまして、山田太郎です。東京で<strong>美容師</strong>をしています。</p>

<h2>趣味</h2>
<ul>
  <li>映画鑑賞</li>
  <li>料理</li>
  <li>ランニング</li>
</ul>

<h2>好きな映画ベスト3</h2>
<ol>
  <li>千と千尋の神隠し</li>
  <li>ショーシャンクの空に</li>
  <li>バック・トゥ・ザ・フューチャー</li>
</ol>

<hr>

<h2>連絡先</h2>
<p>メール: yamada@example.com<br>電話: 03-XXXX-XXXX</p>
<p><em>お気軽にご連絡ください。</em></p>
```

プレビューを見てみましょう。「美容師」が太字になって、連絡先のメールと電話番号がbrタグで改行されて、セクションの間にhrタグの区切り線が入っています。そして最後の一文がemタグで斜体になっています。見出し、段落、箇条書きリスト、番号付きリスト、水平線、改行、太字、斜体。今日学んだすべてのタグが入った自己紹介ページが完成しました。

まだ色も付いていませんし、レイアウトも調整していませんが、文章としてちゃんと構造がありますよね。見出しでセクションが分かれていて、リストで情報が整理されていて、区切り線でコンテンツが区分されている。これがHTMLの役割なんです。見た目を整えるのはこの後パート3で学ぶCSSの仕事です。

ここまでで使ったタグを数えてみると、h1、h2、p、strong、em、ul、ol、li、hr、br。全部で10種類のタグを使っています。たった10種類のタグだけで、これだけ構造のあるページが作れるんですね。

皆さんも、名前や趣味、好きなものを自分のものに書き換えて、ぜひオリジナルの自己紹介ページを作ってみてください。リストの項目を増やしたり、hrで区切り線を追加したり、自由にアレンジしてみましょう。

## まとめ（2分）

それでは、今日のポイントをまとめます。

1つ目。見出しタグは、h1からh6まで6段階あります。h1がページの大見出し、h2がセクション見出し、h3がその中の小見出し。順番に使って、文章の階層構造を表します。

2つ目。リストタグは2種類。ulが順序なしリスト、olが順序ありリスト。どちらもliタグで項目を並べます。順番に意味があるかどうかで使い分けます。

3つ目。strongタグは太字で強い強調、emタグは斜体で柔らかい強調。brタグは改行、hrタグは水平線。これらは画面上の表示を変えるだけでなく、文章の意味や構造を伝える役割があります。

そして覚えておいてほしいのは、これらのタグを全部暗記する必要はないということです。暗記する必要はありません。何を検索すべきかを知ればいいんです。「箇条書きを作りたいな」と思ったら「HTML リスト タグ」で検索すればulとliが出てきます。大事なのは「こういうタグがあるんだな」と知っておくことです。

次の講義では、リンクと画像のタグを学びます。aタグでページとページをつなぐリンクを作り、imgタグで画像を表示する。ウェブの核心とも言える機能です。今日作った自己紹介ページに、写真やリンクが加わると、ぐっと本格的なウェブページに近づきますよ。

それでは、次の講義でお会いしましょう。お疲れ様でした。
