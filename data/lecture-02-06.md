## 本編

### セクション1: テーブルの基本構造（4分）

まず、テーブルの構造を理解するところから始めましょう。

テーブルを一番イメージしやすいのは、エクセルのような表計算ソフトです。エクセルを使ったことがある方はわかると思いますが、横の並びが「行」、縦の並びが「列」ですよね。HTMLのテーブルも、まさにこの行と列の組み合わせで成り立っています。

使うタグは4つです。数だけ聞くと多く感じるかもしれませんが、それぞれの役割がはっきりしているので、すぐ理解できますよ。順番に紹介していきますね。

1つ目は、テーブル全体を囲む「テーブルタグ」。これが表そのものです。エクセルで言えば、ワークシート全体のようなイメージですね。このタグの中にすべてのデータが入ります。

2つ目は、テーブルの中の1行を表す「ティーアールタグ」です。TRというのは「テーブル ロー」の略で、「表の行」という意味です。エクセルで言えば、横一列にあたる部分ですね。

3つ目は、見出しのセルを表す「ティーエイチタグ」。THは「テーブル ヘッダー」の略です。表の一番上の行、たとえば「商品名」とか「料金」とか書いてある部分に使います。

4つ目は、普通のデータセルを表す「ティーディータグ」。TDは「テーブル データ」の略ですね。見出し以外の、実際のデータが入るセルに使います。

この4つの関係を整理すると、テーブルタグの中にティーアールタグで行を作って、その行の中にティーエイチやティーディーでセルを並べていく、という入れ子の構造です。リストタグのときと似ていますよね。外側のタグの中に内側のタグが入る、という形です。

では、CodePenを開いて、実際に書いてみましょう。新しいペンを作成してください。準備はいいですか。HTMLの入力エリアに、まずこう書いてみます。一緒に入力していきましょう。

```html
<table>
  <tr>
    <th>メニュー</th>
    <th>料金</th>
  </tr>
</table>
```

入力できましたか。うまく表示されない方は、閉じタグの書き忘れがないか確認してみてくださいね。では、一行ずつ見ていきましょう。まず1行目、テーブルタグで表全体を開始しています。次のティーアールタグで最初の行を作っています。そしてその中に、ティーエイチタグで「メニュー」と「料金」という見出しセルを2つ並べました。最後にティーアールの閉じタグで行を閉じて、テーブルの閉じタグで表全体を閉じています。

プレビューを見てみましょう。「メニュー」と「料金」が太字で表示されていますね。ティーエイチタグは、中身のテキストを自動的に太字にして、中央揃えにしてくれるんです。見出しだとわかりやすいように、ブラウザが気を利かせてくれているわけですね。今は見出しだけなので少し寂しいですが、ここからデータを足していきます。

いい感じです。では、この見出し行の下に、実際のデータの行を追加してみましょう。テーブルの閉じタグの手前に、新しいティーアール行を追加します。

```html
<table>
  <tr>
    <th>メニュー</th>
    <th>料金</th>
  </tr>
  <tr>
    <td>カット</td>
    <td>4,400円</td>
  </tr>
</table>
```

今度は2つ目のティーアールタグを追加して、その中にティーディータグを使いました。ティーエイチではなく、ティーディーです。ここが見出し行との違いですね。ティーディーは普通のデータセルなので、太字にはなりません。標準の文字の太さで、左揃えで表示されます。

プレビューを見てみてください。「カット」と「4,400円」が見出しの下に表示されていますね。見出しのティーエイチは太字なのに対して、データのティーディーは普通の文字ですよね。この違いで、どこが見出しでどこがデータなのか、見る人にもすぐわかるようになっています。2行2列の、小さな表ができました。

ここで大事なポイントをお伝えしておきますね。テーブルの構造は入れ子になっています。テーブルの中にティーアール、ティーアールの中にティーエイチやティーディー。リストタグのときと同じように、親子関係をしっかり意識してください。コードを書くときにインデント、つまり字下げをきちんとしておくと、この親子関係がとても見やすくなります。今書いたコードでも、ティーアールの中身は2段分の字下げをしていますよね。こういう書き方を習慣にしておくと、後からコードを見返すときにとても助かりますよ。

### セクション2: 料金表を作ってみよう（6分）

さあ、基本の構造がわかったところで、もう少し本格的な料金表を作ってみましょう。美容室の料金表をイメージして、メニューを増やしていきます。

先ほどのコードに、行を追加していきましょう。やり方は簡単です。ティーアールの開始タグからティーアールの閉じタグまでのかたまりをコピーして、メニュー名と料金を変えるだけです。カラー、パーマ、トリートメントの3つを追加してみましょう。

```html
<table>
  <tr>
    <th>メニュー</th>
    <th>料金</th>
  </tr>
  <tr>
    <td>カット</td>
    <td>4,400円</td>
  </tr>
  <tr>
    <td>カラー</td>
    <td>6,600円</td>
  </tr>
  <tr>
    <td>パーマ</td>
    <td>7,700円</td>
  </tr>
  <tr>
    <td>トリートメント</td>
    <td>3,300円</td>
  </tr>
</table>
```

はい、入力できましたか。コードの量が増えてきましたが、よく見ると同じパターンの繰り返しですよね。ティーアールで始まって、中にティーディーが2つ入って、ティーアールで閉じる。この繰り返しです。プレビューを見てみましょう。「メニュー」「料金」の見出しの下に、カット、カラー、パーマ、トリートメントがきれいに並んでいます。4つのメニューが一覧になりましたね。

ただ、今のままだと少し見づらいと思いませんか。枠線がないので、どこからどこまでが一つのセルなのか、パッと見てわかりにくいですよね。実はHTMLのテーブルは、デフォルトでは枠線が表示されないんです。

枠線を付けるにはCSSを使うのが正しい方法なんですが、今日はHTMLのテーブル構造を理解することに集中したいので、ちょっとした便利な方法を使いましょう。テーブルタグの開始タグに、ボーダー属性を追加します。

```html
<table border="1">
```

テーブルタグの開始タグの中に「ボーダーイコールいち」と書き足しました。この部分だけ変更すれば大丈夫です。ほかの行はそのままで構いません。プレビューを見てください。どうですか。枠線が付きましたね。ぐっと表らしくなりました。それぞれのセルが四角い枠で囲まれていて、どこまでが一つのセルなのか、はっきりわかるようになりましたよね。

ちなみに、このボーダー属性は古い書き方で、今ではCSSで指定するのが一般的です。でも、テーブルの構造を学んでいる段階では、こうやってHTMLだけで枠線を付けられると便利なので、今回はこれを使っています。CSSでの見た目の調整については、PART 3でじっくり学んでいきますからね。

さて、せっかくなので、もう一歩踏み込んでみましょう。今は「メニュー」と「料金」の2列ですよね。ここに列を1つ追加して、「所要時間」の情報も入れてみます。お客さんの立場で考えると、料金だけでなく所要時間もわかると嬉しいですよね。

列を増やすのは簡単です。すべての行にセルを1つずつ追加するだけです。まず見出し行に、ティーエイチで「所要時間」を追加します。そして各データ行にも、ティーディーで所要時間を追加していきます。全部の行に1つずつ追加するのを忘れないようにしてくださいね。

```html
<table border="1">
  <tr>
    <th>メニュー</th>
    <th>料金</th>
    <th>所要時間</th>
  </tr>
  <tr>
    <td>カット</td>
    <td>4,400円</td>
    <td>約60分</td>
  </tr>
  <tr>
    <td>カラー</td>
    <td>6,600円</td>
    <td>約90分</td>
  </tr>
  <tr>
    <td>パーマ</td>
    <td>7,700円</td>
    <td>約120分</td>
  </tr>
  <tr>
    <td>トリートメント</td>
    <td>3,300円</td>
    <td>約30分</td>
  </tr>
</table>
```

プレビューを見てください。3列の料金表ができましたね。メニュー、料金、所要時間がきちんと揃って表示されています。見出しの行が太字になっているので、どれが項目名でどれがデータなのか、ひと目でわかります。これ、実際の美容室のホームページで見かける料金表とほぼ同じ構造ですよね。もちろんCSSでデザインを整えれば、色を付けたり角を丸くしたりして、もっときれいにできます。でも、HTMLだけでも「データを整理して見せる」という目的は十分に果たせるんです。

ここで一つ注意点です。列を増やすときは、すべての行で同じ数のセルを揃えてください。見出し行が3つのセルなのに、データ行が2つしかセルがないと、表の構造が崩れてしまいます。エクセルと同じで、列の数は行ごとに統一するのがルールです。もし数が合わないと、ブラウザが勝手に空のセルを補おうとして、意図しない見た目になることがあります。

ちなみに、今回は美容室を例にしましたが、カフェなら「ドリンク名、サイズ、価格」の3列、教室なら「コース名、月謝、回数」の3列、整体院なら「施術名、料金、所要時間」の3列に置き換えるだけです。テーブルの構造自体はまったく同じなので、中身を変えるだけで色々な業種に使い回せますよ。実際にPART 4では、サロンの紹介サイトを作るプロジェクトがあるんですが、そこでも今日学んだテーブルを使って料金表を作ります。今日の内容がそのまま活きてきますので、楽しみにしていてくださいね。

### セクション3: テーブルの使いどころ（2分）

さて、テーブルの作り方がわかったところで、もう一つ知っておいてほしいことがあります。それは、テーブルをどういう場面で使うべきか、ということです。

テーブルが得意なのは、今回のような料金表のほか、営業時間の一覧、商品の比較表、時間割やスケジュールなど、「行と列で整理できるデータ」を表示する場面です。共通しているのは、もとのデータ自体が表の形をしている、ということですね。こういうデータを見せるなら、テーブルタグを使うのが最も適切です。エクセルに入れるようなデータなら、HTMLでもテーブルで表示する、と考えるとわかりやすいかもしれません。

逆に、ページ全体のレイアウトをテーブルで組むのは推奨されていません。表形式のデータにはテーブル、ページのレイアウトにはCSSを使う。この使い分けだけ覚えておいてください。レイアウト用のCSSについてはPART 3で学んでいきます。

もちろん、暗記する必要はありません。テーブルタグの細かい書き方を全部覚えなくても大丈夫です。「表を作りたいときはテーブルタグを使うんだ」「ティーアールで行を作って、ティーエイチとティーディーでセルを作るんだ」という構造がわかっていれば、あとは必要になったときに調べればちゃんと書けます。何を検索すべきかがわかっていることが大事なんです。
