## 本編

### セクション1：HTMLの基本構造をもう少し詳しく見る

【スライド: HTMLの全体像をもう一度】

まずは、前回見たHTMLの基本構造をもう一度出してみましょう。

```html
<!DOCTYPE html>
<html>
  <head>
    <title>はじめてのページ</title>
  </head>
  <body>
    <h1>こんにちは、山田です！</h1>
    <p>ブラウザだけでホームページ作りを始めました。</p>
  </body>
</html>
```

この形を見たときに、前回は「箱の組み合わせ」として見ましたよね。今日は、そこからもう一歩だけ進めて、それぞれが何をしているのかを、今の段階で必要な範囲だけ整理しましょう。

まず、`<head>` です。ここは、画面に大きく見える内容を書く場所ではありません。ブラウザに伝えておきたい設定や、裏側の情報を書く場所です。

たとえば、`<title>` はブラウザのタブに出る名前です。ページの上に大きく表示される見出しではありません。ブラウザのタブや検索結果のタイトルとして使われる、とても大事な情報です。

ほかにも、スマートフォン表示に必要な `meta` タグや、CSSファイルを読み込むための `link` タグなども、本来はheadに入ります。つまり、headは「ブラウザへの説明書き」のような場所なんですね。

ここで `meta` や `link` という言葉が出てくると、急に難しく見えるかもしれません。でも心配しなくて大丈夫です。今は、「そういう設定系のものがheadに集まるんだな」とわかれば十分です。細かい書き方は、必要になったときにまた見れば大丈夫です。

一方、`<body>` は実際に画面に見える内容を書く場所です。見出し、文章、画像、ボタン、リスト。ユーザーが目で見るものは、基本的にbodyに入ります。

【スライド: headは設定、bodyは中身】

ここで大事なのは、役割を混ぜて考えないことです。headは設定、bodyは中身。この2つを分けて考えるだけでも、HTMLがかなり整理しやすくなります。

家で例えると、bodyは実際の部屋です。人が入って見て、使う場所ですね。headは、設計図や設備の説明書のようなものです。普段は見えないけれど、家として機能するために必要な情報が入っている。そう思うとイメージしやすいと思います。

この整理ができるだけでも、あとでAIが生成したHTMLを見たときに「これは画面に見える部分だな」「これは設定側だな」と少しずつ見分けやすくなります。つまり、今日やっていることは、AI時代にコードを読む準備でもあるんです。

AIが出したコードを見たときに、全部が同じように見えてしまうと不安になりますよね。でも、headとbodyの役割が少しでも分かっていると、「全部を理解できなくても、まず見える部分から見ればいいんだ」と落ち着いて向き合えるようになります。これはかなり大きな違いです。

もちろん、今の段階ではheadの中身を全部暗記する必要はありません。大切なのは、「画面に見えるものはbodyに書く」「ブラウザへの設定はheadにある」という大きな整理です。

### セクション2：CodePenのHTMLペインに書いているのは何か

【スライド: CodePenではbodyの中身を書く】

ここで、今までのCodePenの体験とつなげましょう。

前回も前々回も、CodePenのHTMLペインに文字やタグを書きました。でも、そのとき、皆さんは `<!DOCTYPE html>` も `<html>` も `<head>` も、ほとんど書いていません。それでもちゃんと表示されましたよね。

これは、CodePenが学習しやすいように、外側の基本構造をある程度補ってくれているからです。つまり、皆さんがCodePenのHTMLペインに書いているものは、感覚としては主にbodyの中身なんです。

だから、今の段階ではCodePenのHTML欄に見出しや文章を書いていけば大丈夫です。「あれ、外枠を書いていないのに大丈夫なのかな」と不安になる必要はありません。CodePenがその部分を助けてくれているからです。

【スライド: 学習用に複雑さを減らしている】

これは初心者にとって、とても良いことです。なぜなら、最初から外枠まで全部を書くと、どこに何を書いているのかが見えにくくなるからです。

学習の最初は、少し複雑さを減らしてもらったほうが前に進みやすいです。いきなり全部を自分でやるより、「今はここだけ触ればいい」とはっきりしているほうが安心して試せるからです。CodePenは、その意味でとても親切な環境なんですね。

今はまず、bodyの中身を作ることに集中しましょう。画面に見える見出しを書く。文章を書く。あとで画像を入れる。こういう「見えるもの」を積み上げることが、今の段階で一番大切です。

もちろん、後で本格的な環境に進めば、自分でheadや外部CSSの読み込みも扱います。でも、その前に「何を表示したいのか」を作れるようになることのほうが大事です。だから、この順番で進めているんですね。

### セクション3：CodePenの便利な機能を少し使ってみる

【スライド: CodePenの便利機能】

では、ここからはCodePenをより使いやすくする小さな機能を見ていきます。今日は全部を覚えなくて大丈夫です。「こういう便利なものがあるんだな」と知っておくだけで十分です。

まず1つ目は、保存のショートカットです。Windowsなら `Ctrl+S`、Macなら `Command+S` ですね。前回も保存ボタンを使いましたが、ショートカットを知っておくと少し楽になります。

保存を知っておくことは、単に便利というだけではありません。「今の状態を残せる」と思えると、人は少し大胆に試せるようになります。学習では、この安心感が意外と大事なんです。

2つ目は、元に戻す操作です。Windowsなら `Ctrl+Z`、Macなら `Command+Z` です。これはかなり重要です。少し変えてみて、やっぱり戻したいと思ったときにすぐ使えるからです。学習では、安心して試すために「戻せる」ことを知っているのが大切です。

実際、手が止まる理由の一つは、「変えて壊れたらどうしよう」です。でも、戻せるとわかっていると、かなり楽になります。ですから、保存と元に戻す、この2つだけでも覚えておく価値は十分あります。

3つ目は、Emmetです。これは、HTMLを少し楽に書くための補助機能です。名前だけ聞くと難しそうですが、使い方はシンプルです。

たとえば、HTML欄にこう書いてみます。

```html
ul>li*3
```

そして展開すると、こんな形になります。

```html
<ul>
  <li></li>
  <li></li>
  <li></li>
</ul>
```

一気にリストの形ができましたね。初心者のうちは、まず普通にタグを書けることのほうが大切です。でも、「こういう近道もある」と知っておくと、あとで少しずつ楽になります。

【スライド: Emmetは近道、基本が先】

ここで大事なのは、Emmetを覚えることが目的ではない、ということです。あくまで補助です。基本のタグの意味がわからないまま近道だけ覚えても、後で困ります。ですから今日は、「便利そうだな」くらいで大丈夫です。

言い換えると、Emmetは魔法ではなく省略記法です。意味を理解した人が使うと便利。でも、意味がわからないままだと、何が起きたのかが見えにくくなる。だからこの講座では、基本を先にしているんですね。

もう一つ見ておきたいのが、Settingsです。CodePenには、HTMLやCSSに関する設定を追加できる画面があります。特にHTMLのSettingsでは、headに入れたいタグを追加することができます。

たとえば、あとでスマホ表示に必要なmetaタグを入れたいときや、外部のCSSライブラリを読み込みたいときに使います。具体的には、PART 3でおしゃれな日本語フォントを使うときに、このSettings画面からGoogle Fontsを読み込む場面が出てきます。今日の段階では、ここを自由自在に使えなくても問題ありません。ただ、「CodePenのHTML欄はbodyの中身で、headに入れたいものはSettings側で足せる」という感覚を持っておいてください。

この理解があるだけでも、「CodePenのHTML欄にheadを書き始める」ような混乱が減ります。今はbody中心。設定系はSettingsや後の本格環境で扱う。この整理ができていれば十分です。

### セクション4：簡単な自己紹介ページを作ってみよう

【スライド: 自己紹介ページを作る】

では、ここから実習です。前回のHello Worldを少し進めて、簡単な自己紹介ページを作りましょう。

[ライブ開始]

まず、HTML欄に見出しを書きます。ここには自分の名前を入れてみましょう。

```html
<h1>山田 花子</h1>
```

もちろん、本名でなくても大丈夫です。ニックネームでも、仮の名前でも構いません。大事なのは、「自分のページの見出し」として使ってみることです。

次に、その下に短い自己紹介の文章を入れます。

```html
<p>東京で小さな教室を運営しています。ホームページ作りを勉強中です。</p>
```

これだけでも、もうかなり「ページらしさ」が出てきますよね。Hello Worldの一行だけだったものが、見出しと説明のあるページになりました。

ここで自己紹介ページを題材にしているのには理由があります。内容を考えやすいからです。自分の名前、ひと言紹介、イメージ画像。この3つだけでも、かなり自然にページらしくなります。最初の題材としてとても相性がいいんですね。

しかも、自己紹介ページは業種を問わず応用しやすいです。会社員の方なら自己紹介、個人事業の方ならお店紹介の入口、教室運営の方なら講師紹介の土台になります。つまり、練習用でありながら、この先の実用にもつながりやすい題材なんです。

ここで大切なのは、完璧な文章を書くことではありません。とりあえず短く、自分らしい内容を入れてみることです。あとで何度でも直せますから、まずは形にしましょう。

【スライド: 画像にも挑戦してみる】

次は、画像に少しだけ挑戦してみます。今の段階では、自分のパソコンの画像をアップロードするより、まずは外部URLの画像を使うほうが簡単です。

たとえば、こんな形です。

```html
<img src="https://images.unsplash.com/photo-1502685104226-ee32379fefbe?auto=format&fit=crop&w=600&q=80" alt="プロフィールイメージ">
```

これを入れると、画像が表示されます。ここで `alt` という属性も一緒に入れています。今は深く覚えなくて大丈夫ですが、「画像の説明文のようなもの」と理解しておけば十分です。後でアクセシビリティの話をするときに、また出てきます。

もう少しだけ言うと、altは「もし画像が見えないときに、その画像が何なのかを伝えるための言葉」です。今の段階では、完璧に書けなくても問題ありません。でも、「画像には説明を添えることがある」という感覚を持っておくと、あとでとても役に立ちます。

もし画像が大きすぎる場合は、今は気にしすぎなくて大丈夫です。あとでCSSで調整できますし、まずは「画像も入れられるんだ」と知ることが大切です。

そして、ここでも完璧を目指しすぎないことが大切です。少し大きい、少し見づらい、そのくらいなら今は問題ありません。今の目標は、部品を置けることを知ることだからです。

実際の制作でも、最初からすべてが整っていることはほとんどありません。まず部品を置く。そのあと、サイズや位置を調整する。この順番です。今は、まさにその最初の段階をやっています。

### セクション5：少し整えて、CodePenを使いこなす感覚をつかむ

【スライド: 少しずつページらしくなる】

ここまでで、見出し、文章、画像の3つが入りました。もう立派な自己紹介ページの入口ですね。

ここで、少しだけ見直してみましょう。名前が見出しになっているか。文章が自分の内容になっているか。画像が表示されているか。たったこれだけでも、自分のページとしてかなり意味を持ち始めます。

もし文章を変えたくなったら、すぐ直せばいいです。画像を別のものにしたくなったら、URLを変えればいいです。これがCodePenの良いところで、試して、見て、また変える、がとてもやりやすいんですね。

【スライド: 保存して、あとでまた開けるようにする】

ここまで来たら、保存もしておきましょう。ショートカットでもボタンでも大丈夫です。タイトルも、あとで見てわかりやすい名前に変えておくとよいですね。たとえば、「講義07 自己紹介ページ」などで十分です。

保存しておく意味はとても大きいです。なぜなら、今日の作業が次の講義の土台になるからです。学習は毎回ゼロから始めるより、前回の続きから始めたほうがずっと進みやすいです。

もし時間があれば、一度保存したあとに、自分の一覧からそのペンを開き直してみてもよいですね。「ちゃんと残っている」「また戻ってこられる」と確認できると、CodePenが単なる練習画面ではなく、自分の作業場所だと感じやすくなります。

この感覚は、学習を続けるうえでかなり大切です。人は「自分の作業場所」があると、戻ってきやすくなります。毎回ゼロから始める感覚ではなく、「前回の続きがある」と感じられるからです。
