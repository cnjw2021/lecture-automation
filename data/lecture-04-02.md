## 本編

### セクション1: HTML構造を把握する（8分）

では、前回の講義で作ったCodePenを開いてください。Claudeが生成したサロンサイトのコードが入っていますね。プレビューにはヒーローセクション、こだわり紹介、施術メニュー、スタッフ紹介、アクセス情報、お問い合わせフォーム。こういった要素が並んでいるはずです。

一番上にヘッダーがありますね。headerタグの中にナビゲーション、navタグが入っています。サロンの名前がロゴのように表示されていて、その横にメニューリンクが並んでいます。覚えていますか。パート2の4番目の講義で学んだセマンティックタグですね。headerは「ページの上の部分」、navは「ナビゲーション、つまりメニューのリンク集」。意味のあるタグで構造を伝えている。まさにあの講義で学んだことそのものです。

ナビゲーションのリンクも見てみましょう。aタグが並んでいますね。href属性にシャープ付きの文字列が入っているのが見えますか。例えば href="#menu" とか href="#access" のような形です。これはページ内リンクといって、同じページの中の特定の場所にジャンプするためのリンクです。aタグ自体はパート2の3番目の講義で学びましたよね。リンクの使い方の応用です。

その下に、大きな写真と文字が重なった部分がありますね。ここがヒーローセクションです。sectionタグで囲まれていて、クラス名に「hero」とか「hero-section」みたいな名前がついているはずです。パート3のセレクタの講義で学んだクラスですね。ドットの後ろに書く、あのニックネームです。AIも同じようにクラス名を使って、要素を区別しているんです。

ヒーローセクションの中を見ると、h1タグで大きな見出しがあって、pタグでサロンのキャッチコピーや説明文があって、aタグやbuttonタグで予約ボタンがある。全部、パート2で書いたことのあるタグです。h1が大見出し、pが段落、aがリンク。パート2の最初の講義から使ってきたおなじみのタグですよね。

さらに下にスクロールしていくと、こだわり紹介のセクション、施術メニューのセクション、スタッフ紹介のセクション。それぞれがsectionタグで区切られています。パート2で「sectionは章のようなもの」と説明しましたよね。一つの章がひとまとまりの内容を持つ。サロンサイトでもまったく同じ使い方です。そしてそれぞれのsectionにクラス名がついていて、CSSからスタイルを当てるための目印になっています。

施術メニューの部分を見てみましょう。パート2の6番目の講義で料金表を作りましたよね。テーブルタグを使って、施術名と時間と料金を並べました。AIが生成したコードでも、テーブルが使われているか、あるいはdivタグとクラス名を使ったカード形式になっているかもしれません。テーブルならtr、th、tdでおなじみの構造ですし、カード形式でもdivの中にh3で施術名、pで説明文という、やはり見覚えのあるタグの組み合わせです。

お問い合わせフォームの部分も見てみましょう。formタグ、inputタグ、textareaタグ、buttonタグ。パート2の5番目の講義で作った予約フォームと同じタグですよね。inputのタイプも、textやemail、telなど、見覚えのあるものが並んでいるはずです。required属性がついているものもありますね。これも5番目の講義で「入力必須にする属性」として学びました。

最後に、ページの一番下にfooterタグがあります。著作権表示やSNSリンクなどが入っていますね。header、nav、main、section、footer。パート2で学んだセマンティックタグが、そのまま使われているのがわかります。

もう一つ気づいてほしいことがあります。AIが生成したHTMLの中で、クラス名がとても読みやすくなっていることです。hero-section、menu-card、staff-member、contact-form。英語が苦手な方でも、なんとなく意味がわかりますよね。heroは「主役」、menuは「メニュー」、staffは「スタッフ」。AIが自分でわかりやすいクラス名を選んでくれているんです。パート3のセレクタの講義で「クラスはニックネーム」と言いましたよね。AIは上手にニックネームをつけてくれています。

ここで大事なことを確認しましょう。AIが生成したHTMLは、何か特別な技術で作られているわけではありません。皆さんがパート2で学んだタグの組み合わせです。見出し、段落、リスト、リンク、画像、フォーム、テーブル。どれも一度は自分の手で書いたことがあるタグです。量は確かに多い。でも一つ一つを見れば「ああ、これは知っているぞ」と思えるはずです。量が多いだけで、やっていることは同じなんです。

### セクション2: CSS構造を分析する（8分）

次に、CSSペインを見ていきましょう。こちらはHTMLに比べるとコードの量がさらに多いかもしれません。スクロールしても終わらないくらい長いかもしれませんね。でも、焦らなくて大丈夫です。全部を一行ずつ読む必要はありません。大きな構造をつかむことが大事です。パート3で学んだ知識があれば、「このあたりは何をやっている部分か」がわかるようになります。上から順に確認していきましょう。

まず、CSSの一番上のあたりを見てください。ページ全体に適用されるスタイルが書いてあるはずです。アスタリスクやbodyに対して基本的なスタイルが設定されていますね。

アスタリスクに box-sizing: border-box と書いてあったら、覚えていますか。パート3の4番目の講義、ボックスモデルの回で学んだプロパティです。「これさえ知れば計算が楽になる」と言いましたよね。paddingやborderを含めてもサイズが変わらなくなる設定です。AIもこれを最初に書いているんです。プロの開発者と同じ定石ですね。

bodyに対してはフォントの指定がありますね。font-familyでフォントの種類を指定して、colorで文字の色を設定して、margin: 0でページの余白をリセットしている。パート3の3番目の講義、色とタイポグラフィで学んだ内容です。

次に、色の体系を確認してみましょう。AIが生成したCSSの中で使われている色のコードをざっと眺めてみてください。前回のプロンプトで「ベージュと深緑系」とお願いしましたよね。ヘッダーの背景色、ボタンの色、セクションの背景色。それぞれにベージュ系や深緑系のカラーコードが使われているはずです。パート3の3番目の講義で学んだHEXカラーコード。シャープの後に6桁の英数字。これでどんな色かが決まる。AIが選んだ色のコードも、同じ仕組みです。サイト全体で使われている色を見てみると、2色から3色くらいの基本色を決めて、それを統一的に使っていることがわかります。プロのウェブサイトが「色に統一感がある」のは、こういう仕組みなんですね。

では、レイアウトの部分を見てみましょう。ナビゲーションのメニューが横に並んでいますよね。CSSの中に display: flex が見えますか。パート3の5番目の講義で「1行追加するだけで魔法が起きる」と言ったフレックスボックスです。ナビゲーションのリンクが横に並んでいるのも、スタッフ紹介のカードが横に並んでいるのも、施術メニューのカードが横に並んでいるのも、全部フレックスボックスのおかげなんです。

justify-contentやalign-items、gapといったプロパティも見えますか。主軸と交差軸の話を思い出してください。justify-content: center は主軸の中央寄せ、align-items: center は交差軸の中央寄せ。gap: 20px はアイテム間の間隔を20ピクセルに設定。パート3で学んだことがそのまま使われています。

flex-wrapも書いてあるかもしれません。flex-wrap: wrap。これはアイテムが収まりきらないときに折り返す設定でしたね。画面が小さくなったときにカードが下に折り返される。パート3の5番目の講義で触れた内容です。

ボックスモデルの要素も見てみましょう。paddingやmarginがたくさん出てきますよね。カードの内側の余白はpadding、カードとカードの間隔はmarginやgap。パート3の4番目の講義で「すべてのHTML要素は箱だ」と説明したボックスモデル。content、padding、border、margin。このモデルを使ってレイアウトが組まれているのがわかります。

もう一つ確認してほしいのが、レスポンシブ対応の部分です。CSSの下の方にスクロールしていくと、アットマークメディアという記述が見えるはずです。パート3の6番目の講義で学んだメディアクエリですね。画面の幅が768ピクセル以下になったときに、レイアウトを切り替えるための仕組みです。例えば、パソコンでは横に並んでいたカードが、スマートフォンでは縦に1列に並ぶ。flex-directionをcolumnに変えたり、widthを100パーセントにしたり。その切り替えがメディアクエリで実装されています。

試しに、プレビューを見ながらCSSの一つの値を変えてみましょう。例えば、ヘッダーの背景色のカラーコードを別の色に変えてみてください。プレビューが即座に変わりますよね。パート3で何度も体験した「値を変えるとすぐに反映される」という感覚、ここでも同じです。CSSを読めるということは、こうやって自分で好みの見た目に変えられるということです。元の色に戻しておいてくださいね。

ここまでの内容を振り返ると、AIが生成したCSSも、パート3で学んだ知識の組み合わせだということがわかります。フォント、色、ボックスモデル、フレックスボックス、メディアクエリ。特別なことは何もなくて、皆さんがすでに知っている技術で作られている。これはとても大事なポイントです。コードの量は多い。でも、知らないものはほとんどない。AIが作ったものを読んで理解できるということは、必要に応じて自分で修正できるということですから。

ただし、CSSの中にいくつか「あれ、これは見たことないぞ」というプロパティもあったはずです。box-shadowとかborder-radiusとか。安心してください。ここからはまさにその、見たことのないプロパティを一つずつ見ていきます。

### セクション3: 新テクニック紹介 — box-shadow（4分）

さて、新しいCSSテクニックに入ります。今日は4つを取り上げます。でも安心してください。どれも1行追加するだけのシンプルなテクニックです。そして、「暗記してください」とは言いません。こういうものがあるんだと知っていれば、必要なときにAIに「ここにbox-shadowを追加して」と指示できますから。プロパティの名前を知っていることが大事なんです。

まず最初は、box-shadowです。ボックスシャドウ。名前の通り、ボックスに影をつけるプロパティです。

AIが生成したコードの中で、施術メニューのカードやスタッフ紹介のカードの部分を見てください。CSSにbox-shadowという記述がありますね。

```css
box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
```

この1行で、カードにふわっとした影がつきます。数字がたくさん並んでいて難しそうに見えますが、一つずつ見ていけば大丈夫です。最初の0は横方向のずれです。影が左右にずれない。次の2pxは縦方向のずれ。影が少しだけ下にずれる。光が上から当たっている感じですね。3番目の8pxはぼかしの大きさです。この値が大きいほど、影がぼんやりと広がります。そしてrgbaの部分が影の色と透明度です。0, 0, 0は黒。パート3の色の講義でrgbを学びましたよね。rgbaの「a」はアルファ、つまり透明度のことです。最後の0.1は「10パーセントの不透明度」。つまりかなり薄い、うっすらとした影ということですね。

実際に違いを確認してみましょう。この行を一旦削除してみてください。カードが平坦になって、背景に貼り付いたように見えますよね。のっぺりした印象です。もう一度元に戻すと、影のおかげでカードが少し浮き上がって見える。紙のカードがテーブルの上に置いてあるような、ちょっとした立体感が出ますよね。このわずかな影が、プロのウェブサイトっぽさを出してくれるんです。

ちなみに、この値を変えるだけで影の印象がガラッと変わります。例えばぼかしの8pxを20pxに変えると、もっとふわっとした大きな影になります。逆に2pxくらいにすると、くっきりした影になる。いろいろ試してみると面白いですよ。ただ、今は値を覚える必要はありません。AIが適切な値を選んでくれていますし、もし影の印象を変えたければ「もう少し影を強くして」とAIにお願いすれば済みますから。

### セクション4: 新テクニック — border-radius（4分）

次のテクニックは、border-radiusです。ボーダーレイディアス。これは要素の角を丸くするプロパティです。

カードの部分のCSSを見てください。border-radiusという記述がありますね。

```css
border-radius: 8px;
```

この1行で、四角いカードの角がほんの少し丸くなります。8ピクセル分の丸みがつくという意味です。試しにこの値を変えてみましょう。0pxにすると完全な直角になります。20pxにするとかなり丸くなります。数字を大きくすればもっと丸くなりますし、小さくすればほぼ直角に近くなります。元の8pxに戻しておきましょう。

直角のカードと、角が少し丸いカード。見比べてみると、丸い方が柔らかい、親しみやすい印象がありますよね。サロンのサイトですから、硬い直角よりも、こういう優しい雰囲気の方が合っています。こういうデザインの判断を、AIはプロンプトの内容から汲み取って自動的にやってくれているんですね。

スタッフ紹介の写真部分にも注目してみてください。写真に対してこんな指定があるかもしれません。

```css
border-radius: 50%;
```

50パーセントにすると、四角い画像が完全な丸になります。やってみましょう。正方形の画像が丸く切り抜かれたように表示されますよね。これ、実はとてもよく使われるテクニックなんです。プロフィール写真やアイコンを丸く表示するとき、画像編集ソフトで丸く切り抜いているわけではなくて、CSSのborder-radius: 50%で表示時に丸く見せているんです。SNSのプロフィール画像が丸いのも、まさにこのCSSのおかげです。エックスやインスタグラム、いろんなサービスのプロフィール画像が丸いですよね。元の画像は四角いまま、表示だけ丸くしている。そういう仕組みなんです。

ボタンにもborder-radiusがついていますね。試しに、ボタンのborder-radiusを0pxに変えてみてください。角が完全に直角のボタンになります。そしてまた8pxに戻す。比べてみると、少し丸みのあるボタンの方が押したくなりませんか。ほんの少しの丸みが、ボタンを柔らかく親しみやすい印象にしてくれるんです。

### セクション5: 新テクニック — hover効果（4分）

3つ目のテクニックは、hover効果です。ホバー効果。マウスカーソルを要素の上に乗せたときに、見た目が変わる仕組みです。

CSSの中でコロンhoverという記述を探してみてください。ボタンの部分に書いてあるはずです。

```css
.button:hover {
    background-color: #2d5a3d;
}
```

これは「buttonというクラスがついた要素の上にマウスを乗せたとき、背景色をこの色に変える」という意味です。コロンhoverというのは、パート3のセレクタの講義では触れなかった新しい書き方です。擬似クラスと呼ばれるもので、要素の「状態」に応じてスタイルを変えることができます。でも名前は覚えなくて大丈夫。使い方はシンプルです。セレクタの後にコロンhoverをつけるだけで、マウスが乗ったときだけ適用されるスタイルを指定できます。

実際にプレビューで試してみましょう。CodePenのプレビューエリアで、ボタンの上にマウスカーソルをゆっくり持っていってみてください。色が変わりましたよね。マウスを離すと元に戻る。もう一度乗せると変わる、離すと戻る。何度やっても同じように反応します。

これがあると何がいいかというと、クリックできることが直感的にわかるんです。普段ウェブサイトを使っていて、リンクやボタンの上にマウスを乗せると何かが変わりますよね。色が変わったり、下線が出たり。「ああ、これはクリックできるんだな」と自然に理解できる。ユーザーにとっての使いやすさに直結するテクニックです。

ボタンだけでなく、ナビゲーションのリンクやカードにもhover効果がついているかもしれません。CSSの中で「hover」という文字列を探してみると、いくつか見つかるはずですよ。ナビゲーションリンクの色が変わったり、カード全体がほんの少し動いたり。ウェブサイトの「触れると反応がある」という感覚は、ほとんどこのhover効果で実現されています。

ちなみに、hoverの中で変えるのは背景色だけに限りません。文字色を変えたり、border-radiusで角の丸みを変えたり、先ほどのbox-shadowで影を濃くしたり。いろいろな変化を組み合わせることができます。AIが生成したコードの中にも、背景色以外のhover変化が含まれているかもしれませんね。

### セクション6: 新テクニック — transition（4分）

最後、4つ目のテクニックは、transitionです。トランジション。これは変化を「滑らかに」するためのプロパティです。

先ほどのhover効果を思い出してください。ボタンにマウスを乗せたら色が変わりましたよね。ちょっと注意して見てみてください。色がパッと一瞬で切り替わっていませんか。乗せた瞬間にパチッと変わる。これでも悪くはないんですが、ちょっとした工夫で印象がぐっと変わります。

ボタンのCSSを見てください。hover効果が書いてある少し上のあたりに、transitionという記述がありますね。

```css
transition: background-color 0.3s ease;
```

この1行がある状態でボタンにマウスを乗せると、色の変化が0.3秒かけてふわっと切り替わります。パッと変わるのではなく、じんわり変わる。違いがわかりますか。たった1行追加するだけで、サイト全体の印象がぐっとプロっぽくなるんです。

数字の意味を見てみましょう。background-colorは「背景色の変化に対して」という指定です。0.3sは「0.3秒かけて」。easeは「最初と最後をゆっくり、途中を速く」という変化のしかたです。自然界の動きに近いカーブなので、人間にとって一番心地よく感じる変化のパターンと言われています。
