### セクション3: AIが作ったCSSを分析する（6分）

見た目の変化を楽しんだところで、CSSペインのコードを上から順に見ていきましょう。パート3で学んだ知識を総動員して、AIが何を書いたのか読み解いていきます。

まず一番上の方を見てください。おそらく body に対するスタイルが書いてありますね。こんな感じのコードが見えると思います。

```css
body {
  font-family: 'Noto Sans JP', sans-serif;
  background-color: #f8f9fa;
  color: #333333;
  line-height: 1.8;
}
```

これは講義3-3で学んだ内容ですよね。font-family でページ全体のフォントを指定しています。ノトサンスジェーピーというのはGoogleが提供している日本語フォントですね。カンマの後の sans-serif は、もしこのフォントが読み込めなかった場合の代替指定です。

background-color で背景の色、color で文字の色を決めている。値を見ると、シャープで始まる6桁のカラーコードが使われています。シャープ f8f9fa は薄いグレー系、シャープ 333333 は濃いグレー。真っ白と真っ黒ではなく、少し柔らかい色を選んでいるんですね。こういう微妙な色使いが、ページの印象をぐっと良くしてくれます。

line-height は行と行の間隔でしたね。1.8 というのは文字サイズの1.8倍の行間です。body に書くことで、ページ全体の基本設定になっています。皆さんがパート3で書いたCSSにも似たようなプロパティがあったかもしれませんが、AIが選んだ値を見てみてください。プロのウェブデザインでよく使われる値が選ばれているはずです。

次に、少し下を見ていくと、h1 や h2 に対するスタイルが出てきますね。

```css
h1 {
  font-size: 28px;
  color: #2c3e50;
  margin-bottom: 16px;
}
```

h1 や h2 はタグセレクタです。HTMLの見出しタグに直接スタイルを当てている。font-size で文字の大きさを指定して、color で見出しの色を決めて、margin-bottom で見出しの下に余白を取っている。一つ一つのプロパティが、もう読めますよね。

そして見てください。ドットで始まるクラスセレクタも使われていませんか。たとえば .skills とか .contact のような名前です。講義3-2で学んだセレクタの知識で、「ああ、HTMLの中でこのクラス名がついた要素に、このスタイルが当たっているんだな」と読み取れますよね。タグセレクタとクラスセレクタ、両方をAIも使い分けているわけです。

padding や margin の指定もたくさん見つかるはずです。

```css
section {
  padding: 32px 24px;
  margin-bottom: 24px;
}
```

padding が内側の余白で、margin が外側の余白。講義3-4で学んだボックスモデルですね。開発者ツールであの箱の図を見たのを思い出してください。padding: 32px 24px というのは、上下が32ピクセル、左右が24ピクセルの内側余白です。こうやって数値を見れば、どのくらいの余白が設定されているかがピクセル単位で正確にわかります。

余白の数値をよく見比べてみてください。AIはセクションの padding やリスト周りの margin を丁寧に設定していますよね。余白がないと文字や要素がぎゅうぎゅうに詰まって読みにくくなりますが、適度な余白があると、ぐっと見やすくなる。余白のデザインって、実はウェブページの読みやすさにとても影響が大きいんです。

さて、display: flex という記述を見つけた方もいるのではないでしょうか。

```css
nav {
  display: flex;
  justify-content: center;
  gap: 24px;
}
```

来ましたね、フレックスボックスです。講義3-5で学んだ現代ウェブレイアウトの核心です。justify-content で配置を揃えて、gap でアイテム間の間隔を取っている。AIもフレックスボックスを使うんです。それだけ、現代のCSSではフレックスボックスが基本中の基本になっているということですね。

そして、CSSの下の方にアットマーク media で始まるブロックがないか探してみてください。

```css
@media (max-width: 768px) {
  nav {
    flex-direction: column;
    gap: 12px;
  }
}
```

前回の講義で学んだばかりのメディアクエリです。画面幅が768ピクセル以下になったら、ナビゲーションの並びを縦に変える。flex-direction を column に切り替えていますね。AIもちゃんとレスポンシブ対応を入れてくれているんです。「指示していないのにレスポンシブまで考えてくれるの？」と思った方もいるかもしれませんが、モダンなCSSでは当たり前のように入れるものなので、AIも自然とそうするんですね。

さあ、ここまで上から順に読んできました。どうですか。AIが書いたCSSの中に、パート3で学んだことが全部詰まっていますよね。セレクタ、色、フォント、ボックスモデル、フレックスボックス、メディアクエリ。数週間前なら、このCSSはただの記号の羅列に見えたはずです。でも今は、一つ一つの行が何をしているのか、理解できるようになっている。これ、本当にすごいことですよ。

中には見慣れないプロパティが含まれているかもしれません。たとえば角を丸くするプロパティや、要素に影をつけるプロパティ、マウスを乗せたときに色が変わる仕組みなど。これらはパート4で実際にサロンサイトを作るときに出てきますので、今は「こういうこともCSSでできるんだな」くらいの理解で大丈夫です。

大事なのは、知らないプロパティが数行あっても、その周りにある知っているプロパティを手がかりに全体の構造が把握できるということです。全部を100パーセント理解しなくても、「このブロックはヘッダーのスタイル」「このブロックはフレックスボックスで横並びにしている」「ここがレスポンシブの切り替え」と、大まかな地図が読めるようになっている。これがパート3の学びの成果ですよ。

### セクション4: CodePen上で自分で修正してみる（4分）

読めるようになったら、次は自分で直してみましょう。CodePenのCSSペインで、実際にコードを触っていきます。

まず、色を変えてみましょう。CSSの中で background-color や color のプロパティを探してください。ヘッダーの背景色を設定しているところが見つかりましたか。たとえばシャープ2c3e50のような濃い色になっていたとします。この値をシャープ3b82f6に変えてみてください。

プレビューを見てください。ヘッダーの色が、落ち着いた紺色から鮮やかなブルーに変わりましたよね。たった6文字のカラーコードを書き換えるだけで、ページの印象がガラッと変わります。他の色も試してみてください。シャープ ef4444 なら赤系、シャープ 10b981 なら緑系です。講義3-3で Coolors のようなカラーパレットツールを紹介しましたよね。あそこで気に入った配色を見つけて、カラーコードを差し替える。それだけで、自分だけのオリジナルカラーのページになります。

次に、余白を調整してみましょう。padding の値を探して、数字を変えてみてください。たとえば padding が 24px のところを 48px に変えてみましょう。

どうですか。余白がぐっと広がって、ゆったりとした印象になりましたよね。逆に小さくすると、ぎゅっと詰まった感じになる。ボックスモデルを理解しているからこそ、padding を変えたら内側の余白が変わると予測できる。margin を変えたら外側が変わるとわかる。この予測ができることが大事なんです。

フォントサイズも変えてみましょう。font-size の値を見つけて、数字を大きくしたり小さくしたりしてみてください。本文の font-size を 16px から 20px にするとどうなるか。見出しの font-size を変えるとどうなるか。一つ変えるたびにプレビューを確認して、変化を感じてみてください。

line-height も試してみましょう。行と行の間隔を広げたり縮めたりする。1.6 を 2.0 にすると、行間がゆったり広がって読みやすくなりますよね。逆に 1.2 にすると詰まった印象になる。たった一つの数値で、ページ全体の読みやすさが変わるんです。

メディアクエリの部分も調整できますよ。アットマーク media の中の値を変えてみましょう。たとえば max-width: 768px を max-width: 1024px に変えると、タブレットサイズでもモバイル向けのレイアウトに切り替わるようになりますよね。レスポンシブの切り替えポイントを自分で調整する。前回学んだばかりの知識が、もうこうして実践で使えています。

どうですか。色を変えて、余白を広げて、フォントサイズを調整して、レスポンシブの切り替え点も変えた。自分の手でCSSの値を触って、すぐに結果が目に見える。これがCodePenで作業する醍醐味ですよね。

こうやって、AIが作ったCSSの中の値を一つ一つ触って調整していく。これがAI時代のウェブ制作のスタイルなんです。ゼロからCSSを全部書く必要はない。でも、どのプロパティをどう変えれば見た目がどう変わるか。パート3で学んだこの知識があるかないかで、結果がまったく違ってきます。知識がなければ、AIが出したものをそのまま受け入れるしかない。知識があれば、自分好みにカスタマイズできる。その差は、とても大きいですよ。

### セクション5: AIに効果的な修正指示を出す（4分）

さて、ここまでは自分でCSSの値を直接触って微調整する方法を練習しました。色を変える、余白を広げる、フォントサイズを調整する、レスポンシブの切り替え点を変える。これはとても大事なスキルで、ちょっとした調整ならこの方法が一番速いです。

でも、もっと大きな変更をしたい場面もありますよね。たとえば全体の配色をごっそり変えたいとか、ナビゲーションのレイアウトを横並びから縦並びに変えたいとか、新しいセクションにスタイルを追加したいとか。そういう大きな変更を一つ一つ手作業で直すのは大変です。そんなときは、AIにもう一度お願いした方が効率的ですよね。

ただし、ここで非常に大事なポイントがあります。AIへの指示の出し方で、返ってくる結果の質がまったく変わるんです。二つの例を比べてみましょう。

【AIデモ: Claudeに修正リクエスト】

まず、良くない例です。Claudeにこう書いたとします。

「もっときれいにして」

これだと、AIは何をどう変えればいいかわかりません。「きれい」って、色の話でしょうか。余白の話でしょうか。フォントでしょうか。レイアウトでしょうか。基準がまったく示されていないので、AIは自分なりに推測して何かを変えるしかありません。もしかしたら皆さんが気に入っていた部分まで変えてしまうかもしれない。結果が期待通りかどうかは、完全に運任せです。

では、良い例を見てみましょう。同じ「修正してほしい」という目的でも、こう書くと結果がまったく違います。

→ 「ヘッダーの背景色をシャープ 2563EB に変えて、本文の font-size を 18px にして、セクション間の margin-bottom を 40px に広げてください」

→ 送信してみましょう。

AIが修正版のCSSを書いている間に、二つの指示の違いを整理しましょう。

良い例の方を見てください。3つのことが具体的に指定されていますよね。一つ目、ヘッダーの背景色をこの色に変える。「ヘッダー」がどの要素か、「背景色」がどのプロパティか、そしてカラーコードという値。セレクタ、プロパティ、値。この3点セットが全部入っています。二つ目、本文のフォントサイズを18ピクセルに。三つ目、セクション間のマージンを40ピクセルに。どの要素の、どのプロパティを、いくつに変えるか。全部、具体的です。

そしてよく見てください。この指示に使われている言葉、背景色、font-size、margin-bottom、ピクセル。全部、パート3で学んだCSSの言葉ですよね。もし皆さんがパート3を受けていなかったら、こんな指示は書けません。「背景を変えて」とは言えても、「background-color をシャープ何々に」とは書けないわけです。CSSの知識があるから、こういう正確な指示が書けるんです。

出てきましたね。修正されたCSSが返ってきました。コードをコピーして、CodePenのCSSペインに貼り付けて確認してみましょう。

どうですか。ヘッダーの背景色がブルーに変わっていますね。本文のフォントサイズも少し大きくなっている。セクション間の余白も広がっています。3つとも、指示した通りの結果です。

「もっときれいにして」と「ヘッダーの背景色をシャープ 2563EB に変えて」。やっていることは同じ、「AIにCSSの修正を頼む」です。でも結果の精度がまったく違う。この差が生まれるのは、CSSを知っているかどうかの差です。

たとえば皆さんが美容室のサイトを作っていて、「ヘッダーの色がもう少し落ち着いた感じがいいな」と思ったとしましょう。CSSを知らなければ「もっと落ち着いた感じにして」としか言えない。でもCSSを知っていれば、「ヘッダーの background-color をシャープ 1a1a2e に変えて、font-size を 14px に下げて」と伝えられる。結果は一発で思い通りです。

これが今日一番お伝えしたかったメッセージです。CSSを知っている人は、AIに正確な指示ができる。プロパティの名前を知っている。値の意味がわかる。セレクタの仕組みを理解している。この知識があるからこそ、AIに対して「ここをこうして」と的確に伝えられるんです。暗記する必要はありません。でも、何を指定すれば何が変わるか。その対応関係を知っていること。パート3で皆さんが学んできたことは、まさにこのためにあったんです。

## まとめ（2分）

今日のポイントを振り返りましょう。

一つ目。AIにCSSを任せれば、数十秒でページの見た目を整えてくれます。でも大事なのは、AIが書いたCSSを読んで理解できること。セレクタ、プロパティ、値。パート3で学んだ知識で、AIの出力をしっかり読めるようになりましたよね。何が書いてあるかわかるから、安心してAIに任せられるんです。

二つ目。AIが書いたCSSをベースにして、CodePen上で自分で値を変えて微調整できる力。色を変える、余白を広げる、フォントサイズを調整する、レスポンシブの切り替え点を変える。値を触れば結果がすぐに見える。この手を動かして調整する力は、パート3の学びそのものです。

三つ目。AIへの修正指示は、具体的であるほど良い結果が返ってきます。「もっときれいに」ではなく、「ここの background-color をこの値にして、ここの margin を何ピクセルに」。CSSの用語を使って指示できる人は、思い通りの結果を得られる。CSSを知っている人は、AIに正確な指示ができる。これが今日の核心メッセージでした。

さて、今日でパート3のすべての講義が終わりました。パート3では、CSSの仕組みから始まって、セレクタ、色とフォント、ボックスモデル、フレックスボックス、レスポンシブデザイン、そして今日のAI実践まで、7つの講義でCSSの基礎を学んできました。パート2で身につけたHTMLと合わせて、皆さんはもうウェブサイトの骨組みと見た目、両方を読んで理解できる力がついています。

次はパート4です。いよいよ本格的なウェブサイトを一つ完成させます。マッサージサロンのサイトを、AIと一緒にゼロから作り上げていきます。ヘッダー、メニュー表、スタッフ紹介、アクセス情報、お問い合わせフォーム。パート2で学んだHTML、パート3で学んだCSS、そして今日練習したAIへの指示の出し方。ここまで学んできた全てを使って、実際にインターネットに公開できるサイトを完成させますよ。楽しみにしていてください。

お疲れさまでした。
