## 導入（1.5分）

皆さん、前回の講義ではAIが生成したサロンサイトのコードを読み込んで、box-shadowやborder-radius、hover効果、transitionといった新しいCSSテクニックを確認しましたね。コードの中に、パート2やパート3で学んできた要素がたくさん使われていることも確認できたと思います。

では、今のCodePenのプレビューを改めて見てみましょう。どうでしょうか。お店の名前が「スカイサロン」とか「ビューティーサロン」のような仮の名前になっていて、キャッチコピーも「最高のリラクゼーション体験を」みたいな、少し漠然とした表現のままですよね。写真もAIが選んだサンプル画像で、色もAIが初期設定として選んだもの。自分のお店のイメージと少しズレている部分があるかもしれません。

今日は、この「AIが作ったサイト」を「自分のサイト」に変えていきます。テキストを実際のお店の情報に書き換えて、写真も雰囲気に合うものに変えて、色もフォントも自分好みに整えて。最後にはAIに追加の修正を依頼して、機能もさらに充実させます。作業の一つひとつはシンプルで、コードを一から書くわけではありません。でも、全部終わったときの「これは自分が仕上げたサイトだ」という感覚は格別ですよ。

今日やる内容は7ステップです。テキスト変更、画像変更、色の変更、フォントの変更、間隔・サイズの微調整、料金表の修正、そしてAIへの追加依頼。一つひとつ、一緒に進めていきましょう。では、さっそくはじめましょう。

## 本編

### セクション1: テキストを自分の情報に変える（6分）

最初にやるのは、テキストの書き換えです。コードの仕組みをゼロから理解していなくても、「この文字をこの文字に変える」という作業なので、誰でもできます。でも、丁寧にやるかどうかで、サイトのクオリティが大きく変わります。

CodePenのHTMLペインを開いて、上から順番に見ていきましょう。

最初に探すのは、サイトのタイトル、つまりお店の名前です。`<h1>`タグの中に入っているはずです。

```html
<h1>マッサージサロン スカイ</h1>
```

これを、自分の実際のお店の名前に変えてください。まだお店を持っていない方は、将来開きたいサロンの名前を考えてみてもいいですし、自分の名前を使った「〇〇 マッサージ」のような仮の名前でも構いません。大切なのは、「自分が作ったサイトだ」という実感を持ちながら作業を進めることです。

次に、ヒーローセクションのキャッチコピーを変えましょう。`<p>`タグや`<h2>`タグで、「全身を芯からほぐします」とか「あなたの疲れ、全て引き受けます」みたいな文章が入っているはずです。ここは、サイトを見た人が最初に目にする言葉です。自分のサロンのコンセプトやこだわりに合わせて書き換えてみましょう。

難しく考えなくていいです。「駅から徒歩3分の、隠れ家のような小さなサロンです」でも、「経験豊富な女性施術師が、丁寧にほぐします」でも。AIが書いたきれいな文章より、自分の言葉で書いた一文の方が、見た人に伝わることがあります。

続いて、こだわり紹介のセクションです。「当店の3つのこだわり」のような見出しと、それぞれの説明文が並んでいますよね。ここも、自分のサロンの強みや特徴に書き換えてください。「完全個室で周りを気にせずくつろげます」「天然アロマオイルを使用しています」「女性専用サロンです」など、お客さんに一番伝えたいことを書きましょう。

次は、施術メニューと料金表です。ここはパート2の2-6で学んだテーブルで作られているはずです。

```html
<table>
  <tr>
    <th>コース名</th>
    <th>時間</th>
    <th>料金</th>
  </tr>
  <tr>
    <td>全身もみほぐし</td>
    <td>60分</td>
    <td>6,000円</td>
  </tr>
  <tr>
    <td>肩・首集中ケア</td>
    <td>30分</td>
    <td>3,500円</td>
  </tr>
  <tr>
    <td>フットリフレクソロジー</td>
    <td>45分</td>
    <td>4,500円</td>
  </tr>
</table>
```

`<td>`タグの中の文字を、自分のメニュー名と料金に書き換えましょう。行を追加したい場合は、`<tr>`から`</tr>`までの1行分をコピーして、その下に貼り付ければ行が増やせます。逆に、不要な行があれば`<tr>` から `</tr>`ごと削除すればOKです。パート2でテーブルを学んだとき、「PART 4のサロンサイトで実際に使います」とお伝えしましたが、まさにここで使っていますね。

スタッフ紹介のセクションも書き換えましょう。スタッフの名前、施術歴や資格、ひとことメッセージ。ここは、お客さんが「このサロン、行ってみたいな」と思うかどうかを左右する大切な場所です。自分らしい言葉で、丁寧に書いてみてください。

アクセス情報も忘れずに確認しましょう。住所、最寄り駅からの道順、営業時間、定休日、電話番号やメールアドレス。これらの情報が間違っていたり、仮のままになっていたりすると、本当に来てほしいお客さんに情報が届きません。仮のサロンを作っている場合は、近所の架空の住所でも構いませんが、形式はしっかり整えておきましょう。

テキストを一通り書き換えたら、プレビューで確認してみましょう。さっきまでと比べて、一気にリアルなサイトに見えてきましたよね。「AIが適当に入れた文章」から「自分が考えた言葉」に変わるだけで、こんなに印象が変わるんです。

ここで一つ、文章を書くときのヒントをお伝えします。お客さんはサイトを見るとき、「このサロン、自分に合っているかな」「安心して行けそうかな」ということを感じ取ろうとしています。だから、「最高の技術でほぐします」のような大げさな言葉より、「産後ケアを始めて8年、のべ2,000名以上のお客様をサポートしてきました」のような具体的な事実の方が、信頼感につながります。難しく考えなくていいので、「自分のお店のことを、友達に話すとしたらどう話すか」をそのまま書いてみてください。それが一番伝わる文章になります。

### セクション2: 画像を変える（5分）

次は画像の変更です。テキストの次にサイトの印象を大きく決めるのが画像です。特に最初に目に入るヒーローセクションの大きな画像は、そのサイトの世界観を決める最も重要な一枚です。

今サイトに入っている画像はAIが選んだサンプルのURLですね。まず新しい画像を探しに行きましょう。無料で商用利用できる写真素材サイト、UnsplashとPexelsを使います。パート2の2-3でも紹介しましたね。

Chromeで新しいタブを開いて、Unsplashにアクセスしてみましょう。

検索ボックスに「massage」「spa」「relaxation」「wellness salon」などのキーワードを入力して検索してください。英語で検索した方が候補が多く出てきます。

さまざまな写真が出てきましたね。サロンの内装写真、施術の様子の写真、アロマキャンドルの写真など。自分のサロンのコンセプトに合いそうなものを選んでいきましょう。

気に入った画像が見つかったら、その写真をクリックして拡大します。右上に「ダウンロード」ボタンがありますが、今回はダウンロードせずに画像のURLをそのまま使います。

拡大した画像の上で右クリックして、「画像アドレスをコピー」を選んでください。ここでMacの方は「画像アドレスをコピー」、Windowsの方も「画像のアドレスをコピー」という項目をクリックしてください。これで画像のURLがクリップボードに入りました。

CodePenのHTMLペインに戻って、画像が入っている場所を探します。`<img>`タグのsrc属性の中に、今の画像URLが書いてあるはずです。そのURL全体を削除して、コピーしたURLをペーストしてください。

```html
<img src="https://images.unsplash.com/photo-xxxxxxxxx..." alt="マッサージの施術の様子">
```

このとき、alt属性も必ず更新しておきましょう。パート2で学んだように、altには画像の内容を説明するテキストを書きます。「サロンのリラックスした内装」「施術中の様子」「アロマオイルとキャンドル」など、実際の画像に合った説明を入れてください。

プレビューで確認してみましょう。新しい画像に変わりましたね。サイズが合っていない場合は、CSSのwidthやheightを調整すれば大丈夫です。

ヒーローセクションの画像以外にも、こだわり紹介セクションやスタッフ紹介セクションに画像が使われていれば、同じ手順で変えていきましょう。Pexelsも併用してみてください。UnsplashとPexelsはそれぞれ雰囲気の違う写真が揃っているので、両方から探すと選択肢が広がります。

画像を変えると、サイトの雰囲気がテキスト変更のときとはまた違った形でガラッと変わります。「これ、本当に自分のサイトっぽくなってきた」と感じてもらえていたら嬉しいです。

画像選びで迷ったときのコツをお伝えします。まず、「どんな客層に来てほしいか」を考えてみてください。ゆったりしたシニア向けのサロンであれば、落ち着いた照明の内装写真が合います。若い女性向けなら、白を基調にしたおしゃれな空間の写真が映えます。スポーツ系のコースが売りなら、施術の様子そのものを前面に出した写真がいいですね。また、複数のセクションで画像を使っている場合は、色味のトーンをそろえることで、サイト全体が統一感のある見た目になります。全部暖色系にする、あるいは全部モノクロに近いトーンでそろえるというやり方が効果的です。

### セクション3: 色を変える（8分）

次は色の変更です。テキストと画像を変えただけでも随分印象が変わりましたが、色を変えることで、サイト全体の「雰囲気」そのものを変えることができます。たとえば同じサロンでも、深緑とベージュの組み合わせなら落ち着いた和の雰囲気に、チャコールグレーとゴールドならモダンで高級感のある雰囲気に、ピンクとラベンダーならかわいらしい女性向けの雰囲気になります。色はブランドイメージを作る、とても強力な要素です。

まず、どんな配色にするかを決めましょう。何も考えずにCSSを開いても、「どの色をどれに変えたらいいんだろう」と迷ってしまいます。先に「こういう配色にしたい」というイメージを固めてから作業に入る方がスムーズです。

そのために便利なツールが「Coolors」です。Chromeで新しいタブを開いて、CoolorsにアクセスしてみてCoolorsは、5色の組み合わせを自動で提案してくれるツールです。スペースキーを押すたびに、全く新しい配色が出てきます。「この組み合わせはいいな」と思ったら、固定したい色のロックアイコンをクリックして、その色を固定した状態でまたスペースキーを押すと、固定した色に合う他の色を提案してくれます。

何種類か試してみましょう。「スペースキーを押して...また押して...この組み合わせが素敵ですね」というように、実際にいくつかの配色を試してみてください。

サロンサイトに合う配色のヒントをお伝えしますね。リラックス系のセラピーサロンなら、ベージュ・温かみのあるブラウン・淡いグリーンなどが合います。スポーツマッサージやリフレクソロジーならネイビー・グレー・ホワイトのクールな組み合わせが効果的です。女性向けの美容サロンなら、ローズピンク・クリームホワイト・淡いゴールドといった組み合わせが素敵ですよ。

気に入った配色が見つかったら、使いたい色のHEXコードをメモしておきましょう。HEXコードというのは「#」から始まる6文字の色番号です。たとえば「#4A7C6F」といった形式ですね。パート3の3-3で学んだ色の表現方法です。Coolorsの各色の下にこのコードが表示されています。

では、CodePenに戻って、CSSペインで色を変えていきましょう。

まず、今のCSSで使われているカラーコードの場所を把握します。CSSペインの中をクリックしてから「Ctrl+F」、Macは「Command+F」で検索ができます。「#」を検索すると、カラーコードが書かれている場所がハイライトされて、数がわかります。

次に、変えたい色を見つけて書き換えます。たとえば、ナビゲーションバーの背景色がこうなっていたとします。

```css
nav {
  background-color: #2563EB;
}
```

これを、Coolorsでメモしたメインカラーに変えます。

```css
nav {
  background-color: #4A7C6F;
}
```

同じ色コードが複数の場所で使われている場合、CodePenでは検索と置換が使えます。「Ctrl+H」、Macは「Command+H」を押すと検索・置換のパネルが開きます。「検索」の欄に変えたい色コード、「置換」の欄に新しい色コードを入れて、「全て置換」を押せば一括で変わります。

ここで、「CSS変数」という考え方も少し紹介しておきます。CSSの一番上に、こういうブロックを最初から書いておく方法があります。

```css
:root {
  --color-primary: #4A7C6F;
  --color-accent: #E8D5B7;
  --color-text: #333333;
}
```

こうすると、CSSの中でこの変数名を使えます。

```css
nav {
  background-color: var(--color-primary);
}

.btn {
  background-color: var(--color-accent);
}
```

この変数の値を1か所変えるだけで、その変数を使っているすべての場所の色が一括で変わります。今回は既にあるコードを修正するので検索・置換の方がシンプルですが、次に新しいサイトをゼロから作るときはこのCSS変数を最初から使うと、後で色を変えたいときに便利ですよ。

色の変更が終わったら、全体をプレビューで確認しましょう。ナビゲーションバー、ボタン、見出し、フッター。全体を見渡して「ここの色がちょっと合わないな」と感じるところがあれば、また少し調整してみてください。色合わせには正解がありませんので、自分の感覚を信じてやってみましょう。一度変えてみて気に入らなければ戻せばいいだけなので、どんどん試してみてください。

一つ、よくあるミスをお伝えします。Coolorsで5色の配色を選んでも、それを全部サイトに使おうとすると、かえってごちゃごちゃした印象になりがちです。実際のプロのデザインでは、「メインカラー1色」「サブカラー1色」「アクセントカラー1色」の3色、多くても4色程度に抑えるのが基本です。Coolorsで選んだ5色の中から、「この3色を使おう」と絞り込むところから始めてみましょう。メインカラーはナビとフッターの背景に、サブカラーはカードや区切りのラインに、アクセントカラーは予約ボタンや強調したい箇所に使う、というように役割を決めると整理しやすいですよ。

### セクション4: フォントを変える（4分）

色の次は、フォントを変えてみましょう。フォントが変わるだけで、サイトの雰囲気がガラッと変わることがあります。

今、CodePenのCSSペインで使われているフォントを確認してみましょう。`font-family`というプロパティで指定されています。AIが生成したコードには、英語フォントだけが指定されていて、日本語フォントがきちんと設定されていないことも多いです。日本語フォントを明示的に指定しておくことで、文字がより読みやすく、より洗練された見た目になります。

Google Fontsを使って、素敵な日本語フォントを適用してみましょう。パート3の3-3でもやりましたね。

Chromeで新しいタブを開いて、Google Fontsにアクセスします。検索ボックスで「Japanese」と入力して絞り込むと、日本語対応フォントだけが表示されます。

いくつか紹介しますね。「Noto Sans JP」は、Googleが開発したフォントで、シンプルで読みやすく、どんなジャンルのサロンにも合いやすい万能フォントです。「M PLUS Rounded 1c」は角が丸くてやわらかみのある雰囲気で、かわいらしい女性向けのサロンにぴったりです。「Zen Kaku Gothic New」はすっきりとしたモダンな雰囲気で、スタイリッシュなサロンや男性向けのサロンによく合います。

フォントをクリックすると実際の見た目が確認できます。気に入ったフォントが見つかったら、「Get font」ボタンをクリックして、次の画面で「Get embed code」を選びます。「@import」のタブをクリックして、表示されたコードをコピーしてください。

```css
@import url('https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;700&display=swap');
```

このコードをCodePenのCSSペインの一番先頭、つまり他のどのCSSより前に貼り付けます。

次に、`body`のフォント指定を変えます。CSSペインの中で`font-family`と書かれている場所を探して、選んだフォント名に変えましょう。

```css
body {
  font-family: 'Noto Sans JP', sans-serif;
}
```

最後の`sans-serif`は、Google Fontsが何らかの理由で読み込めなかったときの代替フォントです。これはそのままにしておいてください。

プレビューを確認しましょう。フォントが変わりましたね。同じ文章なのに、フォントが変わるだけで雰囲気が全然違いますよね。色の変更と合わせて、サイトの世界観が統一されてきた感じがすると思います。

フォントに関して一つ補足です。サイト全体で2種類以上のフォントを使うときは、「見出し用」と「本文用」に分けると読みやすいサイトになります。たとえば、見出しにデザイン性の高いフォントを使って、本文にはシンプルで読みやすいNoto Sans JPを使うという組み合わせです。ただ、フォントを多用しすぎるとまとまりがなくなりますので、多くても2種類に絞るのがおすすめです。迷ったら、Noto Sans JP一種類だけでもとてもきれいに仕上がりますよ。

### セクション5: 間隔・サイズの微調整（5分）

ここまで、テキスト、画像、色、フォントと順番に変えてきました。もう随分「自分のサイト」らしくなってきましたよね。ここからは少し細かい調整をしていきましょう。

間隔とサイズの微調整です。パート3の3-4で学んだボックスモデル、padding、marginを使います。

まず、全体のプレビューを見渡してみましょう。「ここ、もう少し余白があった方がゆったりした感じになるな」とか「この文字、もう少し大きくした方が読みやすいな」とか「ボタンが少し小さくてクリックしにくそうだな」と感じるところはありますか。

そういった部分を、CSSで少しずつ調整していきましょう。

たとえば、セクションとセクションの間が詰まりすぎていると感じたら、セクション全体の上下の余白を増やします。

```css
section {
  padding: 80px 0;
}
```

ヒーローセクションのキャッチコピーをもっと大きく目立たせたい場合は、`h1`の`font-size`を変えます。

```css
.hero h1 {
  font-size: 3rem;
}
```

レムという単位はパート3で学びましたね。`1rem`は基本のフォントサイズ、つまり大体16ピクセルが基準です。`3rem`なら48ピクセル相当です。数字を変えながらプレビューで確認して、ちょうどいい大きさを見つけてみましょう。

ボタンの中の余白が窮屈に感じたら、paddingを増やします。

```css
.btn {
  padding: 14px 32px;
}
```

上下の14px、左右の32pxというのは、一般的なボタンのバランスです。縦を増やすとボタンが高くなり、横を増やすとボタンが横に広がります。

カードとカードの間の隙間を広げたい場合はgapを使います。Flexboxで並んでいる要素のgapを変えると、アイテム間の間隔が変わります。これもパート3のFlexboxで学んだプロパティですね。

```css
.cards {
  gap: 32px;
}
```

調整するときのコツをお伝えしますね。「一度大きく変えてから、そこから微調整する」というやり方がおすすめです。たとえばfont-sizeを変えるとき、「大きくしたい」と思ったら思い切って倍くらいにしてみる。大きすぎたら少し戻す。このやり取りを繰り返しながら、ちょうどいいバランスを見つけていきます。

CodePenはリアルタイムでプレビューが更新されますから、数字を変えるたびにすぐ結果が確認できます。「変えてみる、確認する、また変える」このサイクルを繰り返しながら、どんどん試してみてください。ためらわずに変えてみることが、クオリティを上げる一番の近道です。

前回学んだbox-shadowの強さや、border-radiusの丸みの加減もこの段階で調整するといいですよ。カードの影を少し濃くしたり、ボタンの角丸をもっと丸くしたり。そういった細かいところへのこだわりが、サイト全体のクオリティを一段上げてくれます。

一つ参考になる考え方をお伝えします。プロのデザイナーやWeb制作者も、最初から完璧なレイアウトを作れるわけではありません。まず大まかに配置して、見ながら少しずつ調整する、この繰り返しがデザインの基本的な進め方です。今皆さんがやっていることは、まさにそのプロと同じ作業です。「ちょっと窮屈かな」と感じる感覚、「ここの余白が足りないな」と気づく目。それ自体がWebデザインのセンスが育ってきている証拠ですから、自信を持って進めてください。

### セクション6: AIに追加修正を依頼する（4分）

ここまでの作業で、サイトはかなり「自分のもの」になってきましたよね。最後に、AIを活用してもう少し機能を追加してみましょう。

この講座でずっとお伝えしてきた「AIへの指示の出し方」を思い出してください。漠然とした指示より、具体的な指示の方が良い結果が返ってきます。悪い例は「もっとカッコよくして」。良い例は「このボタンのCSSを変えて、背景色をオレンジ系の暖色にして、ホバー時には少し大きくなるアニメーションをつけてください」のように、何を、どうしたいかを具体的に伝えることです。

まず、予約ボタンをもっと目立たせてみましょう。サロンサイトで一番大切な行動、つまりお客さんに「予約したい」と思ってもらうこと。そのためにも、予約ボタンはページ内で最も目立つ要素にしたいですよね。

ClaudeのタブをChromeで開いて、こんな風に依頼してみましょう。現在のCSSをコピーして貼り付けて、こう続けます。

「このCSSの予約ボタン（.btn-reserveクラス）を改善してください。背景色はオレンジ系の暖色にして、テキストは白、ホバーしたときに少しだけ大きくなるscaleのアニメーションをつけてください。」

Claudeが改善案を返してくれたら、CodePenのCSSペインに貼り付けて確認しましょう。気に入ったらそのまま使う。もう少し調整が必要なら「色をもう少し落ち着かせて」「文字を太くして」と追加で指示する。AIの提案を一度受け取って、自分の判断で採用するかどうかを決める。これが「ディレクター」としての仕事です。

次に、Googleマップの埋め込みを追加してみましょう。アクセスのセクションに地図が表示されていると、初めて来るお客さんに場所が伝わりやすくなります。Claudeにこう頼んでみましょう。

「このHTMLのアクセスセクションに、Googleマップの埋め込みを追加してください。パート2で学んだiframeタグを使った方法でお願いします。住所は東京都渋谷区〇〇（仮の住所）です。」

Claudeが返してくれたiframeのコードをHTMLに貼り付けると、アクセスセクションに地図が表示されます。実際のGoogleマップと連動した地図が、コードを数行貼るだけで追加できてしまうんです。すごいですよね。

こういった「既存のサイトに機能を追加する」という使い方も、AIの得意分野です。「このサイトに〇〇を追加して」という形で依頼すると、既存のコードのスタイルや構造を崩さずに、自然に機能を追加してくれます。「自分でゼロから書く」のではなく、「AIに追加してもらって、自分で確認して採用する」。この流れが身についてきましたよね。

ここでもう一つ、AIへの依頼でよく使えるパターンをご紹介します。「このCSSの特定の部分だけ変えて」というピンポイントの修正依頼です。たとえば「ナビゲーションバーの高さを60pxから80pxに変えて」とか「h2の色を現在の色から少しだけ明るくして」というように、変えたい箇所と変えたい内容を具体的に伝えます。こうすると、AIは他の部分には手を触れずにその部分だけ修正してくれます。逆に「全体をもっとおしゃれにして」というような曖昧な依頼は、AIが何を変えていいかわからず、大幅な変更を加えてしまうことがあります。AIへの指示は「どこを、どう変えるか」を具体的に。これが一番効率よく、意図した修正を得るコツです。
