## 導入（2分）

前回の講義では、CodePenにログインして、HTML、CSS、JavaScriptの3つの場所とプレビューの関係を見ました。ブラウザだけで始められて、書いた結果がすぐ見える。だから初心者に向いている、という話でしたね。

今日は、そのCodePenを使って、いよいよ「最初のWebページ」を自分の手で表示してみます。ここから、本当に作る感覚が出てきます。

【スライド: 今日のゴール】

今日のゴールは、とてもシンプルです。CodePenで新しいペンを作って、HTMLに文字を入力し、その文字がブラウザに表示されるところまでやります。そして、Webページの基本構造としてよく出てくる `<!DOCTYPE html>`、`<html>`、`<head>`、`<body>` という言葉にも触れます。ただし、最初から全部を完璧に覚える必要はありません。今日は「なるほど、こういう形になっているんだな」とつかめれば十分です。

ここで大事なのは、難しいことを学ぶことではなく、「自分で表示できた」という体験をすることです。最初の成功体験は、とても大きいです。だから今日は、丁寧に、小さく、一歩ずつ進めていきましょう。

## 本編

### セクション1：新しいペンを作って、最初の文字を表示しよう

【スライド: 新しいペンを作る】

では、CodePenを開きましょう。前回ログインした状態から始めます。

[ライブ開始]

まず、新しいペンを作成します。`Create` や `Start Coding` といったボタンから、新規のペンを開きます。画面にHTML、CSS、JSの3つのエリアと、プレビューが出てきましたね。ここまで来たら、もう作る準備はできています。

この時点で、少し手が止まりそうになる方もいると思います。「ここから何を書けばいいんだろう」「空白の画面を見ると緊張するな」と感じるのは、とても自然です。でも安心してください。今日は、その空白を埋めるために、いちばん簡単な一行から始めます。

最初にやることは、本当にシンプルです。HTMLの欄に、たった一行、文字を入れてみましょう。

たとえば、こうです。

```html
Hello World
```

入力してみると、プレビュー側にその文字が表示されます。これが、皆さんの最初のWebページです。

「え、これだけですか？」と思うかもしれません。でも、そうです。最初はこれでいいんです。大事なのは、HTMLの欄に書いた文字が、ブラウザ側に表示された、ということです。この瞬間に、皆さんはもう「Webページを表示させた人」になっています。

ここでは、まだタグも使っていません。ただ文字を書いただけです。でも、それでも表示される。まずはこのシンプルさを知ることが大切です。ホームページ制作というと、最初から難しい記号だらけの世界だと思われがちですが、実際には文字を表示するところから始められるんですね。

【スライド: 最初の一歩はとても小さくていい】

初心者の方ほど、「もっとちゃんとしたものを作らないといけない」と思いがちです。でも、最初の一歩は小さくて大丈夫です。むしろ、小さいほうがいいです。

なぜなら、今は「何を書けばどう表示されるのか」という関係を体で覚える段階だからです。いきなり難しいレイアウトや長いコードに行くより、一行の文字が出るところをしっかり確認するほうが大事なんですね。

たとえば、紙に初めてペンで名前を書くとき、いきなりきれいなポスターを作ろうとはしませんよね。まずは一文字ずつ書いてみる。その積み重ねです。ホームページ制作もまったく同じです。

最初から立派なトップページを目指さなくて大丈夫です。むしろ、今は「書いたら出る」という確信を持つことのほうが重要です。この確信があると、次の講義でタグが出てきても、落ち着いて見られるようになります。

では、この `Hello World` を少し変えてみましょう。

```html
こんにちは、世界！
```

すると、表示もすぐ変わりますね。ここで味わってほしいのは、「書いた瞬間に変わる」という感覚です。この変化がすぐ見えるからこそ、CodePenは学びやすいんです。

この「書いたら変わる」が見えると、コードが急に身近になります。ただの難しい記号ではなく、「画面を変えるための指示」なんだと感じられるようになるんですね。最初は理屈より、この感覚をつかむことが大切です。

### セクション2：Hello Worldはなぜよく使われるのか

【スライド: Hello Worldとは？】

ここで、少しだけ「Hello World」という言葉の意味にも触れておきましょう。

プログラミングやWeb制作の入門では、最初に `Hello World` と表示することがとてもよくあります。直訳すると、「こんにちは、世界」です。少し不思議ですよね。なぜこれなのか。

理由は単純で、「ちゃんと表示できた」ということを一番シンプルに確認しやすいからです。難しい意味はありません。昔から、最初の動作確認として使われてきた定番なんです。

つまり、Hello Worldは、かっこいいから使うわけでも、特別な意味があるから使うわけでもありません。「まずは表示できることを確認しよう」という、とても実用的なスタートなんです。だからこそ、入門にずっと使われ続けています。

ですから、ここで皆さんが `Hello World` を表示したというのは、単に文字が出たというだけではありません。「自分の環境で、ちゃんとWebページが表示できる」という最初の確認ができた、ということでもあります。

この確認が取れると、もし後で何かがうまく表示されない場面が出てきても、「少なくとも自分の環境では動くことは確認できている」と思えます。これは意外と大きな安心材料になります。

【スライド: 表示できた = もう入口は越えている】

ここはとても大事なので、少し強調しておきます。初心者にとって大きいのは、「まだ何もできていない」と感じてしまうことです。でも、今の時点で、皆さんはもう何もできていない状態ではありません。

CodePenを開ける。HTML欄に文字を入力できる。プレビューに表示できる。文字を変えると表示も変わる。これは立派なスタートです。

上級者から見ると小さく見えるかもしれません。でも、学び始めの人にとっては、この最初の成功体験がその後を大きく左右します。だから今日は、この一歩を軽く見ないでください。とても大事な一歩です。

実際、ここで「自分でもできた」と思えると、その後の講義の受け取り方が変わります。逆に、「まだ自分には難しい」と感じたままだと、次に進みにくくなります。だから、今日の講義では、この小さな成功をしっかり手に入れることが重要なんです。

### セクション3：HTMLの基本構造を「おまじない」として見てみる

【スライド: HTMLの基本構造】

さて、ここから少しだけ、HTMLの基本構造を見てみましょう。

普段、WebページのHTMLは、実はもっと大きな形をしています。たとえば、基本形はこんな感じです。

```html
<!DOCTYPE html>
<html>
  <head>
    <title>はじめてのページ</title>
  </head>
  <body>
    こんにちは、世界！
  </body>
</html>
```

ここで初めて見る記号や単語がたくさんあると思います。でも、今日は深く理解しなくて大丈夫です。まずは「HTMLのページには、こういう外枠があるんだな」と見てもらえれば十分です。

今日は4つの名前だけ、軽く知っておきましょう。`<!DOCTYPE html>` は「これはHTMLですよ」と伝える宣言、`<html>` は全体を包む外枠、`<head>` はあとで詳しく扱う設定まわり、`<body>` は実際に表示される中身に近い場所です。

ここで大切なのは、全部を覚えることではありません。「今見ているページの裏側には、こういう基本の並びがあるんだな」と知ることです。AIが出したコードやサンプルコードを見たときに、「うわ、知らないものが並んでいる」と慌てにくくなるからです。

そしてもう一つ、今日の段階では細かい役割分担まで追いかけなくて大丈夫です。どれが何文字で、どの順番で、と暗記する必要もありません。今は、ページには外側の枠があり、その中に私たちが表示したい内容が入っていく、という感覚だけつかめれば十分です。

【スライド: 今はbodyが一番大事】

この講座の今の段階で、一番大事なのは `body` まわりの感覚です。なぜなら、皆さんがまず作りたいのは、画面に見える中身だからです。

もちろん、後で `head` の中身も大切になります。でも、それは次の講義でもう少し整理します。今日は、「今の自分がまず触るのは、表示される中身のほうなんだな」とわかっていれば十分です。

学習では、この「今の段階で何に集中するか」をはっきりさせることが大切です。全部を一度に覚えようとすると、かえって頭に残りません。ですから今日は、まずbodyに集中しましょう。見えるものを作る。それが今日のテーマです。

### セクション4：CodePenではなぜbodyの中だけでよいのか

【スライド: CodePenが裏側を補ってくれている】

ここで、CodePenの便利なところを説明します。

さきほど見たHTMLの基本構造、普通なら全部書く必要があります。でも、CodePenではその外側の基本構造をかなり自動で補ってくれています。だから、皆さんは主に `body` の中身だけを書けばよいんです。

つまり、CodePenのHTML欄にこう書いたとします。

```html
<h1>こんにちは、世界！</h1>
<p>これが私の最初のWebページです。</p>
```

すると、裏側ではCodePenが、ページとして必要な外枠をある程度整えてくれているわけです。だから初心者でも始めやすいんですね。

この仕組みを知っておくと、「なぜ外枠を書いていないのに表示されるんだろう」という疑問が減ります。CodePenが学習しやすいように準備してくれている、と理解すれば大丈夫です。

これは、料理教室で最初から調理台や火加減が整っているのと少し似ています。皆さんは、まず材料をどう切るか、どう盛りつけるかに集中できる。CodePenも同じで、最初の複雑さを少し減らしてくれているんです。

だから、「本当は全部自分で書けないとダメなんじゃないか」と不安にならなくて大丈夫です。今は、見える中身を作る楽しさを知る段階です。必要なことは、順番に学んでいけば十分です。

【スライド: だから今はbodyの中身に集中する】

ですから、この講座の最初の段階では、CodePenのHTML欄に「表示したい中身」を書くことに集中しましょう。見出しを書いてみる。文章を書いてみる。あとでリストや画像も入れていく。そうやって少しずつ、bodyの中身を作る感覚を身につけていきます。

あとで本格的な環境に進むと、自分で `<!DOCTYPE html>` や `<head>` も書く場面が出てきます。でも、今はその前段階です。まずは「見える中身を作ること」に集中していきましょう。

この順番には意味があります。最初に楽しさと達成感を知る。そのあとで仕組みを少しずつ理解する。そのほうが、無理なく続けやすいんですね。

### セクション5：画面を見ながら作る習慣を身につけよう

【スライド: 入力欄とプレビューを行き来する】

ここで、もう一つとても大事な習慣をお伝えします。それは、「書いたら、すぐプレビューを見る」という習慣です。

初心者のうちは、どうしても入力欄だけをじっと見続けてしまいがちです。でも、本当に大事なのは、入力した結果がどう見えているかをすぐ確認することなんですね。CodePenは、この確認をとてもやりやすくしてくれます。

たとえば、1行書いたらプレビューを見る。文字を変えたらまた見る。思った表示になっていなければ、すぐ戻って直す。この往復ができるようになると、学習のスピードがかなり上がります。

しかもこの習慣は、あとでAIを使うときにもそのまま役立ちます。AIが長いコードを出したときも、まず貼ってみる。そして、何がどう表示されるかを見る。違和感があれば、また直す。今日のHello Worldは小さいですが、この作り方の原型なんです。

もし思った通りに変わらなくても、慌てなくて大丈夫です。まず「今どこの欄を触っているか」を確認する。HTML欄なのか、CSS欄なのか。次に、プレビューに少し目をやる。この2つを落ち着いて見るだけでも、かなり整理できます。

### セクション6：文字を修正すると結果が変わる魔法

【スライド: テキストを変えてみよう】

では、先ほどの内容を少しずつ変えてみます。

[ライブ開始]

HTML欄を、こんなふうにしてみましょう。

```html
<h1>こんにちは、世界！</h1>
<p>これは私が作った最初のページです。</p>
```

プレビューを見ると、見出しが大きく表示され、その下に文章が出ていますね。

ここで、さっきのただの文字と少し違うことに気づくと思います。今度は `<h1>` や `<p>` という形が付いていますよね。これがタグです。今日は詳しく覚えなくて大丈夫ですが、「ただ文字を書くだけ」と「タグを付けて意味を持たせる」は違う、ということだけ感じておいてください。

では、見出しを変えてみます。

```html
<h1>こんにちは、山田です！</h1>
```

すると、すぐに表示が変わります。今度は、もっと自分に近いページになりましたね。

この「自分の名前に変える」というのも、小さいけれど大切な作業です。サンプルをそのまま眺めるのではなく、自分の内容に置き換える瞬間に、学習はぐっと自分事になります。

ここで、ただの見本だったものが、少し自分のページらしくなりました。こういう変化が積み重なると、ホームページ制作は急に身近になります。最初は借り物の言葉でも、少し変えるだけで自分のものに近づいていくんです。

次に、段落の文章も少し変えてみましょう。

```html
<p>ブラウザだけでホームページ作りを始めました。</p>
```

これもすぐ変わります。

だからこそ、今後の実習でも、まずはサンプルをそのまま写し、そのあとで自分の言葉に変える、という流れをよく使います。最初から全部オリジナルにしなくて大丈夫です。まずは動くものを見て、そのあと自分用に寄せていけばいいんです。

ここで感じてほしいのは、「修正が怖くない」ということです。変えてみて、見て、違ったらまた直せばいい。ホームページ制作は、この繰り返しです。

学校のテストのように、一度書いたら終わりではありません。むしろ、何度も修正できることが前提です。だから、最初から完璧な文を考え込むより、とりあえず入れてみて、見て、直す。この流れに慣れるほうが大切です。

【スライド: 変えて、見て、また直す】

実際の制作では、最初から一発で完璧に決まることはほとんどありません。文言を変える。順番を変える。色を変える。余白を変える。そうやって少しずつ整えていきます。

AIを使うときも、実は同じです。AIが下書きを出してくれたら、それを見て直す。今日やっていることは小さく見えても、AI時代の制作の基本動作そのものなんです。

CodePenのように、変化が即座に見える環境では、この「変えて、見て、また直す」がとてもやりやすいです。だからこそ、初心者が感覚をつかむには最適なんですね。

ここで、前回の講義とのつながりも思い出してみてください。前回は、CodePenがHTML、CSS、JSの3つに分かれていて、プレビューがすぐ見えるという話でした。今日は、その中でもHTMLの役割に集中して、「何を表示するか」を自分の手で決めています。こうやって、一つずつ役割を体で覚えていきます。

そして次の段階では、「ただ表示する」から「意味を持った構造で表示する」へ進んでいきます。今日の講義は、その橋渡しでもあります。

[ライブ終了]

### セクション7：今、皆さんはもうWeb開発者の入口に立っている

【スライド: おめでとうございます】

少し大げさに聞こえるかもしれませんが、ここで一つ言わせてください。おめでとうございます。皆さんは今、Webページを自分で表示させました。

もちろん、まだ大きなサイトではありません。写真もないですし、ボタンもないですし、デザインもまだこれからです。でも、HTMLを書いて、ブラウザに表示して、文字を変えて、結果を確認する。この基本の流れを、もう経験しました。

これはとても大きいです。なぜなら、ホームページ制作は結局、この積み重ねだからです。最初は1行。次は見出しと文章。その次はリスト。さらに画像。さらに色。そうやって、一つずつ積み上がっていきます。

【スライド: 小さな成功体験を積み重ねる】

学習で大事なのは、最初から難しいことを一気にやることではありません。小さな成功体験を積み重ねることです。

今日なら、「Hello Worldが表示できた」「自分の名前に変えられた」「文章を直したら結果も変わった」。こういう小さな成功が、次の講義につながります。

逆に、最初から完璧を目指しすぎると、苦しくなります。だからこの講座では、極めてゆっくり進みます。一つも取りこぼさない感覚で、少しずつ前に進みましょう。

### セクション8：今日の内容がこの先どうつながるか

【スライド: 今日の経験はこの先の土台になる】

今日やったことは、ただのHello Worldではありません。この先のすべての土台です。

今後、見出しを作るときも、文章を書くときも、画像を入れるときも、結局はHTML欄に何かを書いて、それがどう表示されるかを見ていくことになります。今日の体験は、その一番小さい形なんですね。

そして次の講義では、今日名前だけ見た `head` や `body` をもう少し整理しながら、CodePenの便利な機能も使っていきます。今日やったことが「ただ文字を出した」だけではなく、「ページの部品を少しずつ置いている」という感覚に変わっていくはずです。

そして、AIを使うときも同じです。AIがHTMLのたたき台を作ってくれたとしても、皆さんはその中身をCodePenで見て、必要なら直します。つまり、今日の「書く」「見る」「直す」という流れは、AI時代にもそのまま必要なんです。

ですから、今日の講義は単に昔ながらの基礎ではありません。これからAIと一緒にホームページを作っていくための、本当に大事な基礎です。

この意味で、Hello Worldは昔の儀式のようなものではなく、今でも有効な最初の一歩です。AIがどれだけ進化しても、自分で表示を確認し、必要なら直す、この基本動作はなくなりません。

### 講師メモ
- この講義は「最初の成功体験」を最優先にする。高度な説明より、表示できた実感を強く残す
- CodePenのHTML欄ではbodyの中身だけ書けばよいことを繰り返し明言する

- ライブでは `Hello World` → 日本語文 → 自分の名前への変更、の順で変化を見せる
- 自動生成収録では、プレビュー反映が一瞬遅れる可能性があるため、ナレーションは少し間を取れるようにする
- 「皆さんは今Web開発者です」という前向きな一言を、照れずに入れてよい
