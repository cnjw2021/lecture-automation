# 講義05：開発環境セットアップ — CodePen入門

- 所属: PART 1
- 形式: ライブ
- 想定時間: 20分
- 到達目標: CodePenにアクセスし、GitHubアカウントでログインできる。CodePenのHTML / CSS / JS の3ペイン構成とプレビューの見方を理解できる。HTMLとCSSを少し入力し、保存と共有までの基本操作を説明できる。

## 導入（2分）

前回の講義では、AIがWeb制作をどう変えたのかを見てきました。AIは最初のたたき台を速く作ってくれる。でも、最後は人間が判断して直す必要がある。だから私たちは、コーダーではなくディレクターとしてAIを使っていく、という話でしたね。

今日は、その考え方を実際の手の動きに変えていく最初の一歩です。いよいよ、皆さんが自分でホームページを作るための作業環境を用意します。

ここで「環境セットアップ」と聞くと、ちょっと身構えてしまう方もいるかもしれません。「何か難しいソフトをインストールするんですか」「途中でうまくいかなくて止まったらどうしよう」と感じる方もいると思います。でも安心してください。今日使うのは、ブラウザだけで始められるCodePenというサービスです。インストールは不要です。まずは、すぐ作り始められる環境を手に入れる。それが今日のゴールです。

【スライド: 今日やること】

今日は大きく4つ進めます。まず、なぜ最初の環境としてCodePenを使うのかを説明します。次に、GitHubアカウントを作ります。このアカウントは、あとでPART 4のStackBlitzでも使いますので、ここで整えておく意味があります。次に、CodePenにGitHubアカウントでログインします。そして最後に、CodePenの画面構成を一緒に見て、HTMLやCSSを少し入力し、保存と共有まで試してみましょう。

## 本編

### セクション1：なぜ最初はCodePenなのか

【スライド: インストール不要、ブラウザだけで始められる】

まず最初に、なぜこの講座の入口をCodePenにしているのかをお話しします。

ホームページ制作と聞くと、普通は「エディタを入れて、フォルダを作って、ファイルを保存して、ブラウザで開いて……」という流れをイメージするかもしれません。もちろん、最終的にはそういうやり方も大事です。実際、この講座の後半では、より本格的にファイルを扱うStackBlitzも使います。

ただ、初心者の最初の壁は、HTMLやCSSそのものよりも、「始めるまでの準備」であることがとても多いんです。ソフトを入れる。初期設定をする。どこに保存したかわからなくなる。ブラウザで表示されない。そういうところでエネルギーを使ってしまうと、本来学びたい「ホームページってどう作るんだろう」が見えにくくなります。

特に、非エンジニアの方ほど、「自分はまだ作る段階まで来ていない」と感じてしまいがちです。でも実際には、最初の環境さえ整えば、かなり早い段階で作り始められます。今日の講義は、そのための土台を一緒に作る時間だと思ってください。準備で終わる講義ではなく、制作の入口を開く講義です。

CodePenの良いところは、この準備の負担をかなり減らしてくれることです。ブラウザで開けばすぐ使える。HTMLを書く場所、CSSを書く場所、JavaScriptを書く場所が最初から分かれている。下に結果がすぐ表示される。つまり、「今何を触っていて、何が変わったのか」が非常にわかりやすいんですね。

料理で例えると、まな板と包丁と材料が最初からきれいに並んでいる状態です。皆さんは、まず切り方や味付けに集中できます。道具を箱から出すところで疲れなくていい。CodePenは、そういうスタートに向いた環境だと思ってください。

【スライド: CodePenは入門に向いている理由】

入門に向いている理由は、大きく3つあります。

1つ目は、結果がすぐ見えることです。HTMLに文字を1行入れると、すぐ下のプレビューに出ます。CSSで色を変えると、その場で変化が見えます。これは初心者にとってとても大きいです。自分の操作がどこに影響したのかが、すぐわかるからです。

2つ目は、HTML、CSS、JavaScriptが分かれて見えることです。前回の講義で学んだ3要素が、そのまま画面上に並んでいるわけですね。HTMLが骨格、CSSが見た目、JavaScriptが動き。この関係を、実際の画面でそのまま確認できます。

3つ目は、共有しやすいことです。保存するとURLができて、そのURLを相手に送るだけで見てもらえます。講師に見せる、友人に見せる、自分であとから見返す。こういうことがとても簡単です。

もちろん、CodePenだけで全部が完結するわけではありません。実際の本格的なホームページ制作では、複数ファイルや画像管理も必要になります。だから後でStackBlitzに進みます。でも、最初の第一歩としては、CodePenは非常に優秀です。今日は「まず始めるための場所」として一緒に触っていきましょう。

### セクション2：GitHubアカウントを作ろう

【スライド: GitHubアカウントを作成】

では、ここから実際の画面を見ながら進めていきます。

[ライブ開始]

まず、Chromeで `https://github.com` にアクセスします。GitHubというのは、コードやファイルを保存したり、共有したりするためのとても有名なサービスです。今の段階では、「この講座の最後まで使うアカウントをここで1つ作る」と思っていただければ十分です。

すでにGitHubアカウントを持っている方は、新しく作らなくて大丈夫です。そのままそのアカウントを使ってください。まだ持っていない方は、ここで一緒に作りましょう。

画面右上、あるいは中央付近に `Sign up` のようなボタンが見えると思います。これをクリックします。すると、メールアドレス、パスワード、ユーザー名などを入力する流れが出てきます。

ここで少しだけ注意点をお伝えします。メールアドレスは、あとで確認メールを受け取れるものを使ってください。パスワードは、使い回しではなく、普段使っているパスワード管理の方法に合わせて安全に管理してください。そしてユーザー名は、今後URLの一部になることがあるので、あまり気軽すぎる名前より、自分で長く使える名前にしておくと安心です。

ここで「どんなユーザー名にしたらいいかわからない」と迷う方もいるかもしれません。その場合は、本名をそのまま出しすぎない範囲で、自分があとで見てもわかる名前にしておくとよいです。仕事で使う可能性がある方なら、極端にふざけた名前より、落ち着いた名前のほうが後で困りにくいですね。

【スライド: GitHubアカウントは講座の最後まで使う】

このアカウントは、今日のCodePenログインだけで終わりません。PART 4のStackBlitzでも使いますし、後半で公開の流れに入るときにも関係してきます。ですから、「とりあえず今だけの仮アカウント」ではなく、この講座の最後まで使うつもりで作ってください。

メール認証の画面が出たら、受信ボックスを確認しましょう。認証メールが届いているはずです。メールを開いて、認証を進めます。こういうメールは、迷惑メールフォルダに入ることもありますので、見つからないときはそちらも確認してください。

もしここで少し時間がかかっても大丈夫です。アカウント作成は、最初だけ少し手間がかかります。でも、一度作ってしまえば、今後の講義がかなりスムーズになります。ここは焦らず、確実に進めましょう。

この講座全体を通して大事なのは、「途中で止まらないこと」です。少し時間がかかっても、今日ここでアカウントを整えておけば、次からは作ることに集中できます。逆に、この段階をあいまいにしたまま進むと、毎回ログインで迷ったり、保存できなかったりして、細かいストレスが増えてしまいます。ですから、今日は地味に見えても大事な回なんです。

[ライブ終了]

### セクション3：CodePenにGitHubでログインする

【スライド: CodePenにアクセス】

次は、CodePenに移ります。ブラウザで `https://codepen.io` にアクセスしましょう。

[ライブ開始]

CodePenのトップページが開きました。初めて見る方は、「英語のサイトで少し難しそうだな」と思うかもしれません。でも大丈夫です。今日使う場所は限られていますし、必要な流れはとてもシンプルです。

ここで大事なのは、ログインして使うことです。見るだけならログインなしでもできる場面がありますが、ペンを作る、保存する、あとで開く、共有する、といった基本的なことをきちんとやるには、ログインしておいたほうが確実です。

画面の `Sign Up` あるいは `Log In` 付近を見てみましょう。CodePenは複数のログイン方法がありますが、この講座ではGitHubアカウントでログインする流れにそろえます。先ほど作った、あるいはすでに持っているGitHubアカウントを使います。

GitHubでのログインボタンを押すと、GitHub側の認証画面に移ることがあります。「このサービスと連携していいですか」というような確認ですね。ここでは、内容をざっと確認して、許可して進みます。

すると、CodePenに戻ってきます。ユーザーアイコンやアカウント表示が出ていれば、ログイン成功です。

ここで少し安心してほしいのは、CodePenにログインしたからといって、いきなり難しいことを設定しなくていいということです。最初の段階では、「ログインできた」「新しいペンを作れる」「保存できる」、まずこの3つで十分です。設定項目を全部理解しようとしなくて大丈夫です。

【スライド: ログインして使う理由】

ここで、なぜログインが必要なのかを整理しておきます。

1つ目は、作ったものを保存するためです。せっかく作っても、保存できなければ次の講義で使えません。

2つ目は、URLで共有するためです。CodePenでは、作ったペンをURLで見せられます。これはとても便利です。

3つ目は、自分の作業履歴を残すためです。あとで見返すと、「最初はここから始めたんだな」とわかります。学習では、この積み重ねがとても大事です。

もう一つ言うと、ログインしていると「これは自分の作業場所なんだ」という感覚が持ちやすくなります。学習では、この感覚が意外と大切です。毎回ゼロから何かを眺めるのではなく、自分の場所に戻ってきて、前回の続きをやる。この流れができると、学習が習慣になりやすいんですね。

ここで「1分で終わります」と言いたいところですが、実際には人によって2分かかることも3分かかることもあります。でも、それで大丈夫です。大事なのは、ここでちゃんとログインして、次から迷わず入れる状態にすることです。

[ライブ終了]

### セクション4：CodePenの画面構成を理解しよう

【スライド: CodePenの3ペイン構成】

では、実際に新しいペンを開いて、画面を見ていきましょう。

[ライブ開始]

`Create` や `Start Coding` のようなボタンから、新しいペンを作成します。すると、エディタ画面が開きます。

ここでまず見てほしいのが、3つの入力欄です。上や横に `HTML`、`CSS`、`JS` と書かれたエリアが並んでいるはずです。これがCodePenの基本画面です。

HTMLの欄は、ページの中身を書く場所です。見出し、文章、画像、ボタン。そういう骨格を書きます。

CSSの欄は、見た目を書く場所です。色、余白、文字サイズ、配置。そういうデザインや見え方のルールを書きます。

JS、つまりJavaScriptの欄は、動きや反応を書く場所です。この講座の最初の段階では、ここはまだあまり触りません。今は「動きを入れたくなったらここを使うんだな」と知っておけば十分です。

そして、画面の下側や右側にあるのがプレビューです。ここに、今書いている内容の結果が表示されます。これがCodePenのとても良いところで、保存しなくても、入力した変化がかなりすぐに見えます。

この「すぐ見える」という感覚は、本当に大きいです。特に最初の段階では、コードそのものを長く眺めるより、「この1行を書いたらこう変わるんだ」と対応を見ながら覚えるほうが理解しやすいです。頭で理屈だけを覚えるより、画面変化と一緒に覚えるほうが自然なんですね。

【スライド: 前回学んだ3要素がそのまま並んでいる】

前回の講義で、HTMLは骨格、CSSは見た目、JavaScriptは動き、と学びましたよね。CodePenの画面は、まさにその3要素がそのまま並んでいます。ですから、ただ操作するだけでも、頭の中が整理されやすいです。

ここで一つ、初心者の方が最初に持ちやすい不安を取り除いておきます。「間違えて変なことを書いたら壊れるんじゃないか」と感じるかもしれません。でも、CodePenは練習の場です。間違えても直せばいいですし、最悪、新しく作り直せば大丈夫です。むしろ、少し触って変化を見ること自体が勉強になります。

ホームページ制作の学習では、「失敗しないこと」よりも、「変化を見ながら慣れること」のほうが大事です。ですから、今日は完璧な作品を作る日ではありません。画面のどこで何を触ると、どこがどう変わるのか。その感覚をつかむ日です。

### セクション5：基本操作を一緒に試してみる

【スライド: HTMLを1行入れてみる】

では、実際に少しだけ入力してみましょう。まずはHTMLの欄です。

ここに、シンプルに文字を書いてみます。たとえば、こんな感じです。

```html
<h1>はじめてのCodePenです</h1>
```

入力すると、プレビュー側に大きな文字が表示されますね。これがHTMLの役割です。「何を表示するか」を決めています。

次に、そのすぐ下に文章も入れてみましょう。

```html
<h1>はじめてのCodePenです</h1>
<p>ブラウザだけでホームページ作りを始めています。</p>
```

すると、見出しの下に文章が出てきます。ここで大事なのは、入力した内容と表示結果のつながりを目で確認することです。「この1行がこの見た目になるんだな」と対応を見ながら進めると、理解が速くなります。

【スライド: CSSで見た目を変えてみる】

次はCSSの欄です。ここでは、見た目を少し変えましょう。

たとえば、こんなコードを入れてみます。色のコードの意味は今はわからなくて大丈夫です。CSSはPART 3で詳しく学びますので、今は「CSSを少し変えると見た目が変わるんだな」と感じてもらえれば十分です。

```css
body {
  background: #f7f3eb;
  color: #4b3621;
  font-family: sans-serif;
}

h1 {
  color: #7a4b2b;
}
```

すると、背景色が変わって、文字の色も変わります。HTMLはそのままでも、CSSを加えるだけで印象が変わることがわかりますね。前回の「HTMLが骨格、CSSが見た目」という話が、そのままここで体験できます。

ここでぜひ持ってほしい感覚があります。それは、「自分の入力が画面を変えている」という実感です。これが出てくると、ホームページ制作は急に身近になります。見るだけの世界から、自分が作る側に入っていく感じですね。

この瞬間が、とても大事です。ホームページ制作は、何か特別な魔法ではなく、テキストを入力して、その結果を確かめながら作る作業なんだ、ということが実感できるからです。

【スライド: 変更が即座に見えるメリット】

CodePenでは、変化がすぐ見えるので、試行錯誤しやすいです。色を変えてみる。文字を変えてみる。少し余白を入れてみる。そういう小さな実験がしやすいんですね。

これが初心者に向いている理由でもあります。最初から完璧を目指す必要はありません。1文字変えたらどうなるか、1色変えたらどうなるか。その積み重ねで十分です。

実際、上達する人ほど、小さく試します。大きく一気に書くより、少し入れて、見る。少し変えて、見る。この繰り返しです。CodePenは、その小さな試行錯誤をとてもやりやすくしてくれます。

では、タイトルも付けてみましょう。CodePenでは、ペンに名前を付けておくとあとで見つけやすくなります。たとえば、「講義05 はじめてのCodePen」など、後から見てわかる名前にしておくと便利です。

次に、保存もしてみましょう。`Save` のボタンを押すと、今の状態が記録されます。これで、後からもう一度開けるようになります。

【スライド: 保存と共有】

保存したら、URLを確認してみましょう。CodePenでは、このURLを人に送れば、その作品を見てもらえます。まだシンプルな内容でも構いません。大事なのは、「作ったものが保存され、URLとして持てる」という経験です。

この経験は、小さく見えてかなり重要です。自分が触った結果が、ただその場で消えるのではなく、保存され、また開けて、人にも見せられる。これがわかると、学習が点ではなく線になります。今日のペンが、次の講義のスタート地点になるわけです。

ここまで来ると、「自分でも作れそうだな」という感覚が少し出てくるはずです。実際、今日はもう作り始めています。まだ見出しと文章と色だけですが、立派な第一歩です。

[ライブ終了]

### セクション6：今日の位置づけと次へのつながり

【スライド: 今日は準備ではなく、すでに制作の入口】

ここで、今日の位置づけを整理しておきましょう。今日やったのは、単なる設定作業ではありません。すでに制作の入口に立っています。

GitHubアカウントを作った。CodePenにログインした。HTMLとCSSの場所を確認した。少し入力して、表示が変わるところまで見た。これは、単なる事務作業ではなく、ホームページ制作を始めるための土台づくりです。

特に大事なのは、「ブラウザだけで始められる」という実感です。ここでつまずかなければ、次の講義からは、実際に文字を書き、タグを書き、少しずつ形を作っていけます。非エンジニアの方にとっては、この最初の心理的ハードルを越えることがとても大きいんです。

前回の講義で、AIはたたき台を作ってくれると学びました。今日の講義で、そのたたき台を貼り付けたり、直したり、結果を見たりする場所ができました。つまり、前回の話と今日の話が、ここでつながったわけです。

そして、ひとつ先の話もしておきましょう。PART 4に入ると、StackBlitzというツールも使い始めます。StackBlitzは、ファイルを複数に分けて管理できる、より本格的な制作環境です。ただ、今の段階ではまだ必要ありません。まずはCodePenで、書く・見る・直すの基本感覚を育てることが先です。本格的にファイルを分けて作る段階になったら、StackBlitzに進みます。

次の講義からは、このCodePenの中で、もっとはっきり「自分で作っている感覚」が出てきます。今日の時点ではまだ準備と小さな確認が中心ですが、この土台があるからこそ、次からスムーズに進めます。
