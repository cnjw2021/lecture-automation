## 導入（1.5分）

皆さん、こんにちは。前回の講義では Flexbox を学びましたね。ディスプレイフレックスを1行書くだけで要素が横に並ぶ、あの魔法のような体験、覚えていますか。ジャスティファイコンテントで横の配置を決めて、アラインアイテムズで縦の位置を揃えて、ギャップで間隔を調整する。カード3枚を横に並べたり、ナビゲーションバーを作ったり、Flexbox のおかげでレイアウトが格段に楽になりました。

さて、今日のテーマは「レスポンシブデザイン」です。前回 Flexbox できれいに3列に並べたカード、思い出してください。パソコンの大きな画面ではいい感じに並んでいますよね。でも、もしこのページをスマートフォンで見たら、どうなると思いますか？

## 本編

### セクション1: なぜレスポンシブが必要なのか（3分）

【スライド: モバイルトラフィック統計】

まず、なぜレスポンシブデザインが必要なのか、数字で見てみましょう。

世界中の Web トラフィックのうち、約60パーセントがスマートフォンからのアクセスです。日本に限っても、スマートフォンからのアクセスは70パーセントを超えるというデータがあります。つまり、皆さんが作るホームページを見に来る人の半分以上は、パソコンではなくスマートフォンで見ているんですね。

たとえば、美容室のホームページを作ったとしましょう。お客さんの多くは、電車の中やカフェで、スマートフォンから「近くの美容室」と検索して見つけてくれます。そのとき、画面いっぱいに小さな文字がぎゅっと詰まったページが表示されたら、すぐに閉じてしまいますよね。逆に、スマートフォンでも読みやすく、ボタンもタップしやすいページなら、そのまま予約してもらえるかもしれません。

これは美容室に限った話ではありません。整体院のサイトでも、英会話教室のサイトでも、お客さんがスマートフォンで見たときに「あ、ちゃんとしたサイトだな」と感じてもらえるかどうかは、とても大きいんです。

【スライド: PC表示 vs スマホ表示の比較】

レスポンシブデザインというのは、一つのHTMLファイルで、画面サイズに合わせてレイアウトを自動的に調整する手法のことです。パソコン用とスマートフォン用で別々のページを作る必要はありません。CSSの書き方を工夫するだけで、同じHTMLが画面サイズに応じて姿を変えてくれるんです。

今の時代、レスポンシブ対応していないホームページは、それだけで信頼性が下がってしまいます。Google の検索結果でも、モバイル対応しているサイトが優先されるようになっています。だからこそ、レスポンシブデザインは「あると便利」ではなく、「なくてはならない」ものなんですね。

### セクション2: メディアクエリの概念（3分）

【スライド: メディアクエリの概念図】

では、どうやって「画面サイズに応じてCSSを変える」のでしょうか。ここで登場するのが「メディアクエリ」です。

メディアクエリというのは、CSSの中で「もし画面の幅がこれ以下だったら、このスタイルを適用してね」と条件を書ける仕組みです。日本語にすると「メディアへの問い合わせ」ですね。ブラウザに対して「今の画面幅はいくつですか？」と問い合わせて、その答えに応じてスタイルを切り替える、というイメージです。

たとえば、こんな感じです。「画面幅が768ピクセル以下なら、カードを1列に並べてね」。これを CSS の言葉で書くのがメディアクエリです。

ここで一つ、大事な数字の目安をお伝えしておきます。

768ピクセル以下が、だいたいタブレットやスマートフォンの画面幅です。480ピクセル以下なら、ほぼスマートフォンだけ。1024ピクセル以上なら、パソコンの画面です。

この数字、暗記する必要はありません。「だいたいこのくらい」という目安を知っておけば十分です。実際の現場でも、この辺りの数字をよく使います。

メディアクエリのすごいところは、HTMLは一切変えなくていいことです。CSSに条件付きのスタイルを追加するだけ。HTMLの骨組みはそのままで、表示だけが画面サイズに合わせて変わる。これがレスポンシブデザインの基本的な仕組みです。

### セクション3: メディアクエリの基本文法（5分）

では、CodePen を開いて実際に書いてみましょう。前回の Flexbox の講義で作ったカード3枚のペンを使います。開いてください。

今、カード3枚が横に並んでいる状態ですよね。HTML にはカードコンテナの中にカードが3つ、CSS にはカードコンテナにディスプレイフレックスやギャップが指定されている状態です。

では、CSSの一番下に、メディアクエリを書いてみましょう。

```css
@media (max-width: 768px) {
  .card-container {
    flex-direction: column;
  }
}
```

一行ずつ見ていきましょう。

まず、アットマークメディアで始まります。これが「メディアクエリですよ」という宣言です。CSSの中で特別な指示を出すときに使う書き方ですね。

次に、丸括弧の中に「マックスウィドゥス コロン 768ピクセル」と書いてあります。これは「画面の幅が最大768ピクセルまでのとき」という条件です。つまり、768ピクセル以下の画面幅でだけ、この中のスタイルが有効になります。

そして、波括弧の中に普通のCSSを書いています。「カードコンテナのフレックスディレクションをカラムにする」と書いてありますね。フレックスディレクションのカラムは、前回やりましたね。横並びを縦並びに変える指定です。

つまり、この3行で「画面幅が768ピクセル以下のときは、カードを縦に並べてね」という指示になるわけです。

ここで注目してほしいのが、メディアクエリの「構造」です。アットマークメディアの後に条件を書いて、その中に波括弧で普通のCSSを囲む。メディアクエリの波括弧の中に、さらにセレクタと波括弧が入っている形になりますよね。波括弧が二重になっているわけです。最初はちょっと見慣れないかもしれませんが、「外側の波括弧が条件の範囲、内側の波括弧がいつもどおりのCSS」と考えると、すっきりすると思います。

では、プレビューで確認してみましょう。CodePen のプレビューエリアの端をドラッグして、幅を狭くしてみてください。ある幅を下回ったところで、パッとカードが縦並びに変わりましたよね。今度は幅を広げてみてください。また横並びに戻ります。

面白いですよね。CSSにたった4行書き足しただけで、画面幅に応じてレイアウトが切り替わるようになりました。これがメディアクエリの効果です。768ピクセルを境にして、パッと切り替わる。まるでスイッチのように、条件に合ったときだけスタイルがオンになるんですね。

### セクション4: ブレイクポイントを増やす（5分）

一つのメディアクエリで縦並びにできましたね。でも実際の Web サイトでは、もう少し細かく対応することが多いです。パソコンでは3列、タブレットでは2列、スマートフォンでは1列、という3段階の切り替えをやってみましょう。

この「レイアウトが切り替わる画面幅」のことを、ブレイクポイントと呼びます。「ここで折れる」というポイントですね。今日は768ピクセルと480ピクセル、2つのブレイクポイントを設定してみます。

まず、先ほど書いたメディアクエリを一度消してください。新しく書き直します。

ここで、3列から2列への切り替えを実現するために、前回少しだけ触れたフレックスラップが重要になってきます。カードコンテナの既存のCSSを確認してください。ディスプレイフレックスの下に、フレックスラップのラップが書いてありますか？もし書いていなければ追加してください。

フレックスラップのラップは、要素が入りきらないとき次の行に折り返す指定でしたね。これがないと、カードの幅を50パーセントにしても、ブラウザが無理やり1行に詰め込もうとしてしまいます。フレックスラップがあるからこそ、「入りきらないカードは下の行へどうぞ」と、自然な折り返しが実現できるわけです。

次に、カードのスタイルを追加しましょう。

```css
.card {
  flex: 1 1 calc(33.333% - 20px);
}
```

フレックスプロパティの最後の値、「カルク 33.333パーセント マイナス 20ピクセル」、これはカードの基本幅を全体の3分の1に設定しています。カルクというのは CSS の計算関数で、パーセントからピクセルを引くといった異なる単位の計算ができる便利な機能です。20ピクセルを引いているのは、ギャップの分を考慮しているためです。これで、3枚のカードがぴったり1行に収まります。

カルクは今日はこの使い方だけ覚えれば大丈夫です。「パーセントから固定のピクセルを引ける便利な関数がある」くらいの認識で十分です。

では、メディアクエリを2つ書きましょう。

```css
@media (max-width: 768px) {
  .card {
    flex: 1 1 calc(50% - 10px);
  }
}

@media (max-width: 480px) {
  .card {
    flex: 1 1 100%;
  }
}
```

1つ目のメディアクエリ、768ピクセル以下のとき。カードの幅を50パーセントにしています。50パーセントということは、1行に2枚入るサイズですね。3枚のうち2枚が1行目に並んで、3枚目は次の行に折り返されます。ここでフレックスラップが効いてくるわけです。

2つ目のメディアクエリ、480ピクセル以下のとき。カードの幅を100パーセントにしています。100パーセントは画面の全幅ということですから、1行に1枚しか入りません。つまり、カードが縦に1列で並びます。

では、プレビューの幅を変えながら確認してみましょう。まず広い画面から始めます。3枚のカードが横に並んでいますよね。ゆっくり幅を狭くしていってください。768ピクセルあたりで、2列に変わりましたね。さらに狭くしていくと、480ピクセルあたりで1列に変わります。今度は逆に広げてみてください。1列から2列、2列から3列へとスムーズに切り替わりますよね。

これが、レスポンシブデザインの基本パターンです。メディアクエリを複数書いて、画面幅に応じてレイアウトを段階的に変えていく。実際の Web 制作でも、まさにこのパターンをとてもよく使います。

ちなみに、メディアクエリは CSSの中で上から順に読まれます。だから、大きい数字の768ピクセルを先に書いて、小さい数字の480ピクセルを後に書きます。もし順序を逆にすると、480ピクセルの指定が768ピクセルの指定で上書きされてしまうことがあるので、書く順番にはちょっと注意してくださいね。

### セクション5: ブラウザサイズを変えてリアルタイム確認（3分）

ここまで、CodePen のプレビューエリアの幅を変えて確認してきましたが、実際の Web 制作では「開発者ツール」を使ってモバイル表示を確認します。パート3の最初のほうでチラッと触れた、Chrome の開発者ツールですね。

ここではその中の「デバイスモード」という機能を紹介します。Chrome で Web ページを開いた状態で、キーボードのエフ12キーを押すと、開発者ツールが開きます。マックをお使いの方は、コマンドキーとオプションキーとアイキーを同時に押しても同じです。

開発者ツールが開いたら、ツールバーの左のほうにある小さなアイコンを探してください。スマートフォンとタブレットが重なったようなアイコンです。これをクリックすると、画面がモバイル表示のシミュレーションモードに切り替わります。

ここで画面上部のドロップダウンからアイフォンやギャラクシーなど、いろいろなデバイスを選ぶことができるんです。選んだデバイスの画面サイズで、ページがどう見えるかをその場で確認できます。たとえばアイフォン14を選ぶと、横幅390ピクセルで表示されます。480ピクセル以下ですから、さっき書いた2つ目のメディアクエリが効いて、カードが1列になっているはずですよね。

「レスポンシブ」というモードを選ぶと、幅の数字を直接入力することもできます。768と入力してみてください。2列になりましたね。480にすると1列。こうやって、自分で書いたブレイクポイントの前後で表示がどう変わるかを、ピンポイントで確認できるんです。

この開発者ツールのデバイスモードは、レスポンシブデザインを作るときの必須ツールです。今日は見方だけ覚えておけば大丈夫です。この後の実習でも使いますし、パート4以降でもっと活用していきますので。

### セクション6: viewportメタタグ（3.5分）

レイアウトの切り替えができるようになりましたが、実はもう一つ、とても大事な設定があります。viewportメタタグというものです。

【スライド: viewportメタタグの説明】

これがないと何が起きるかというと、スマートフォンのブラウザが「このページはパソコン用だな」と判断して、画面を勝手にぐーっと縮小して全体を表示しようとするんです。せっかくメディアクエリで「768ピクセル以下なら2列にしてね」と書いても、スマートフォンが「自分は1024ピクセルの画面ですよ」のように嘘をついてしまう。だから、メディアクエリが発動しないんです。

なぜこんなことが起きるかというと、スマートフォンが登場した初期のころ、ほとんどの Web サイトはパソコン用に作られていました。だから、スマートフォンのブラウザは「パソコン用のページをそのまま縮小して見せる」のがデフォルトの動作になっているんです。レスポンシブデザインが一般的になった今でも、この初期設定は残っています。

これを防ぐのがviewportメタタグです。HTMLのヘッドタグの中に書く一行で、「画面の幅をデバイスの実際の幅に合わせてね」とブラウザに伝えます。

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

メタタグのネームがビューポート、コンテンツに「ウィドゥス イコール デバイスウィドゥス、イニシャルスケール イコール 1.0」と書いてあります。これは「画面幅をデバイスの実際の幅に合わせて、初期の拡大率は1倍にしてね」という意味です。

この一行がないと、レスポンシブデザインは正しく動きません。逆に言えば、この一行さえ入れておけば、メディアクエリがきちんと機能してくれます。

CodePen を使っている場合の設定方法を説明しますね。CodePen では、HTMLの入力エリアにはボディタグの中身だけを書く仕組みになっているので、ヘッドタグの中に何かを書きたいときは、別の場所から設定する必要があります。

CodePen の画面右上にある「Settings」、セッティングスをクリックしてください。設定画面が開いたら、左のメニューから「HTML」を選びます。すると「Stuff for ヘッド」という入力欄がありますね。ここに、先ほどのviewportメタタグをコピーして貼り付けてください。最後に「Save & Close」を押せば完了です。

これで、CodePen のプレビューでもviewportメタタグが有効になります。少し面倒に感じるかもしれませんが、この一行のためだけの設定です。パート4以降でエディタを使うようになったら、HTMLファイルに直接書けるようになりますので、ここだけ我慢してくださいね。

ちなみに、エーアイにHTMLを生成してもらうと、ほぼ必ずこのviewportメタタグが含まれています。エーアイもこの一行が大事だと知っているわけです。だから覚える必要はありません。「レスポンシブにはviewportメタタグが必要。なかったら追加する」。これだけ覚えておけば大丈夫です。

### セクション7: 実習 — 自己紹介ページをレスポンシブに（5.5分）

さて、ここからは実習の時間です。これまでの講義で作ってきた自己紹介ページのペンを開いてください。前回の Flexbox の講義でカードを並べたりしたペンですね。

今から、この自己紹介ページをレスポンシブ対応にアップグレードしていきましょう。やることは3つです。

まず1つ目、viewportメタタグの追加です。先ほど説明したとおり、セッティングスからHTMLのStuff for ヘッドにviewportメタタグを追加してください。まだ追加していない方は、ここで設定しておきましょう。

2つ目、メディアクエリの追加です。CSSの一番下に、メディアクエリを書いていきましょう。まずは768ピクセル以下の指定からです。

```css
@media (max-width: 768px) {
  .card-container {
    flex-direction: column;
  }

  h1 {
    font-size: 24px;
  }

  body {
    padding: 10px;
  }
}
```

ここでは3つのことをしています。まず、カードコンテナを縦並びに変えています。次に、見出しの文字サイズを24ピクセルに変更しています。パソコン用のCSSでは36ピクセルや40ピクセルなど、大きめに指定していることが多いですよね。でも、スマートフォンの横幅は390ピクセル程度しかありません。そこに40ピクセルの文字を表示すると、1行に数文字しか入らず、見出しだけで何行にもなってしまいます。24ピクセルにすれば、スマートフォンでも1行にきちんと収まるサイズです。

そして、ボディのパディングも10ピクセルに減らしています。パソコンでは20ピクセルや30ピクセルの余白があると見やすいですが、横幅390ピクセルの画面で左右30ピクセルずつ取ると、コンテンツに使える幅が330ピクセルしか残りません。10ピクセルに抑えれば、370ピクセル使えます。この40ピクセルの差は、スマートフォンでは大きいんです。

このように、メディアクエリの中には複数のセレクタをまとめて書くことができます。一つのメディアクエリの中で、レイアウトもフォントサイズもパディングも、同じ条件のものをまとめて調整できるわけですね。

3つ目、プレビューで確認です。CodePen のプレビューエリアの幅を変えて、レイアウトが切り替わることを確認してください。カードが縦に並んで、見出しの文字サイズが変わりましたか？

もう一歩やってみましょう。新しいタブでプレビューを開いて、開発者ツールで確認する方法です。CodePen のプレビューエリアの右上にある「Change View」をクリックしてください。メニューが出てきますので、その中から「Debug mode」を選びます。すると、プレビューが新しいタブで開きます。

新しいタブが開いたら、エフ12キーを押して開発者ツールを開いてください。そして、さっき紹介したスマートフォンとタブレットのアイコンをクリックして、デバイスモードに切り替えます。上部のドロップダウンからアイフォンを選んでみてください。画面がスマートフォンサイズになりましたよね。

ここで注目してほしいのが、画面上部に表示されている数字です。たとえばアイフォン14なら「390 x 844」のように表示されています。これが、そのデバイスの画面サイズです。横幅が390ピクセルなので、さっき書いた768ピクセル以下のメディアクエリが効いている状態ですね。

アイパッドエアーを選ぶと、横幅が820ピクセルになります。768ピクセルより大きいので、パソコン用のレイアウトが表示されるはずです。このように、デバイスを切り替えながら、自分のメディアクエリがどのデバイスで効くのかを確認できます。

どうですか？自己紹介ページが、スマートフォンの画面幅でもきれいに表示されていますか？パソコンでは横に並んでいたカードが、スマートフォンでは縦に並んで、一枚ずつしっかり読めるようになっていますよね。

これで、皆さんの自己紹介ページはレスポンシブ対応になりました。パソコンでもスマートフォンでも、きれいに表示されるページの完成です。

### セクション8: レスポンシブデザインのコツ（2.5分）

最後に、レスポンシブデザインのコツを2つお伝えしておきますね。

1つ目は、「モバイルファースト」という考え方です。今日はパソコン用のCSSを先に書いて、メディアクエリで「画面が小さいときは、こう変えてね」と調整しましたよね。これは「デスクトップファースト」と呼ばれるアプローチです。

実は、プロの現場ではこれを逆にすることが多いんです。まずモバイル用のシンプルなCSSを書いて、メディアクエリで「画面が大きいときは、横に並べてね」と指定する。このとき使うのは、マックスウィドゥスではなく「ミンウィドゥス」です。「最小幅がいくつ以上のとき」という条件ですね。モバイルの画面はシンプルなので、まずシンプルに作って、画面が広いときに複雑なレイアウトを追加する。そのほうがCSSがすっきりすることが多いんです。

ただ、今の段階ではこの「パソコンから作って、モバイルに合わせる」やり方で十分です。考え方は同じですから。「そういうアプローチもあるんだな」と、頭の片隅に置いておいてください。

2つ目は、メディアクエリで変えられるのはレイアウトだけではない、ということです。今日はフレックスディレクションの変更と見出しの文字サイズ、パディングを調整しましたが、同じように、マージンを変えたり、画像の大きさを変えたり、あるいはパソコンでは横に表示しているナビゲーションをスマートフォンではメニューアイコンに変えたり。画面サイズに応じて、CSSで指定できることなら何でも切り替えられます。

大事なのは、暗記することではなく「画面サイズに応じてCSSを変えられる」ということを知っていることです。細かい数値は、そのつど調整すればいいんです。そしてエーアイにレスポンシブ対応をお願いするときも、「768ピクセル以下でカードを1列に変えて、フォントサイズも少し小さくして」のように具体的に指示できれば、きちんと対応してくれます。CSSの仕組みを理解しているからこそ、エーアイへの指示も的確になるわけですね。

## まとめ（1.5分）

今日の講義のポイントを3つ振り返りましょう。

1つ目。レスポンシブデザインは、一つのHTMLで画面サイズに応じた表示ができる手法です。スマートフォンからのアクセスが半数を超える今の時代、レスポンシブ対応は必須です。

2つ目。メディアクエリ、アットマークメディアで「画面幅がいくつ以下のときは、こう表示する」というCSSの条件分岐が書けます。768ピクセルと480ピクセルが、よく使われるブレイクポイントの目安です。

3つ目。viewportメタタグをHTMLに入れないと、スマートフォンでメディアクエリが正しく動きません。この一行を忘れずに入れること。エーアイに生成してもらったHTMLには大抵入っていますが、なかったら自分で追加できるようになりましたね。

今日で、Flexbox とメディアクエリを組み合わせて、画面サイズに応じたレイアウト変更ができるようになりました。パソコンでは3列、タブレットでは2列、スマートフォンでは1列。この切り替えパターンは、実際の Web 制作でとてもよく使います。

次回の講義では、よく使うCSSレシピ集をお届けします。ボタンにホバーエフェクトをつけたり、カードに影をつけたり、角を丸くしたり。見た目をぐっとプロっぽくする、実践的なテクニックを一気に学びます。お楽しみに。

### 講師メモ
- 前回のFlexbox講義で作ったカード3枚のペンを再利用する。受講生が見つけられない場合はフォーク方法を案内
- プレビューエリアの幅を手動で縮める操作は画面収録でわかりやすく見せる
- ブレイクポイントの数値（768px, 480px）は暗記不要と必ず伝える
