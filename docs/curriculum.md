## PART 1: AI時代のWeb、そして私たちの戦略（2時間30分）

> **このパートの目標**: 受講生が「この講座を受ける理由」を確信し、Webの全体像を理解する。

### 1-1. オリエンテーション（20分）[PPT]
- 講座ロードマップの全体俯瞰
- 「コーディングを学ぶのではなく、AIと協業する方法を学ぶ」
- 完成目標：この講座が終わったら皆さんが手に入れるもの
  - 自分で作ったWebサイト2〜3個
  - AIに正確に指示する能力
  - インターネットにデプロイされた実際のURL
- 受講前の準備物確認（PC/Mac、インターネット）

### 1-2. Webはどのように動くのか（25分）[PPT]
- ブラウザにURLを入力すると何が起きるか（ダイアグラム）
- サーバー、ドメイン、IPアドレス — 宅配便に例える
- HTTPとは？「ブラウザとサーバーの会話ルール」
- HTTPSとセキュリティ — 鍵アイコンの意味
- 実習予告：「まもなく私たちが直接この過程を目で確認します」

### 1-3. Webサイトの3要素：HTML、CSS、JavaScript（20分）[PPT + ライブ]
- 家を建てる例え：HTMLは骨組み、CSSはインテリア、JSは家電製品
- [ライブ] 実際のWebサイトでF12開発者ツールを開いてみる
  - Yahoo! JAPAN、Google、AppleサイトのHTMLソースを見学
  - CSSをオフにするとどうなるか実演
  - 「これら全てがテキストファイルである」という気づきへ導く

### 1-4. AIが変えたWeb制作の風景（25分）[PPT + ライブ]
- [PPT] 2020年 vs 2025年：Webサイトを1つ作るのにかかる時間の比較
- [PPT] AIツールマップ：どのAIがどの役割を果たすか
  - コード生成：Claude、ChatGPT（本講座のメインツール）
  - UI生成：v0.dev、Bolt.new
  - デザイン：Figma AI、Midjourney
  - デプロイ：自動化ツール
- [ライブ] ライブデモ：Claudeに「カフェのホームページを作って」と言うと？
  - 30秒で出てくる成果物を見せる
  - 「でもこれを修正するには？ここから私たちの役割が始まります」
- [PPT] AI時代のWeb制作者の役割再定義
  - コーダー（Coder）→ ディレクター（Director）
  - 「書く」のではなく「指示し、検収し、修正する」

### 1-5. 開発環境セットアップ — CodePen入門（20分）[ライブ]
- 「インストール不要、ブラウザだけで始められる」— 受講生の最大の障壁を除去
- CodePen（https://codepen.io）にアクセス
- GitHubアカウント作成（PART 4のStackBlitzでも同じアカウントを使用）
  - GitHub（https://github.com）にアクセス → Sign up → メール認証
  - 「このアカウントは講座の最後まで使います。大切に覚えておいてください」
- CodePenにGitHubアカウントでログイン
  - ペンの作成・保存・共有にはログインが必要
  - 「1分で終わります。一緒にやりましょう」
- CodePenの画面構成
  - 3ペイン構成：HTML / CSS / JS — 「この3つがWebの3要素そのまま」
  - 下部プレビュー：コードを変えると即座に結果が反映される
- 基本操作の確認
  - HTMLペインにコードを入力 → リアルタイムプレビュー
  - CSSペインでスタイル適用 → 即座に見た目が変わる
  - ペンの保存と共有（URL共有）
- 「PART 4で本格的なプロジェクトに入る時に、ファイル管理ができるStackBlitzというツールも使います」— 予告

### 1-6. 初めてのWebページ — Hello World（20分）[ライブ]
- CodePenで新しいペンを作成
- HTMLペインに「Hello World」を入力 → 即座にプレビューに表示
- HTMLの基本構造を手で書いてみる
  - `<!DOCTYPE html>`、`<html>`、`<head>`、`<body>` — 「おまじない」として紹介
  - 「CodePenではこの基本構造を自動で補ってくれるので、bodyの中身だけ書けばOK」
- テキストを修正するとリアルタイムで変わる魔法
- 「おめでとうございます、皆さんは今Web開発者になりました」
- **チャレンジ課題**：自分の名前に変えてみよう

### 1-7. HTML基本構造とCodePenの活用（20分）[ライブ]
- HTMLの基本構造をより詳しく理解
  - `<head>`の役割：ブラウザへの設定情報（title、meta、link）
  - `<body>`の役割：画面に表示される内容
  - 「CodePenのHTMLペインに書くのはbodyの中身」
- CodePenの便利な機能
  - Emmet対応：`ul>li*3`で一気にリスト作成
  - ショートカット：Ctrl+S（保存）、Ctrl+Z（元に戻す）
  - Settings → HTML：`<head>`にタグを追加する方法
  - 外部CSSライブラリの読み込み方法（後のパートで使用）
- 実習：自己紹介の簡単なページを作成
  - 見出し（h1）で自分の名前
  - 段落（p）で一言自己紹介
  - 画像（img）の追加にチャレンジ
- 「ここまでできれば、次のPART 2はスムーズに進めます」

## PART 2: HTML — Webの骨格を読む（3時間30分）

> **このパートの目標**: HTMLを「暗記する」のではなく「読める」状態になる。AIが生成したHTMLを見て構造を把握できる。

### 2-1. HTMLのルール：タグという約束（20分）[PPT + ライブ]
- [PPT] タグの構造：`<開くタグ>内容</閉じタグ>`
- [PPT] 親子関係：マトリョーシカ人形に例える
- [ライブ] CodePenで直接タグを作成 → 結果確認
- よくあるミス：タグの閉じ忘れ、タイプミス → エラー画面を見せる

### 2-2. テキストを扱うタグ（25分）[ライブ]
- 見出し：`<h1>` 〜 `<h6>` — 新聞記事のヘッドラインのように
- 段落：`<p>` — 本文はここに
- 強調：`<strong>`、`<em>` — ボールド、イタリック
- リスト：`<ul>`、`<ol>`、`<li>` — 順序あり/なしリスト
- 改行：`<br>` — 「Enterを押しても改行されない理由」
- 水平線：`<hr>` — 区切り線
- 実習：CodePenで自己紹介ページに見出し、紹介文、趣味リストを追加

### 2-3. リンクと画像（25分）[ライブ]
- リンク：`<a href="">` — Webの核心、ページを結ぶ糸
- 新しいタブで開く：`target="_blank"`
- 画像：`<img src="" alt="">` — セルフクロージングタグ
- 画像パスの理解
  - [PPT] 相対パス vs 絶対パス — 概念を図で説明
  - CodePenでは外部URL（絶対パス）を使用 —「PART 4のStackBlitzで相対パスを実際に体験します」
- 無料画像ソース：Unsplash、Pexelsの紹介
  - 画像URLを直接コピーして`src`に貼り付ける方法
- 実習：CodePenで自己紹介ページに写真（外部URL）とSNSリンクを追加

### 2-4. 構造を組み立てるタグ（25分）[PPT + ライブ]
- [PPT] セマンティックタグの概念：「機械（AI、検索エンジン）も読める構造」
- `<header>`、`<nav>`、`<main>`、`<section>`、`<article>`、`<footer>`
- `<div>` — 「意味はないが、まとめてくれる箱」
- [ライブ] CodePenで自己紹介ページをセマンティック構造にリファクタリング
  - 「CodePenでは1つのペインに書いていますが、実際のWebサイトではこれがindex.html全体の構造になります」
- [ライブ] 構造変更の前後比較 — 「見た目は同じだが意味が違う」

### 2-5. フォーム（Form）— ユーザー入力を受け取る（30分）[ライブ]
- `<form>`、`<input>`、`<textarea>`、`<select>`、`<button>`
- inputタイプ：text、email、password、number、date
- placeholder、required属性
- 実習：CodePenでシンプルなお問い合わせフォーム作成
- 「このフォームを実際に動かすには？」→ サーバーの必要性に触れる（後で無料サービスで解決）

### 2-6. テーブル（Table）基礎（15分）[ライブ]
- `<table>`、`<tr>`、`<th>`、`<td>`
- シンプルな料金表を作る
- 「最近はテーブルよりCSS Gridを多く使う」— 予告

### 2-7. AIにHTMLを任せる — 初めての実践（30分）[ライブ]
- Claude/ChatGPTに「レストランのメニューページのHTMLを作って」とリクエスト
- AIが生成したコードをCodePenに貼り付けて確認
- AIが作ったコードを1行ずつ読んでみる
  - 「このタグは何？」→ 学んだことで理解可能
  - 「これは見たことない」→ 検索/AIに聞く
- AI生成物の問題点を発見する
  - セマンティックタグの欠落
  - altテキストの不備
  - 不要なネスト
- CodePen上でAI生成物を修正する — 「これこそHTMLを学んだ理由」
- **核心メッセージ**：「AIが作ったものをそのまま使う人 vs 読んで直せる人」

### 2-8. PART 2まとめ + Q&A（10分）[PPT]
- 学んだタグの総整理（チートシート）
- 「全部暗記する必要はない、このチートシートを横に置いて作業すればいい」
- 次のパート予告：「骨格に服を着せよう」

## PART 3: CSS — Webに服を着せる（4時間）

> **このパートの目標**: CSSを「設計する」のではなく、AIが生成したCSSを「理解して調整」できる。色、サイズ、配置を自由に修正できる。

### 3-1. CSSの仕組み（20分）[PPT + ライブ]
- [PPT] CSSの3つの適用方法：インライン、内部、外部
- [PPT] 「なぜ外部CSSファイルを使うのか」— メンテナンス性
  - 実際のWebサイトでは`<link rel="stylesheet" href="style.css">`でCSSファイルを読み込む
  - 「CodePenが裏側でこの接続を自動的にやってくれている。PART 4のStackBlitzで自分で書いてみます」
- [ライブ] CodePenのCSSペインを使ってスタイル適用
- [ライブ] 初めてのCSS：背景色を変える —「お、変わった！」

### 3-2. セレクタ — CSSがHTMLを見つける方法（25分）[PPT + ライブ]
- [PPT] セレクタ三銃士：タグ、クラス（`.`）、ID（`#`）
- [PPT] 「クラスはニックネーム、IDはマイナンバー」
- [ライブ] 実習：同じ`<p>`タグなのに違うスタイルを適用する
- よく使うセレクタパターン5つ
- 「AIが作ったCSSで`.hero-section`が何か、もうわかる」

### 3-3. 色とタイポグラフィ（25分）[ライブ]
- 色の表現：名前、HEX、RGB
- カラーパレットツール紹介：Coolors、Adobe Color
- フォント適用：Google Fontsの使い方
  - Google Fontsサイトにアクセス → フォント選択
  - CodePenの場合：Settings → CSS → External Stylesheets にGoogle Fonts URLを追加 → CSSで適用
  - 「実際のWebサイトでは`<link>`タグをHTMLに追加する方法を使います」
  - 日本語フォント推奨：Noto Sans JP、M PLUS Rounded 1c、Zen Kaku Gothic New
- font-size、font-weight、line-height、letter-spacing
- 実習：CodePenで自己紹介ページにおしゃれなフォントと色を適用

### 3-4. ボックスモデル — CSSの核心概念（25分）[PPT + ライブ]
- [PPT] すべてのHTML要素は「箱」だ — ダイアグラム
- content、padding、border、margin
- [ライブ] 開発者ツールでボックスモデルを直接確認
- [ライブ] paddingとmarginの違いを目で見る
- box-sizing: border-box — 「これさえ知れば計算が楽になる」
- 実習：CodePenでカード型レイアウト作成

### 3-5. Flexbox — 現代Webレイアウトの核心（35分）[PPT + ライブ]
- [PPT] Flexbox以前：floatの苦しみ（昔の話、ありがたみを感じる）
- [PPT] Flexbox核心概念：コンテナとアイテム、主軸と交差軸
- [ライブ] display: flex —「1行追加するだけで魔法が起きる」
- justify-content：横並び（center、space-between、space-around）
- align-items：縦並び
- flex-direction：横↔縦切り替え
- gap：アイテム間の間隔
- flex-wrap：折り返し
- 実習：CodePenでナビゲーションバー作成
- 実習：CodePenでカード3枚を横に並べる
- 便利なツール紹介：Flexbox Froggyゲーム（学習用）

### 3-6. レスポンシブデザイン — モバイルでもきれいに（30分）[PPT + ライブ]
- [PPT] なぜレスポンシブが必要か — モバイルトラフィック統計
- [PPT] メディアクエリの概念：「画面サイズに応じてCSSを変える」
- [ライブ] @media 基本文法
- [ライブ] デスクトップ3列 → タブレット2列 → モバイル1列の切り替え実演
- [ライブ] ブラウザサイズを縮めながらリアルタイム確認
- viewport metaタグ — 「これを入れないとモバイルで崩れる」
  - CodePenの場合：Settings → HTML → Stuff for `<head>` にviewportタグを追加
- 実習：CodePenで自己紹介ページをレスポンシブにアップグレード
  - CodePenの「Change View」→ 新しいタブでプレビューを開き、ブラウザの開発者ツールでモバイル表示を確認

### 3-7. よく使うCSSレシピ集（25分）[ライブ]
- ボタンスタイリング（ホバーエフェクト付き）
- 影効果（box-shadow）
- 角丸（border-radius）
- グラデーション背景
- シンプルなトランジション効果（transition）
- スムーズスクロール（scroll-behavior: smooth）
- 「これらのレシピを組み合わせれば、大抵のデザインが可能」

### 3-8. AIにCSSを任せる — 2回目の実践（30分）[ライブ]
- Claudeに「このHTMLにモダンなCSSを当てて」とリクエスト
- AIが生成したCSSをCodePenのCSSペインに貼り付けて確認
- AIが作ったCSSを分析する
  - セレクタ構造の把握
  - 色/フォント体系の理解
  - レイアウト方式の確認（Flexbox? Grid?）
- 「この部分が気に入らない」→ CodePen上で自分で修正
  - 色の変更
  - 間隔の調整
  - フォントの変更
  - モバイル対応の追加/修正
- AIへの修正リクエスト：効果的なプロンプトの書き方
  - 悪い例：「もっときれいにして」
  - 良い例：「ヘッダーの背景を#2563EBに変えて、カード間の間隔を24pxに広げて」
- **核心メッセージ**：「CSSを知っている人は、AIに正確な指示ができる」

### 3-9. PART 3まとめ（10分）[PPT]
- CSSプロパティチートシート
- 「全部暗記する必要はない。検索とAIがある。しかし何を検索すべきかは知らなければならない」
- 次のパート予告：「いよいよ本物のサイトを作る」

## PART 4: AIと一緒に作る実践Webサイト（6時間）

> **このパートの目標**: AIツールを実践で活用し、実際に使えるWebサイトを作る。「指示 → 確認 → 修正」のワークフローを身につける。

### 4-1. AI Web制作ツール完全ガイド（50分）[PPT 30% + ライブ 70%]

#### Claude / ChatGPTでコード生成（20分）
- AIへの効果的な指示の出し方
  - 具体的にリクエストする（ページ構成、色、セクション指定）
  - 参考サイトのURL提示
  - 段階的にリクエストする（一度に全体 vs セクションごと）
- Claude Artifacts活用：コードプレビュー機能
- リアルタイム実演：同じリクエストをClaudeとChatGPTにそれぞれ → 結果比較

#### v0.dev 紹介（10分）[PPT]
- v0.devの画面と機能を紹介（スクリーンショットベース）
- プロンプトでUI生成の仕組み
- 制約：React基盤なので純粋なHTML/CSSとは異なる場合あり
- 無料クレジットに上限あり（月$5相当）— 本講座では使用しない

#### StackBlitz入門 — 複数ファイルで作るWeb制作（15分）[ライブ]
- PART 2〜3ではCodePenで1ペインずつ書いた → 本格的なサイトはファイルを分ける
- StackBlitz（https://stackblitz.com）にアクセス
  - GitHubアカウントでログイン（1-5でGitHubアカウント作成済み）
- StackBlitzの画面構成
  - VS Code風エディタ：ファイルツリー + エディタ + プレビュー の3カラム
  - ファイルツリー：index.html、style.css、script.js — 「実務と同じ構成」
  - ライブプレビュー：変更が即座に反映
  - プレビューURL：URLを共有すれば誰でも閲覧可能
- CodePenとの違い
  - CodePen：1ペン = 1ファイル → HTML/CSS入門に最適
  - StackBlitz：複数ファイル + フォルダ → 実際のWebサイト制作に最適
  - 「プロの開発者が使うVS Codeとほぼ同じ画面です」
- CodePenで省略されていたことをStackBlitzで体験
  - `<link rel="stylesheet" href="style.css">` — HTMLとCSSの接続を自分で書く
  - 相対パス（`./images/photo.jpg`）— ファイルツリーで実際に確認
  - ファイルツリーから画像ファイルのアップロード
- 「ここからはStackBlitzを使って、実際のWebサイトを作っていきます」

#### ローカル開発ツール紹介（5分）[PPT]
- [PPT] VS Code、Cursor — プロの現場で使われるエディタ
  - インストールが必要、豊富な機能、拡張機能で自由にカスタマイズ
  - 「この講座の後、さらに本格的に学びたくなったらぜひ試してみてください」
- 「今の私たちにはブラウザだけで十分。まずは作ることに集中しましょう」

### 4-2. プロジェクト1：個人ポートフォリオサイト（90分）[ライブ]

> 難易度：★★ | 学習ポイント：ページ全体の構成、ナビゲーション、セクション構成

#### 企画 + StackBlitz操作確認（15分）
- ポートフォリオの目的：「自分を紹介する1ページ」
- セクション構成の決定：ヒーロー、自己紹介、スキル、プロジェクト、連絡先
- 参考サイト3つを一緒に見学
- StackBlitzで新しいプロジェクトを作成（HTML/CSS/JS テンプレート選択）
  - ファイル構成確認：index.html、style.css が揃っていることを確認

#### AIで初稿生成（10分）
- Claudeに詳細なプロンプトを送信
  - 「個人ポートフォリオ。ヒーローセクションに名前と職業、スキルセクションはアイコン+説明カード4つ、プロジェクトセクションは画像+説明、お問い合わせフォーム。色はネイビー+ホワイト系。レスポンシブ対応。」
- 生成されたHTMLをStackBlitzのindex.htmlに、CSSをstyle.cssに貼り付け
- ライブプレビューで確認 — 「わあ、もうこんなに！」

#### コードを理解する（15分）
- HTML構造の把握：どのタグがどこに使われているか
- CSS分析：レイアウトはFlexboxか、色の体系は？
- 「この部分はPART 2、3で学んだことだ」— つながりの確認

#### カスタマイズ（30分）
- 自分の情報にテキスト変更
- プロフィール写真の変更
  - StackBlitzのファイルツリーに画像をドラッグ＆ドロップでアップロード → 相対パスを`src`に記述
  - 画像サイズの最適化のヒント（大きすぎる画像は表示が遅くなる）
- カラーパレットの変更

## PART 5: 世界に公開する — デプロイと運用（4時間）

> **このパートの目標**: 作ったサイトを実際にインターネットにデプロイし、ドメインを接続し、検索に表示させる。

### 5-1. デプロイ前の最終チェック（20分）[ライブ]
- StackBlitzプロジェクトのファイル/フォルダ構成の整理
- すべての画像パス確認
- 壊れたリンクの確認
- HTMLバリデーション（W3C Validator）
- 様々なブラウザでの確認（Chrome、Safari、Firefox）
- StackBlitzからGitHubへの接続
  - StackBlitzはGitHubと直接連携可能（同じGitHubアカウントでログイン済み）
  - 「Connect Repository」でGitHubリポジトリに接続
  - 接続後、メニューバーに「Commit」ボタンが表示される → 変更をGitHubに反映するには手動でCommitが必要
  - 「自動保存ではなく、自分のタイミングで"保存ボタンを押す"イメージです」

### 5-2. GitHubでコード管理 — 最小限だけ（40分）[ライブ]
- [PPT 5分] Gitとは？「コードのタイムマシン」— バージョン管理の必要性
- GitHubアカウントの確認（1-5で作成済み、CodePen・StackBlitzログインにも使用中）
- リポジトリ（Repository）作成
  - GitHub Webインターフェースで新規リポジトリ作成
  - 「リポジトリ = プロジェクトの保管場所」
- StackBlitzプロジェクトをGitHubリポジトリに接続
  - 5-1で設定した「Connect Repository」を使って、プロジェクト1（ポートフォリオ）をGitHubにCommit
  - StackBlitzのメニューバー「Commit」ボタンをクリック → GitHubに反映
  - 「自分のコードがGitHubに保存された！」
- 初めてのコミット：GitHub Web上でファイルを編集 → コミット
- 修正 → コミットの繰り返し — 「こうやって履歴が積み重なる」
- 注意：すべてブラウザ上で操作、インストール不要
- 「Gitの深い内容はこの講座の範囲外。でもこれだけで十分です」

### 5-3. GitHub Pagesで無料デプロイ（30分）[ライブ]
- リポジトリ Settings → Pages → デプロイ設定
- デプロイ完了 → 実際のURLにアクセス — 「皆さんのサイトがインターネットに公開されました！」
- GitHub Web上でファイル修正 → コミット → 自動再デプロイ確認
- プロジェクト1（ポートフォリオ）を実際にデプロイ
