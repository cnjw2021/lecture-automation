# AI時代のWeb制作マスター【基礎編】— 約10時間カリキュラム

---

## シリーズ構成

| 編 | テーマ | 時間 | 状態 |
|----|--------|------|------|
| **基礎編（本カリキュラム）** | HTML + CSS + AI活用 + デプロイ初体験 | ~10h | 制作中 |
| **応用編（別途企画）** | 複数ファイル構成、実践プロジェクト2〜3本、GitHub、SEO、運用 | ~10h | 未着手 |

> 基礎編は「20時間講座を10時間に要約」したものではない。HTML/CSSを基礎編でより丁寧に扱い、応用編でより深い内容を展開する**分権化**である。

---

## 全体構成

| パート | テーマ | 時間 | 形式 |
|--------|--------|------|------|
| **PART 1** | Webの世界へ — AI時代の第一歩 | 1.5h | PPT 60% + ライブ 40% |
| **PART 2** | HTML — Webの骨格を読む | 2.5h | PPT 20% + ライブ 80% |
| **PART 3** | CSS — Webに服を着せる | 3.0h | PPT 15% + ライブ 85% |
| **PART 4** | AIと一緒に作るサロンサイト | 2.0h | PPT 10% + ライブ 90% |
| **PART 5** | 世界に公開する | 1.1h | PPT 20% + ライブ 80% |
| **合計** | | **約10.1h** | **PPT ~20% / ライブ ~80%** |

### 基礎編の到達目標

受講完了後、受講生は以下ができるようになる：

1. HTMLとCSSの基本構造を**読んで理解**できる
2. AIに適切に指示してWebページのコードを**生成**させられる
3. AI生成物を読んで**修正・カスタマイズ**できる
4. 実際のサロン/整体院サイトを1ページ**完成**させられる
5. Netlifyで自分のサイトを**インターネットに公開**できる

### 使用ツール

| 用途 | ツール | 備考 |
|------|--------|------|
| コードエディタ | **CodePen** | 全パート通じて使用。ログインにGitHubアカウント必要 |
| AI | **Claude** / **ChatGPT** | コード生成・修正の実習 |
| デプロイ | **Netlify** | ドラッグ&ドロップで公開 |

## PART 1: Webの世界へ — AI時代の第一歩（1時間30分 / 4講義）

> **このパートの目標**: 受講生が「この講座を受ける理由」を確信し、Webの全体像をつかみ、最初のコードを書く。

### 1-1. オリエンテーション（20分）[PPT]
- 講座ロードマップの全体俯瞰
- 「コーディングを学ぶのではなく、AIと協業する方法を学ぶ」
- 基礎編の完成目標
  - AIに指示してWebページを作る力
  - 自分で作ったサロン/整体院サイト1つ
  - インターネットに公開された実際のURL
- 受講前の準備物確認（PC/Mac、インターネット、ブラウザ）

### 1-2. Webの仕組みと3つの言語（25分）[PPT + ライブ]
- [PPT] ブラウザにURLを入力すると何が起きるか（ダイアグラム）
- [PPT] サーバー、ドメイン、IPアドレス — 宅配便に例える
- [PPT] HTTPとHTTPS — 鍵アイコンの意味
- [PPT] Webサイトの3要素：HTML＝骨組み、CSS＝インテリア、JS＝家電製品
- [ライブ] 実際のWebサイトでF12開発者ツールを開いてみる
  - Yahoo! JAPAN、サロン系サイトのHTMLソースを見学
  - CSSをオフにするとどうなるか実演
  - 「これら全てがテキストファイルである」という気づきへ導く

### 1-3. AIが変えたWeb制作の風景（25分）[PPT + ライブ]
- [PPT] 2020年 vs 2025年：Webサイトを1つ作るのにかかる時間の比較
- [PPT] AI/制作ツールマップ
  - コード生成（本講座で実際に使う）：Claude、ChatGPT
  - その他（紹介のみ）：v0.dev、Bolt.new、Canva、Figma Make
  - 公開（PART 5で使う）：Netlify
- [ライブ] ライブデモ：Claudeに「マッサージサロンのホームページを作って」と言うと？
  - AIが数十秒で出す成果物を見せる
  - 「でもこれを修正するには？ここから私たちの役割が始まります」
- [PPT] AI時代のWeb制作者の役割再定義
  - コーダー → ディレクター
  - 「書く」のではなく「指示し、検収し、修正する」

### 1-4. CodePenセットアップ + Hello World（20分）[ライブ]
- GitHubアカウント作成
  - GitHub（https://github.com）にアクセス → Sign up → メール認証
  - 「このアカウントはCodePenログインに使います。大切に覚えておいてください」
- CodePen（https://codepen.io）にGitHubアカウントでログイン
- CodePenの画面構成
  - 3ペイン：HTML / CSS / JS — 「この3つがWebの3要素そのまま」
  - 下部プレビュー：コードを変えると即座に結果が反映される
- HTMLペインに「Hello World」を入力 → 即座にプレビューに表示
- `<h1>`タグで見出し、`<p>`タグで段落 → リアルタイムで変化を確認
- 「おめでとうございます、皆さんは今Web開発者になりました」
- **チャレンジ課題**：自分の名前に変えてみよう

## PART 2: HTML — Webの骨格を読む（2時間30分 / 7講義）

> **このパートの目標**: HTMLを「暗記する」のではなく「読める」状態になる。AIが生成したHTMLを見て構造を把握できる。

### 2-1. HTMLのルール：タグという約束（20分）[PPT + ライブ]
- [PPT] タグの構造：`<開くタグ>内容</閉じタグ>`
- [PPT] 属性（attribute）：タグへの追加情報
- [PPT] 親子関係：マトリョーシカ人形に例える
- [ライブ] CodePenで直接タグを作成 → 結果確認
- よくあるミス：タグの閉じ忘れ、タイプミス → エラー画面を見せる
- HTMLの基本構造の簡単な紹介
  - `<!DOCTYPE html>`、`<html>`、`<head>`、`<body>` — 「おまじない」
  - 「CodePenではこの基本構造を自動で補ってくれるので、bodyの中身だけ書けばOK」

### 2-2. テキストを扱うタグ（25分）[ライブ]
- 見出し：`<h1>` 〜 `<h6>` — 新聞記事のヘッドラインのように
- 段落：`<p>` — 本文はここに
- 強調：`<strong>`、`<em>` — ボールド、イタリック
- リスト：`<ul>`、`<ol>`、`<li>` — 順序あり/なしリスト
- 改行：`<br>` — 「Enterを押しても改行されない理由」
- 水平線：`<hr>` — 区切り線
- 実習：CodePenで自己紹介ページに見出し、紹介文、趣味リストを追加

### 2-3. リンクと画像（25分）[ライブ]
- リンク：`<a href="">` — Webの核心、ページを結ぶ糸
- 新しいタブで開く：`target="_blank"`
- 画像：`<img src="" alt="">` — セルフクロージングタグ
- 画像パスの理解
  - CodePenでは外部URL（絶対パス）を使用
  - 「相対パスは、ファイルを分けて管理するときに使います。今はURLをそのまま貼ればOK」
- 無料画像ソース：Unsplash、Pexelsの紹介
  - 画像URLを直接コピーして`src`に貼り付ける方法
- 実習：CodePenで自己紹介ページに写真（外部URL）とSNSリンクを追加

### 2-4. 構造を組み立てるタグ（25分）[PPT + ライブ]
- [PPT] セマンティックタグの概念：「機械（AI、検索エンジン）も読める構造」
- `<header>`、`<nav>`、`<main>`、`<section>`、`<article>`、`<footer>`
- `<div>` — 「意味はないが、まとめてくれる箱」
- [ライブ] CodePenで自己紹介ページをセマンティック構造にリファクタリング
- [ライブ] 構造変更の前後比較 — 「見た目は同じだが意味が違う」
- サロンサイトの構造を先取り：「ヘッダー、メニュー、アクセス、フッター — これがセマンティックタグそのもの」

### 2-5. フォーム — ユーザー入力を受け取る（20分）[ライブ]
- `<form>`、`<input>`、`<textarea>`、`<select>`、`<button>`
- inputタイプ：text、email、tel、date
- placeholder、required属性
- 実習：CodePenでシンプルな予約フォーム作成（サロンの予約を想定）
- 「このフォームを実際に動かすにはサーバーが必要です。今はHTMLで"見た目"を作る練習です」

### 2-6. テーブル基礎（15分）[ライブ]
- `<table>`、`<tr>`、`<th>`、`<td>`
- シンプルな施術メニュー・料金表を作る
- 「PART 4のサロンサイトで実際に使います」

### 2-7. AIにHTMLを任せる — 初めての実践（25分）[ライブ]
- Claude/ChatGPTに「サロンのメニューページのHTMLを作って」とリクエスト
- AIが生成したコードをCodePenに貼り付けて確認
- AIが作ったコードを1行ずつ読んでみる
  - 「このタグは何？」→ 学んだことで理解可能
  - 「これは見たことない」→ 検索/AIに聞く
- AI生成物の問題点を発見する
  - セマンティックタグの欠落
  - altテキストの不備
  - 不要なネスト
- CodePen上でAI生成物を修正する — 「これこそHTMLを学んだ理由」
- **核心メッセージ**：「AIが作ったものをそのまま使う人 vs 読んで直せる人」

## PART 3: CSS — Webに服を着せる（3時間 / 7講義）

> **このパートの目標**: CSSを「設計する」のではなく、AIが生成したCSSを「理解して調整」できる。色、サイズ、配置を自由に修正できる。

### 3-1. CSSの仕組み（20分）[PPT + ライブ]
- [PPT] CSSの3つの適用方法：インライン、内部、外部
- [PPT] 「なぜ外部CSSファイルを使うのか」— メンテナンス性
  - 実際のWebサイトでは`<link rel="stylesheet" href="style.css">`でCSSファイルを読み込む
  - 「CodePenが裏側でこの接続を自動的にやってくれている」
- [ライブ] CodePenのCSSペインを使ってスタイル適用
- [ライブ] 初めてのCSS：背景色を変える —「お、変わった！」

### 3-2. セレクタ — CSSがHTMLを見つける方法（25分）[PPT + ライブ]
- [PPT] セレクタ三銃士：タグ、クラス（`.`）、ID（`#`）
- [PPT] 「クラスはニックネーム、IDはマイナンバー」
- [ライブ] 実習：同じ`<p>`タグなのに違うスタイルを適用する
- よく使うセレクタパターン5つ
- 「AIが作ったCSSで`.hero-section`が何か、もうわかる」

### 3-3. 色とタイポグラフィ（25分）[ライブ]
- 色の表現：名前、HEX、RGB
- カラーパレットツール紹介：Coolors、Adobe Color
- フォント適用：Google Fontsの使い方
  - Google Fontsサイトにアクセス → フォント選択
  - CodePenの場合：Settings → CSS → External Stylesheets にGoogle Fonts URLを追加 → CSSで適用
  - 日本語フォント推奨：Noto Sans JP、M PLUS Rounded 1c、Zen Kaku Gothic New
- font-size、font-weight、line-height、letter-spacing
- 実習：CodePenで自己紹介ページにおしゃれなフォントと色を適用

### 3-4. ボックスモデル — CSSの核心概念（25分）[PPT + ライブ]
- [PPT] すべてのHTML要素は「箱」だ — ダイアグラム
- content、padding、border、margin
- [ライブ] 開発者ツールでボックスモデルを直接確認
- [ライブ] paddingとmarginの違いを目で見る
- box-sizing: border-box — 「これさえ知れば計算が楽になる」
- 実習：CodePenでカード型レイアウト作成

### 3-5. Flexbox — 現代Webレイアウトの核心（30分）[PPT + ライブ]
- [PPT] Flexbox以前：floatの苦しみ（昔の話、ありがたみを感じる）
- [PPT] Flexbox核心概念：コンテナとアイテム、主軸と交差軸
- [ライブ] display: flex —「1行追加するだけで魔法が起きる」
- justify-content：横並び（center、space-between、space-around）
- align-items：縦並び
- flex-direction：横↔縦切り替え
- gap：アイテム間の間隔
- flex-wrap：折り返し
- 実習：CodePenでナビゲーションバー作成
- 実習：CodePenでカード3枚を横に並べる（サロンのメニューカードを想定）
- 便利なツール紹介：Flexbox Froggyゲーム（学習用）

### 3-6. レスポンシブデザイン — モバイルでもきれいに（30分）[PPT + ライブ]
- [PPT] なぜレスポンシブが必要か — モバイルトラフィック統計
- [PPT] メディアクエリの概念：「画面サイズに応じてCSSを変える」
- [ライブ] @media 基本文法
- [ライブ] デスクトップ3列 → モバイル1列の切り替え実演
- [ライブ] ブラウザサイズを縮めながらリアルタイム確認
- viewport metaタグ — 「これを入れないとモバイルで崩れる」
  - CodePenの場合：Settings → HTML → Stuff for `<head>` にviewportタグを追加
- 実習：CodePenで自己紹介ページをレスポンシブにアップグレード
  - CodePenの「Change View」→ 新しいタブでプレビューを開き、ブラウザの開発者ツールでモバイル表示を確認

### 3-7. AIにCSSを任せる — 2回目の実践（25分）[ライブ]
- Claudeに「このHTMLにモダンなCSSを当てて」とリクエスト
- AIが生成したCSSをCodePenのCSSペインに貼り付けて確認
- AIが作ったCSSを分析する
  - セレクタ構造の把握
  - 色/フォント体系の理解
  - レイアウト方式の確認（Flexbox?）
- 「この部分が気に入らない」→ CodePen上で自分で修正
  - 色の変更、間隔の調整、フォントの変更、モバイル対応の追加/修正
- AIへの修正リクエスト：効果的な指示の出し方
  - 悪い例：「もっときれいにして」
  - 良い例：「ヘッダーの背景を#2563EBに変えて、カード間の間隔を24pxに広げて」
- **核心メッセージ**：「CSSを知っている人は、AIに正確な指示ができる」

## PART 4: AIと一緒に作るサロンサイト（2時間 / 4講義）

> **このパートの目標**: PART 2-3で学んだHTML/CSSの知識を総動員し、AIと協力して実際のサロン/整体院Webサイトを完成させる。

### 4-1. サロンサイト企画 + AI初稿生成（30分）[PPT + ライブ]
- [PPT] サロンサイトに必要な要素
  - ヒーロー（店名 + 雰囲気写真）
  - こだわり / コンセプト
  - 施術メニュー + 料金表
  - スタッフ紹介
  - アクセス / 営業時間
  - 予約 / お問い合わせ
- [PPT] 参考サイト3つを一緒に見学（実際のサロン/整体院サイト）
- [PPT] 「今回はマッサージサロンを例にしていますが、美容室なら施術メニュー、ネイルサロンならデザインギャラリー、ヨガ教室ならレッスンスケジュールに置き換えるだけです」
- [ライブ] Claudeに詳細なプロンプトを送信
  - 「マッサージサロンの1ページWebサイト。ヒーローセクション、こだわり紹介、施術メニュー表（3コース、料金付き）、スタッフ紹介カード2枚、アクセス情報と営業時間、予約フォーム。色はベージュ+深緑系。日本語フォント使用。レスポンシブ対応。」
- [ライブ] 生成されたHTML+CSSをCodePenに貼り付け → プレビューで確認
- 「わあ、もうこんなに！」

### 4-2. コードを読む + 新テクニック（30分）[ライブ]
- AI生成コードのHTML構造を把握
  - 使われているセマンティックタグの確認
  - 「この部分はPART 2で学んだ〇〇だ」— つながりの確認
- AI生成コードのCSS分析
  - レイアウト方式の確認（Flexbox）
  - 色の体系・フォント設定の確認
- PART 4で初めて出会うCSSテクニック
  - box-shadow — カードに影をつける
  - border-radius — 写真やボタンの角丸
  - hover効果 — ボタンにマウスを乗せると色が変わる
  - transition — 変化を滑らかにする
  - 「これらは全て1行追加するだけ。AIが書いたコードの中にすでに入っています」

### 4-3. カスタマイズ（35分）[ライブ]
- 自分の情報にテキスト変更
  - 店名、コンセプト、メニュー内容、スタッフ名
- 画像の変更
  - Unsplash/Pexelsからサロン系の無料写真を取得
  - 外部画像URLをsrcに貼り付け
- カラーパレットの変更
  - Coolorsで配色を選ぶ
  - CSS変数またはカラーコードを一括置換
- フォント変更（Google Fonts）
- 間隔/サイズの微調整
  - padding、margin、font-sizeの値を変えて結果を確認
- メニュー料金表の修正（テーブルの内容変更）
- AIに追加修正を依頼
  - 「予約ボタンをもっと目立たせて」
  - 「営業時間の部分にGoogleマップの埋め込みを追加して」

### 4-4. 仕上げ + レビュー（25分）[ライブ]
- モバイル確認
  - CodePenの「Change View」→ 新しいタブでプレビュー → 開発者ツールでモバイル表示
  - レスポンシブの問題があればAIに修正依頼
- 全体チェック
  - リンクが正しく動作するか
  - 画像が全て表示されているか
  - テキストに誤りがないか
- CodePen上でコード整理
- 完成したサイトのスクリーンショットを撮る — 達成感
- 「このサイトを次のPART 5で実際にインターネットに公開します」

## PART 5: 世界に公開する（1時間5分 / 3講義）

> **このパートの目標**: 自分が作ったサロンサイトを実際にインターネットに公開し、URLを手に入れる。

### 5-1. CodePenからファイルへ — StackBlitzで変換（15分）[ライブ]
- 「CodePenは学習に最適でしたが、公開するには"本物のファイル"にする必要があります」
- StackBlitz（https://stackblitz.com）にGitHubアカウントでログイン
  - 「PART 1で作ったGitHubアカウントを使います」
- StackBlitzで新しいHTML/CSS/JSプロジェクトを作成
- StackBlitzの画面を簡単に紹介
  - 左側：ファイルツリー（index.html、style.css）— 「これが本物のファイル構成です」
  - 右側：プレビュー
  - 「今日はCodePenのコードをファイルにするために使います。操作は簡単なので安心してください」
- CodePen → StackBlitzへコードを移す
  - CodePenのHTMLペインの内容 → StackBlitzの`index.html`の`<body>`内にペースト
  - CodePenのCSSペインの内容 → StackBlitzの`style.css`にペースト
  - `<link rel="stylesheet" href="style.css">` — 「CodePenが裏で自動的にやってくれていた接続です」
  - CodePen Settingsに入れていたGoogle Fonts URL → `<head>`に`<link>`タグで追加
  - viewport metaタグの追加
- StackBlitzのプレビューで表示確認 — 「CodePenと同じ見た目ですね」
- StackBlitzからzipダウンロード — 「これがNetlifyにアップロードするファイルです」

### 5-2. Netlifyでデプロイ（25分）[ライブ]
- Netlifyアカウント作成（メールアドレスで登録）
- ドラッグ&ドロップデプロイ
  - ダウンロードしたzipを解凍
  - Netlifyダッシュボード → フォルダをドラッグ&ドロップ
  - 数秒でデプロイ完了 → 実際のURLにアクセス
  - 「皆さんのサイトがインターネットに公開されました！」
- 公開したURLをスマートフォンで開いてみる — レスポンシブ確認
- 修正が必要な場合
  - CodePenでコード修正 → 5-1の変換手順を繰り返し → Netlifyに再デプロイ
  - 「Netlifyは何度でも上書きデプロイできます」
- [PPT 3分] カスタムドメインの紹介
  - ドメインとは？「住所のようなもの」
  - 「興味がある方はドメイン購入サービスで取得できます」

### 5-3. まとめと次のステップ（25分）[PPT]
- [PPT] 基礎編で学んだことの全体振り返り
  - Webの仕組み → HTML → CSS → AI活用 → サイト完成 → 公開
